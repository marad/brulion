## Design

Chisel-scale: extract the shared action list, add a selection-driven toolbar
ViewPlugin (mirroring the context-menu's DOM/teardown pattern), style it, wire it.

New `src/format-actions.ts`:
- `export interface MenuItem { label: string; run: (state: EditorState) => TransactionSpec | null }`.
- `export const FORMAT_ITEMS: MenuItem[]` — the seven actions currently inlined in
  context-menu.ts (Bold/Italic/Code via `toggleInline`, H1–H3 via `setHeadingLines`,
  Clear via `clearFormatting`).

`src/context-menu.ts`: delete the local `MenuItem`/`ITEMS`, import them from
`./format-actions`; `itemsFor` keeps prepending the wikilink toggle to `FORMAT_ITEMS`.

New `src/selection-toolbar.ts`:
- `const TOUCH = window.matchMedia("(pointer: coarse), (max-width: 40rem)")` — the
  touch/narrow gate.
- A `ViewPlugin` that, on `update` (selectionSet/docChanged/focusChanged), shows a
  single floating toolbar element when `TOUCH.matches`, the view has focus, and the
  main selection is non-empty; otherwise hides it. Buttons are built from
  `FORMAT_ITEMS` (compact labels: B, I, code, H1, H2, H3, Clear); each
  `mousedown`/`pointerdown` calls `preventDefault()` (keep the selection), and `click`
  dispatches `item.run(view.state)` then refocuses. Positioned above the selection's
  head via `view.coordsAtPos`, clamped into the viewport (like the context menu).
  Torn down on `destroy` (element + listeners removed).
- `export const selectionToolbar: Extension = ViewPlugin...`.

`src/styles.css`: `.cm-selection-toolbar` — a small floating pill of buttons
(position: fixed, z-index above content, shadow), matching the context-menu look.

`src/editor.ts`: add `selectionToolbar` to the extensions (next to `contextMenu`).

## Implementation steps (chisel — test before body)

1. e2e first (`e2e/selection-toolbar.spec.ts`, red): at a **narrow** viewport (375px,
   so the gate matches) open a folder and type text; double-click a word to select it
   → assert the toolbar appears (AC-1); tap Bold → `start.md` shows `**word**` (AC-2);
   select a line and tap H2 → `## …` (AC-3); collapse the selection → toolbar gone
   (AC-1). A **desktop** case (1280px, fine pointer) selects text and asserts the
   toolbar never appears (AC-4).
2. Extract `format-actions.ts`, refactor `context-menu.ts`, implement
   `selection-toolbar.ts`, style, wire `editor.ts`.
3. Unit (`src/format-actions.test.ts`): `FORMAT_ITEMS` has the seven expected labels,
   and `run` for Bold/H2 over a sample selection returns the same transform the
   transforms module produces (AC-5) — the shared-source guarantee. (The toolbar's
   show/hide/positioning is layout-dependent → covered by e2e, like the context menu.)
