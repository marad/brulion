## Design

A new `src/settings.ts` owns the model, the disk IO, and the apply layer (pure
functions + two async IO functions + one DOM-effecting `applySettings`). The editor
theme is taught to read two CSS vars; `main.ts` loads/applies settings on folder
open and routes the `Ctrl/Cmd+;` Vim toggle through the settings; the idb Vim
accessors are deleted.

`src/settings.ts` (new module — built with `excavate`):
- `EditorWidth = "narrow" | "wider" | "full"`.
- `interface Settings { font: string[]; textSize: number; editorWidth: EditorWidth; vim: boolean }`.
- `DEFAULT_SETTINGS: Settings = { font: [], textSize: 16, editorWidth: "narrow", vim: false }`.
- `SETTINGS_FILE = ".brulion.json"`, `GENERIC_FLOOR = "sans-serif"`, `MIN_SIZE = 12`,
  `MAX_SIZE = 24`, and a `WIDTH_MEASURE: Record<EditorWidth,string>` =
  `{ narrow: "68ch", wider: "90ch", full: "none" }`.
- `normalizeSettings(raw: unknown): Settings` — pure/total. Reads fields off a
  non-null object; `textSize` → `Math.round` then clamp to [MIN,MAX] (NaN → 16);
  `editorWidth` kept only if it's one of the three literals else `"narrow"`; `font`
  → `Array.isArray` then `.filter(x => typeof x === "string")` else `[]`; `vim` →
  `Boolean(raw.vim)`. Non-object input → `{ ...DEFAULT_SETTINGS }`.
- `buildFontStack(fonts: string[]): string` — pure. Each family needing quoting
  (`/[^A-Za-z0-9-]/`, i.e. spaces or punctuation) wrapped in double quotes, others
  bare; comma-join in order; append `GENERIC_FLOOR`. Empty list → `GENERIC_FLOOR`.
- `loadSettings(dir): Promise<Settings>` — `getFileHandle(SETTINGS_FILE)` →
  `getFile()` → `text()` → `JSON.parse` → `normalizeSettings`. Any throw (absent
  file, bad JSON) → `{ ...DEFAULT_SETTINGS }`.
- `saveSettings(dir, settings): Promise<void>` — `getFileHandle(SETTINGS_FILE,
  {create:true})` → `createWritable()` → write `JSON.stringify(settings, null, 2)`
  → `close()`. Root-level only (no subfolder traversal).
- `applySettings(view, settings, root = document.documentElement): void` — sets
  `--editor-font-size = ${textSize}px`, `--editor-measure = WIDTH_MEASURE[width]`;
  if `font.length` set `--font-stack = buildFontStack(font)` else
  `root.style.removeProperty("--font-stack")`; then `setVimMode(view, settings.vim)`.
  Imports `setVimMode` from `./editor` (no cycle — editor doesn't import settings).

`src/editor.ts` theme tweak (two values, var-driven with the current defaults as
fallbacks so the no-folder look is unchanged):
- `"&"`: `fontSize: "var(--editor-font-size, 16px)"` (was `"16px"`).
- `.cm-content`: `maxWidth: "var(--editor-measure, 68ch)"` (was `"68ch"`).

`src/session.ts`: delete `VIM_KEY`, `saveVimMode`, `loadVimMode` (Vim leaves idb).

`src/main.ts` rewire:
- imports: drop `saveVimMode`/`loadVimMode`; add
  `{ loadSettings, saveSettings, applySettings, DEFAULT_SETTINGS, type Settings } from "./settings"`.
- module state: `let currentSettings: Settings = DEFAULT_SETTINGS`,
  `let settingsDir: FileSystemDirectoryHandle | null = null`.
- `const persistSettings = () => settingsDir ? saveSettings(settingsDir, currentSettings) : Promise.resolve()`.
- `const syncVimButton = () => toggleVimEl.setAttribute("aria-pressed", String(currentSettings.vim))`.
- `const updateSettings = (patch: Partial<Settings>) => { currentSettings = { ...currentSettings, ...patch }; applySettings(view, currentSettings); syncVimButton(); void persistSettings() }`.
- `const toggleVim = () => updateSettings({ vim: !currentSettings.vim })`.
- in `openNote(dir)`, before `controller.open(dir)`:
  `currentSettings = await loadSettings(dir); settingsDir = dir; applySettings(view, currentSettings); syncVimButton()`.
- replace the `void loadVimMode().then(...)` block: wire the header Vim button's
  click to `toggleVim` and bind the `Ctrl/Cmd+;` capture-phase shortcut (same guard:
  `(ctrl||meta) && !alt && !shift && code==="Semicolon" && workspaceShown`) to
  `toggleVim`. (The button stays in P1; it's removed in P2.)
