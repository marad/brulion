# Sync plan — FEAT-0016 Block-construct rendering

Started: 2026-06-20T16:20:00.673Z
Snapshot state: new
Drift summary: 0 added (whole spec)

All work extends the existing decoration layer in `src/markdown-render.ts`. Today
`markdownSyntaxRanges(state, from, to)` returns `{ hidden, marks }`; this phase
adds a third output, `lines: LineMark[]` (`{ from: <line-start pos>, cls }`), for
block-level styling a span/replace decoration can't express (a code block's
background, a quote's border, a list bullet). The view plugin turns each into a
`Decoration.line({class})` at the line start and folds them into the same
decoration set (sorted by `Decoration.set(arr, true)`); line decos coexist with
the existing replace/mark decos. The function stays pure and `[from, to)`-scoped.

Handling is dispatched in the existing `iterate(...).enter(node)` by node name,
alongside the current heading/inline path:

- **FencedCode** — collect the `CodeMark` children. Only collapse when the block
  is *closed* (two CodeMarks); an unclosed block being typed stays fully visible
  (same philosophy as a bare `#` heading — don't vanish what's mid-typing). Hide
  the opening fence line including its trailing newline (`[openLine.from,
  openLine.to + 1)`) and the closing fence line including its leading newline
  (`[closeLine.from - 1, closeLine.to)`). If the two hide ranges would overlap (an
  empty block with no body line), emit a single merged hide over the whole node
  instead. Emit a `cm-code-block` line mark for each body line strictly between
  the fences (clamped to `[from, to)`).
- **Blockquote** — walk the node's whole subtree for every `QuoteMark` descendant
  (they nest inside the inner `Paragraph` on continuation lines). Hide each
  `QuoteMark` plus one trailing space if present. Emit a `cm-blockquote` line mark
  for each line the blockquote covers (clamped).
- **ListItem** (under `BulletList` only — skip `OrderedList`) — hide the
  `ListMark` plus one trailing space. Emit a `cm-list-item` line mark on the
  item's first line; the bullet glyph is drawn by CSS `::before`, so the collapsed
  `*`/`-` is replaced on screen by a real bullet.

Hidden runs continue to feed `EditorView.atomicRanges`, so the caret steps over
the collapsed block markers atomically (AC-7), and the plugin still rebuilds only
on doc/viewport change, never on caret move, so block markup stays hidden on the
caret line (AC-5). `markdownSyntaxRanges` never mutates the document, so bytes
round-trip verbatim (AC-6).

New `EditorView.baseTheme` rules in the same file style `.cm-code-block` (monospace
+ subtle background, spanning lines), `.cm-blockquote` (left border + muted), and
`.cm-list-item::before` (the bullet).

## AC-1 (added) — Fenced-code fences hidden, body styled as a code block

FencedCode handling above hides both fence lines and emits `cm-code-block` line
marks for the body. Unit-tested via `markdownSyntaxRanges`: a closed block hides
both fence lines (incl. language tag) and the body lines carry the code-block
line mark. e2e: a block renders with no ```` ``` ```` on screen and `.cm-code-block`
present with a monospace font-family.
Files: `src/markdown-render.ts`, `src/markdown-render.test.ts`, `e2e/rendering.spec.ts`

## AC-2 (added) — Blockquote marker hidden, content styled as a quote

Blockquote handling hides the leading `> ` and emits a `cm-blockquote` line mark.
Unit-tested: `> quoted` hides `> ` and the line carries the blockquote line mark.
e2e: rendered text has no `>` and `.cm-blockquote` is present.
Files: `src/markdown-render.ts`, `src/markdown-render.test.ts`, `e2e/rendering.spec.ts`

## AC-3 (added) — Every marker hidden across a multi-line blockquote

The subtree walk collects the `QuoteMark` on the first line and the nested
continuation `QuoteMark`s. Unit-tested: a two-line `> a\n> b` hides both `> `
markers.
Files: `src/markdown-render.ts`, `src/markdown-render.test.ts`

## AC-4 (added) — Unordered-list marker hidden, bullet rendered

ListItem handling hides the `* `/`- ` marker and emits a `cm-list-item` line mark;
CSS `::before` draws the bullet. Unit-tested: `* item` hides `* ` and the line
carries the list-item line mark. e2e: rendered text has no leading `*`, the item
text is present, and a bullet `::before` is rendered.
Files: `src/markdown-render.ts`, `src/markdown-render.test.ts`, `e2e/rendering.spec.ts`

## AC-5 (added) — Block markup stays hidden on the caret line

The plugin rebuilds only on doc/viewport change and never special-cases the caret
line, so block markers stay hidden when the caret is on the line. e2e: place the
caret on a bullet line, assert the rendered text still has no `*`.
Files: `src/markdown-render.ts`, `e2e/rendering.spec.ts`

## AC-6 (added) — Rendering does not alter the saved document

`markdownSyntaxRanges` only reads state. e2e: type a code block + a quote + a
bullet, Ctrl+S, assert on-disk `start.md` equals the exact CommonMark typed.
Files: `src/markdown-render.ts`, `e2e/rendering.spec.ts`

## AC-7 (added) — Hidden block markers are atomic to the caret

Hidden runs are provided as `EditorView.atomicRanges` (unchanged plumbing, now
fed the block-marker runs too). e2e: on a `* item` line, Home then ArrowRight
jumps the whole `* ` run so an inserted char lands before the text, not between
`*` and the space.
Files: `src/markdown-render.ts`, `e2e/rendering.spec.ts`

## AC-8 (added) — Plain prose and out-of-scope constructs untouched

Only FencedCode/Blockquote/BulletList-ListItem are handled; `OrderedList` and
plain paragraphs hit no branch. Unit-tested: a plain paragraph and a `1. item`
ordered-list line yield empty `hidden`/`marks`/`lines`.
Files: `src/markdown-render.ts`, `src/markdown-render.test.ts`
