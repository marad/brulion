## Approach

New pure module **`src/tree-nav.ts`** holds the testable core: given the ordered
list of *visible* rows (each `{ path, kind, expanded, depth }`, depth = count of
`/` in the path) and the current index, `resolveTreeKey(key, rows, current)`
returns an action ‚Äî `{type:"focus",index}` (Up/Down/Home/End, and Right‚Üífirst
child / Left‚Üíparent), `{type:"expand"|"collapse",index}` (Right on a collapsed
folder / Left on an expanded one), `{type:"activate",index}` (Enter/Space), or
`{type:"none"}` (no-op at an end, Right on a note, Left at the root). No DOM, no
FSA ‚Äî unit-tested directly (AC-1..AC-7 movement/expansion logic; AC-2 no-wrap).

DOM glue in **`src/ui.ts`**:
- A single `keydown` handler on the list container (wired once, like the root
  drop zone) reads the visible rows from the DOM (`.folder-header`/`.note-name`
  focusables, skipping any inside a `hidden` `.folder-children`), derives their
  descriptors, calls `resolveTreeKey`, and executes: `focus` ‚Üí focus the target
  + move the roving tab stop; `expand`/`collapse` ‚Üí `.click()` the folder header
  (reusing FEAT-0043's toggle + `onToggleFolder` persistence ‚Äî AC-9); `activate`
  ‚Üí `.click()` the focused row (open note / toggle folder ‚Äî AC-7).
- **Roving tabindex** (AC-8): a helper sets `tabindex="-1"` on every row
  focusable and `"0"` on one ‚Äî the active note's row, or the first row when none
  ‚Äî called on each render and updated as focus moves. A folder header gains a
  `data-path` so its descriptor is readable; note rows already carry `data-path`.
- Movement/expansion never call add/remove/move/rename ‚Äî AC-10 holds by
  construction (the handler only focuses elements and clicks
  header/name buttons, whose click paths are select/toggle, never a file write).

Files: `src/tree-nav.ts` (new), `src/tree-nav.test.ts` (new), `src/ui.ts`,
`src/ui.test.ts`, `e2e/tree-keyboard-nav.spec.ts` (new).

AC-9 (persistence) and AC-7/AC-8/AC-10 (real focus/tab semantics) are verified
in the Playwright e2e; the pure movement logic and the roving-tabindex DOM
wiring are covered by vitest.
