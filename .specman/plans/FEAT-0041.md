# Sync plan — FEAT-0041 Rebase a moved note's own outbound links

## Approach

A pure rebase function plus a one-call driver in the existing rename flow. Reuses
the FEAT-0040 path primitives; no new module.

### Pure: `rebaseOutboundLinks` (AC-1..AC-4) — `src/link-rewrite.ts`

NEW export `rebaseOutboundLinks(text, oldPath, newPath): string | null`. Reuses the
private `markdownLinkDests(text)` iterator (extracted in this change, shared with
`rewriteLinksForRename`), `isExternalDest`, `applyEdits`, plus `resolveNotePath` /
`relativeLink` (FEAT-0040). For each inline markdown destination:
- skip external (`isExternalDest`);
- `resolved = resolveNotePath(oldPath, dest)`; skip if `null` (escapes root / not a
  note);
- self-link follows the note: `target = resolved === oldPath ? newPath : resolved`
  (else a self-link rebases to the stale old path and dangles);
- skip if `resolveNotePath(newPath, dest) === target` (still resolves from the new
  location — the same-folder case → AC-1);
- else edit the URL span to `markdownDest(relativeLink(newPath, target))` (AC-2/3).
Wikilinks are not scanned at all (AC-4: untouched). Round-trip:
`resolveNotePath(newPath, rebased) === target`.

### Controller: `rebaseMovedNote` in `renameActive` (AC-5/AC-6) — `src/note-controller.ts`

NEW local helper run after `moveNote` success and BEFORE `activate`, inside the
serialize slot, wrapped best-effort (a throw never fails the rename):
- `if (folderOf(from) === folderOf(to)) return` — a same-folder rename can't break
  the moved note's own relative links, so skip the file read entirely (AC-6).
  `folderOf` is a new module-level helper (folder portion of a path, `""` at root).
- else `readNote(dir, to)` → `rebaseOutboundLinks(content, from, to)` →
  if non-null `saveNote(dir, to, rebased, lastModified)` (stale-write guarded;
  conflict skipped). `activate(dir, to)` then loads the rebased content (AC-5).
