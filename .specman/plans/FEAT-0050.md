## Design

Chisel-scale: add `@codemirror/lang-yaml`, a small pure helper + a few lines in
`frontmatter.ts`'s expanded branch, reusing M15 P1's `tok-*` CSS palette
(`codeTokenTheme`, already mounted via `markdownRendering`) — so no new theme.

`src/frontmatter.ts`:
- `export function frontmatterYamlMarks(state, range: FrontmatterRange): {from,to,cls}[]`
  — pure. Compute the inner YAML range: the lines between the opening `---` (line at
  `range.from`) and the closing delimiter line (line at `range.to`). If there are no
  inner lines, return `[]`. Slice that text, parse it with
  `yamlLanguage.parser.parse(text)`, run `highlightTree(tree, classHighlighter, …)`,
  and push `{from: innerFrom + f, to: innerFrom + t, cls}` (offset back to absolute
  document positions). Error-tolerant: the lezer YAML parser never throws (AC-5).
- In `buildFrontmatter`, **expanded branch only**, after the per-line decorations,
  push a `Decoration.mark({ class: cls }).range(from, to)` for each
  `frontmatterYamlMarks(state, r)` entry. `Decoration.set(deco, true)` already sorts.
  The collapsed branch is unchanged (chip replaces the block — nothing to color).

The marks carry the same `tok-*` classes M15 P1 emits, so the existing
`codeTokenTheme` colors them — frontmatter and code blocks stay visually consistent
and will pick up a future (M18) theme together. Nothing is hidden or made atomic;
no bytes change.

Note: the YAML grammar tags keys, quoted strings, and comments (plain scalars stay
plain) — identical to a fenced ```` ```yaml ```` block, which is the intended
consistency.

## Implementation steps (chisel — test before body)

1. e2e first (`e2e/frontmatter-yaml.spec.ts`, red): seed a note with a `---…---` block
   (key + quoted value + `#` comment); open the folder. Collapsed by default → assert
   the metadata chip is shown and there are **no** `tok-*` spans (AC-2). Click the chip
   to expand → assert `tok-*` token spans now appear over the YAML (AC-1). Save and
   assert the bytes are unchanged (AC-3).
2. Add `@codemirror/lang-yaml`; implement `frontmatterYamlMarks` + wire the expanded
   branch.
3. Unit (`src/frontmatter.test.ts` add cases): `frontmatterYamlMarks` over a doc with
   frontmatter returns marks (a `tok-propertyName…` over a key, a `tok-comment` over a
   `#` line) within the inner range; an empty-body block (`---\n---`) yields `[]`;
   malformed YAML still returns without throwing (AC-5).
