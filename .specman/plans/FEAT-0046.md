## Design

Chisel-scale: one new thin side-effect module + a one-line wire, reusing the
FEAT-0045 `serializeCopy`. No new serialization logic, no new contract.

New `src/vim-yank.ts`:
- `installVimMarkdownYank()` — calls `Vim.defineOperator("yank", fn)` once. The
  override mirrors the stock yank operator
  (`@replit/codemirror-vim` `operators.yank`) exactly, except the text:
  - `text = serializeCopy(view.state, view.state.selection.ranges)` where
    `view = cm.cm6` (the package documents the live selection as matching the
    operator's input range, so this equals the stock `cm.getSelection()` — repaired).
  - `endPos` replicated: visual mode → `cursorMin(vim.sel.anchor, vim.sel.head,
    ranges[0].head, ranges[0].anchor)` (small local `cursorMin`/`cursorIsBefore` on
    `{line, ch}`); else `oldAnchor`.
  - push via `Vim.getRegisterController().pushText(args.registerName, "yank", text,
    args.linewise, vim.visualBlock)` — same routing (unnamed/`0`/named/`+` clipboard).
  - Guarded so it installs once (module-level boolean).

Wire: call `installVimMarkdownYank()` at module load in `editor.ts` (next to the
vim imports). Global + idempotent; only ever invoked while Vim mode is active.

## Implementation steps

1. e2e first (`e2e/vim-yank.spec.ts`): enable Vim (model on `e2e/vim.spec.ts`),
   type a heading, `Esc`, visual-select the visible text (`v$` or `0v$`), `y`, open
   a new line, `p`, assert the pasted line is `# …`. Second case: a bold word →
   `**…**`. (Red before the override is wired.)
2. Implement `src/vim-yank.ts` and wire into `editor.ts`.
3. Confirm the override delegates to `serializeCopy` (already unit-tested in
   `src/copy-markdown.test.ts`); no new unit test for the serializer itself.
