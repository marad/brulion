# Sync plan — FEAT-0020 Collapsible note sidebar

Started: 2026-06-20T23:23:10.994Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Add a sidebar collapse toggle as DOM glue (`ui.ts`) plus a persisted boolean
(`session.ts`) — no new module (chisel). Collapse is a CSS class on `.workspace`
(`sidebar-collapsed`), kept separate from the `#sidebar[hidden]` attribute that
already encodes folder-open, so the two states are orthogonal (AC-6). The toggle
button lives in the `<header>` (outside the sidebar), so it stays reachable when
the sidebar is collapsed (AC-2). A `Ctrl+\` shortcut (no existing editor/CM
binding uses it) toggles the same state via a window keydown listener — it
bubbles past CodeMirror, which has no binding for it, so the editor shortcuts and
Enter are untouched (AC-3). The collapsed flag persists via the existing
`idb-keyval` layer under `brulion:sidebar-collapsed` (AC-4); nothing is written to
the user's folder (AC-5).

Files: `index.html` (toggle button), `src/styles.css` (collapsed layout + button),
`src/session.ts` (`saveSidebarCollapsed`/`loadSidebarCollapsed`), `src/ui.ts`
(`wireSidebarToggle`), `src/main.ts` (load flag, wire button + shortcut, reveal
toggle on folder open), `src/session.test.ts`, `src/ui.test.ts`,
`e2e/sidebar.spec.ts`.

`wireSidebarToggle(button, workspace, { initialCollapsed, onChange })` owns the
boolean: applies the class + `aria-pressed` on wire (reflecting the restored
state), flips on click, calls `onChange(collapsed)`, and returns `{ toggle() }`
for the keyboard shortcut. Pure DOM — unit-tested in happy-dom without mocks.

## AC-1 (added) — A control collapses and restores the sidebar

Clicking the toggle adds/removes `.sidebar-collapsed` on `.workspace`; CSS
`.workspace.sidebar-collapsed #sidebar { display: none }` hides the list and the
editor (`flex: 1`) fills the width. Unit: click toggles the class both ways. e2e:
open folder, click toggle → sidebar hidden; click again → visible.
Files: `index.html`, `src/styles.css`, `src/ui.ts`, `src/ui.test.ts`,
`e2e/sidebar.spec.ts`

## AC-2 (added) — The toggle stays reachable when collapsed

The button is in `<header>`, not in `#sidebar`, so collapsing the sidebar never
hides it. e2e: after collapse, the toggle is still visible and re-expands.
Files: `index.html`, `e2e/sidebar.spec.ts`

## AC-3 (added) — A keyboard shortcut toggles without clashing

`Ctrl+\` on `window` (CM has no binding for it so it bubbles) calls
`sidebar.toggle()` and `preventDefault()`s. e2e: confirm a Ctrl+B bold still works
in the editor, then `Ctrl+\` toggles the sidebar.
Files: `src/main.ts`, `e2e/sidebar.spec.ts`

## AC-4 (added) — The collapsed state persists across a reload

`onChange` persists via `saveSidebarCollapsed`; on load `loadSidebarCollapsed`
seeds `initialCollapsed`. Unit (session): round-trip + default `false`. e2e:
collapse, reload, folder re-opens → sidebar still collapsed.
Files: `src/session.ts`, `src/main.ts`, `src/session.test.ts`,
`e2e/sidebar.spec.ts`

## AC-5 (added) — Collapse does not write to the user's folder

The toggle only sets a class and writes the `idb-keyval` key; no folder write.
e2e: collapse, then assert `start.md` bytes are unchanged from what was typed.
Files: `e2e/sidebar.spec.ts`

## AC-6 (added) — Collapse and folder-open state are independent

`#sidebar[hidden]` (folder closed) and `.workspace.sidebar-collapsed` (preference)
are separate; opening a folder sets `hidden=false` but the collapsed class still
hides the sidebar. Unit: applying collapsed toggles only the `.workspace` class,
never the `#sidebar` `hidden` attribute. e2e: collapse, reload → still collapsed
after the folder re-opens.
Files: `src/ui.ts`, `src/ui.test.ts`, `e2e/sidebar.spec.ts`
