# Sync plan — FEAT-0021 Opt-in Vim mode

Started: 2026-06-20T23:35:58.712Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Wire `@replit/codemirror-vim`'s `vim()` through a new CodeMirror compartment in
`editor.ts` (mirroring the existing read-only `editable` compartment) — no new
module. The compartment is placed **first** in the extensions array, so the Vim
ViewPlugin's keydown handler runs at higher precedence than the editor's keymaps.
Verified against the library source: Vim binds `<CR>`, `<C-b>`, `<C-e>`, `<C-i>`
only in `normal` context, so in insert mode it ignores them and they fall through
to our completion/format/Enter keymaps; in normal mode Vim owns them (the point of
opting in). `vim()`'s `hideNativeSelection` is scoped to the compartment, so with
Vim off the editor config is byte-identical to today.

`setVimMode(view, on)` reconfigures the compartment (`on ? vim() : []`). The choice
persists via `session.ts` (`saveVimMode`/`loadVimMode`, key `brulion:vim`, default
false). A header toggle button (revealed with the others on folder open) flips it.

Reuse: the sidebar's `wireSidebarToggle` is generalized to a `wireToggle(button,
{ initialOn, apply, onChange })` helper (applies `apply(on)` on wire + each flip,
persists via `onChange` only on a user flip, reflects `aria-pressed`) so the Vim
toggle and the sidebar toggle share one tested mechanism instead of duplicating it.
The sidebar's `apply` sets the `.sidebar-collapsed` class; Vim's calls
`setVimMode`. FEAT-0020's behavior is unchanged (a pure refactor; its spec/snapshot
is untouched, and its unit + e2e tests still pass).

Files: `src/editor.ts` (vim compartment, `setVimMode`), `src/ui.ts` (`wireToggle`,
drop `wireSidebarToggle`), `src/session.ts` (`saveVimMode`/`loadVimMode`),
`src/main.ts` (wire both toggles via `wireToggle`, apply initial Vim state),
`index.html` (Vim toggle button), `src/styles.css` (pressed state),
`src/session.test.ts`, `src/ui.test.ts`, `e2e/vim.spec.ts`.

## AC-1 (added) — Vim mode is off by default

`loadVimMode` defaults to false; the compartment holds `[]`. e2e (fresh profile):
click editor, press `j` → a literal `j` is inserted (no motion), and `i` inserts
`i` (no mode change).
Files: `src/session.ts`, `src/editor.ts`, `e2e/vim.spec.ts`

## AC-2 (added) — A control toggles Vim mode on and off

Header button flips `setVimMode`. e2e: enable Vim, press Esc then `j`/`k` → cursor
moves without inserting; the `.cm-vimMode` class / mode indicator is present;
toggle off → default model returns.
Files: `index.html`, `src/styles.css`, `src/ui.ts`, `src/main.ts`,
`e2e/vim.spec.ts`

## AC-3 (added) — The Vim choice persists across a reload

`onChange` persists via `saveVimMode`; on load `loadVimMode` seeds the initial
compartment + toggle state. Unit (session): round-trip + default false. e2e:
enable Vim, reload → Vim still on (motions work after Esc).
Files: `src/session.ts`, `src/main.ts`, `src/session.test.ts`, `e2e/vim.spec.ts`

## AC-4 (added) — With Vim on, the slash menu still works

Vim leaves `/` and Enter alone in insert mode. e2e: Vim on, in insert mode type
`/h2` + Enter → the line becomes an H2.
Files: `e2e/vim.spec.ts`

## AC-5 (added) — With Vim on, the format shortcuts still work

Vim binds Ctrl+B/E/I only in normal context. e2e: Vim on, select a word in insert
mode, Ctrl+B → `**word**` on save.
Files: `e2e/vim.spec.ts`

## AC-6 (added) — With Vim on, the markdown-aware Enter still works

Vim binds `<CR>` only in normal context. e2e: Vim on, type `* a` in insert mode,
Enter, `b`, save → `* a\n* b` (FEAT-0018 continuation, not a bare Vim newline).
Files: `e2e/vim.spec.ts`

## AC-7 (added) — With Vim on, normal-mode commands are active

e2e: Vim on, type `hello`, Esc, `0` then `x` → deletes the first char (`ello`),
proving the Vim layer is engaged.
Files: `e2e/vim.spec.ts`

## AC-8 (added) — Toggling Vim does not write to the user's folder

The toggle reconfigures the editor + writes the `idb-keyval` key; no folder write.
e2e: type + save a note, toggle Vim, assert `start.md` bytes unchanged.
Files: `e2e/vim.spec.ts`
