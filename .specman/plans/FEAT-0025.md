# Sync plan — FEAT-0025 Links between notes

Started: 2026-06-21T10:26:50.199Z
Snapshot state: new
Drift summary: 0 added (whole spec)

## Approach (whole spec)

No new module. Pure link-path logic joins `note-name.ts` (already the home of
path normalization); `markdown-render.ts` gains link decorations fed by a new
`linkContext` facet; `editor.ts` exposes the facet (compartment) and a
Ctrl/Cmd+click handler; `main.ts` wires the context and the follow behavior;
`styles.css` styles links. Following routes through the existing controller.

### AC-1, AC-2, AC-3 — pure resolution + classification (`note-name.ts`)
NEW `isExternalLink(href)`: true for a `scheme:` prefix (`/^[a-z][a-z0-9+.-]*:/i`)
or protocol-relative `//` — so `https://…`, `mailto:…` are external, `sub/b.md`
is not (AC-3). NEW `resolveNotePath(fromNote, href): string | null`: strip a
surrounding `<>`; start from `fromNote`'s folder segments; fold `href`'s segments
(`""`/`.` skipped, `..` pops — returning `null` if it pops above the root); join
with `/`; return `null` unless the result ends in `.md` (AC-1, AC-2). Pure, no
DOM/FSA, unit-tested.

### AC-4, AC-5 — render links (`markdown-render.ts`)
NEW `linkContext` facet: `Facet<LinkContext, LinkContext>` with
`LinkContext = { activeNote: string; notePaths: ReadonlySet<string> }`, combine →
first value or an empty default. Extend `MarkRange` with optional
`attrs?: Record<string,string>`; `buildDecorations` passes `attributes: m.attrs`
to `Decoration.mark`. In `markdownSyntaxRanges`, read `state.facet(linkContext)`
and handle `node.name === "Link"`: collect `LinkMark` children + the `URL` child;
require a `URL` and ≥4 marks (an inline `[text](url)`; skip shortcut `[x]`); with
non-empty text, hide `[` (marks[0]) and `](url)` (marks[1].from → last mark.to),
and emit a mark over the text `[marks[0].to, marks[1].from]` with class
`cm-link` plus `cm-link-broken` when `isExternalLink(url)` is false and
`resolveNotePath(ctx.activeNote, url)` is null or not in `ctx.notePaths`; attrs
`{ "data-href": url }`. The inline `renderPlugin.update` also rebuilds when the
`linkContext` facet changes (so broken-state tracks the folder), and the hidden
link runs join the existing atomic hidden set.

### AC-6, AC-7, AC-8 — follow on Ctrl/Cmd+click (`editor.ts` + `main.ts`)
`editor.ts`: a `linkCtx` compartment holding `linkContext.of(EMPTY)`, exported
`setLinkContext(view, ctx)` to reconfigure it; `EditorOptions.onFollowLink?(href)`;
and `EditorView.domEventHandlers({ mousedown })` that, when `event.metaKey ||
event.ctrlKey` and the target is within `[data-href]`, reads the href, calls
`onFollowLink`, and returns true (prevent caret move). `main.ts`: track
`currentActive`/`currentNotes` from `onListChanged`, call `setLinkContext(view,
{ activeNote: active, notePaths: new Set(notes) })` there, and implement
`onFollowLink(href)`: external → `window.open(href, "_blank", "noopener")` (AC-8);
else `resolveNotePath(currentActive, href)` — null → ignore; in `currentNotes` →
`controller.switchTo(target)` (AC-6); otherwise `window.confirm(...)` then
`controller.addNote(target)` (AC-7).

### styling (`styles.css`)
`.cm-link` (link color + underline, normal cursor) and `.cm-link-broken` (muted
red / dashed underline) under the editor theme — add to `renderTheme` in
`markdown-render.ts` (where the other `.cm-*` content styles live) for cohesion.

### Tests
- `note-name.test.ts`: `isExternalLink` (AC-3) and `resolveNotePath` (AC-1/AC-2:
  relative, `..`, `./`, escape→null, non-`.md`→null, `<>`-wrapped).
- `markdown-render.test.ts`: with a `linkContext`-configured state, a link hides
  `[`/`](url)` and styles the text `cm-link` with `data-href` (AC-4); a missing
  target adds `cm-link-broken` (AC-5); a shortcut `[x]` and external link behave
  (external not broken).
- NEW `e2e/links.spec.ts`: Ctrl+click an internal link switches notes (AC-6);
  Ctrl+click a missing link, accept the prompt, the note is created+opened (AC-7);
  an external link's `[data-href]` carries the URL and is classed `cm-link`
  (AC-8 — opening a real new tab isn't asserted; the classification/handler path
  is).
