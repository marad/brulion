# Sync plan — FEAT-0018 Markdown-aware Enter: continue and exit lists/quotes

Started: 2026-06-20T19:31:05.828Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Bind Enter to `insertNewlineContinueMarkup` from `@codemirror/lang-markdown` in
`src/editor.ts`. It continues lists/blockquotes and removes the marker on an empty
item (the exit). Place it in the keymap **after** `completionKeymap` and **before**
`defaultKeymap`, so: when the slash popup is open the completion's Enter accepts
the option first; otherwise the markdown command handles Enter (continue/exit);
and on a plain line it returns false and falls through to `defaultKeymap`'s normal
newline. No new module — a one-line keymap addition reusing the library command.

Tested via Playwright (real CodeMirror Enter handling and the real save path),
since this is editor-key behavior happy-dom can't exercise faithfully.

## AC-1 (added) — Enter continues an unordered list

`insertNewlineContinueMarkup` continues `*`/`-` items. e2e: type `* a`, Enter,
`b`, save → `* a\n* b`.
Files: `src/editor.ts`, `e2e/list-quote-continuation.spec.ts`

## AC-2 (added) — Enter continues a blockquote

Same command continues `>`. e2e: type `> a`, Enter, `b`, save → `> a\n> b`.
Files: `src/editor.ts`, `e2e/list-quote-continuation.spec.ts`

## AC-3 (added) — Enter on an empty list item exits the list

e2e: type `* a`, Enter (continues to `* `), Enter (empty → exit), `b`, save →
`* a\nb` (no stray `* ` line, `b` on a plain line).
Files: `src/editor.ts`, `e2e/list-quote-continuation.spec.ts`

## AC-4 (added) — Enter on an empty blockquote line exits the quote

e2e: type `> a`, Enter, Enter, `b`, save → `> a\nb`.
Files: `src/editor.ts`, `e2e/list-quote-continuation.spec.ts`

## AC-5 (added) — Enter on a plain line is unchanged

The command returns false on a non-list/quote line, so `defaultKeymap` inserts a
normal newline. e2e: type `a`, Enter, `b`, save → `a\nb`.
Files: `src/editor.ts`, `e2e/list-quote-continuation.spec.ts`

## AC-6 (added) — The slash menu still accepts on Enter

`completionKeymap` precedes the new binding, so its Enter accepts when the popup is
open. e2e: type `/h1`, press Enter → the line becomes a heading (not a newline).
Files: `src/editor.ts`, `e2e/list-quote-continuation.spec.ts`
