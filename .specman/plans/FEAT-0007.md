# Sync plan — FEAT-0007 Editor formatting shortcuts

Started: 2026-06-20T07:11:10.612Z

Drift summary: 0 added (whole spec)

All formatting logic is pure transforms in `src/markdown-transforms.ts`
(`toggleInline`, `promoteHeading`/`demoteHeading`/`setHeading`,
`headingLevelOf`/`withHeadingLevel`) on `(EditorState, selection) →
TransactionSpec | null`. `src/markdown-commands.ts` binds them via a high-
precedence keymap; thin `command()` adapters dispatch the spec (skipping null
no-ops but still consuming the key). Added to the editor in `src/editor.ts`.
Heavy correctness coverage is in vitest unit tests; e2e proves the keys fire in
real Chromium and edit the on-disk markdown.

## AC-1/AC-2/AC-3/AC-4/AC-8 — Inline toggles (bold/italic/code)

`toggleInline` wraps the selection in the marker, or unwraps every same-type
span intersecting the selection (using the parsed tree, so `*`-in-`**` is never
mis-detected and an overlap never emits invalid `****`). Empty selection inserts
the pair with the caret between. Unit-tested in `markdown-transforms.test.ts`;
e2e in `shortcuts.spec.ts` (AC-1/2/3/4).
Files: `src/markdown-transforms.ts`, `src/markdown-commands.ts`, `src/markdown-transforms.test.ts`, `e2e/shortcuts.spec.ts`

## AC-5/AC-6/AC-7 — Heading level cycle and direct set

`promotedLevel`/`demotedLevel` give the paragraph↔H3↔H2↔H1 cycle; `changeHeading`
rewrites only the line's `#` prefix and returns null when unchanged. `Ctrl+↑/↓`
and `Ctrl+Shift+1/2/3` bound at high precedence (verified not shadowed by a
default — e2e drives them in real Chromium).
Files: `src/markdown-transforms.ts`, `src/markdown-commands.ts`, `src/markdown-transforms.test.ts`, `e2e/shortcuts.spec.ts`

## AC-9 — Ctrl+U unsupported

No `Mod-u` binding exists; e2e confirms Ctrl+U leaves the document unchanged (no
HTML, no `__`).
Files: `src/markdown-commands.ts`, `e2e/shortcuts.spec.ts`
