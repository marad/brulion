# Sync plan — FEAT-0074 Text inputs don't invite password-manager autofill suggestions

Started: 2026-07-15T06:55:42.501Z
Snapshot state: new
Drift summary: 0 added (whole spec)

## Implementation

Whole-spec plan (brand-new). Pure attribute additions — no behavior change,
no new module.

### 1. `index.html` — four static inputs

`#switcher-input`, `#palette-input`, `#move-input`, `#dialog-input` each
gain `data-lpignore="true" data-1p-ignore data-bwignore="true"
data-form-type="other"` alongside their existing `autocomplete="off"`.

### 2. `src/ui.ts` — the header's inline note-rename input

The rename input built in `mountNoteIdentity` (already sets
`input.autocomplete = "off"`) gains the same four `data-*` attributes via
`setAttribute`.

### 3. `src/settings-modal.ts` — the journal-path input

`journalInput` currently has no `autocomplete` at all — gains
`autocomplete="off"` plus the same four `data-*` attributes.

### 4. Tests

A small `hasAntiAutofillAttrs(el)` helper (used across
`quick-switcher.test.ts`/`command-palette.test.ts`/`move-picker.test.ts`/
`dialog.test.ts`/`settings-modal.test.ts`/`ui.test.ts` — wherever each input
already has element-lookup coverage) asserts the five attributes on the
relevant input in each file's existing mount, rather than one new test file.
