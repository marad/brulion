## Design

New module `src/copy-markdown.ts` (excavate it), plus a small wiring change in
`src/editor.ts`.

**Core (pure):** `serializeCopy(state: EditorState, ranges: readonly {from,to}[]): string`
â€” maps each range to `prefix + sliceDoc(from,to) + suffix` and joins with
`state.lineBreak`. The two repairs:

- `lineMarkerPrefix(state, from)` â†’ the leading block-marker run of the line at
  `from`, read verbatim, when `from` is at/past the end of that run (and `from` is
  past the line start). Detect via the markdown syntax tree: a `HeaderMark`,
  `QuoteMark`(s), or `BulletList > ListItem > ListMark` at the line start whose run
  (with its single trailing space, or the contiguous `>` runs for quotes) ends at
  or before `from`. Returns "" when the first line is a plain paragraph or fully
  selected from its start. First selected line only.
- Inline repair via the syntax tree: walk ancestors of `from` (open side) and `to`
  (close side). For each `StrongEmphasis`/`Emphasis`/`InlineCode` ancestor:
  - open side: if its opening mark ends at/before `from` and `from` is before its
    closing mark, the span's opening delimiter (verbatim) is missing â†’ prepend.
    Collected outermost-first (ascending node.from).
  - close side: if its closing mark starts at/after `to` and `to` is after its
    opening mark, the closing delimiter is missing â†’ append. Collected
    innermost-first (descending node.from).
  `prefix = lineMarkerPrefix + inlineOpens.join("")`, `suffix = inlineCloses.join("")`.

**Wiring:** `EditorView.domEventHandlers({ copy, cut })` in editor.ts. On a
non-empty selection: build text via `serializeCopy(view.state, view.state.selection.ranges)`,
`event.clipboardData.setData("text/plain", text)`, `event.preventDefault()`, and
for `cut` dispatch a transaction deleting `selection.ranges` (userEvent "delete").
Empty selection â†’ return false (default behavior). Export the handler extension and
add it to the editor extension list.
