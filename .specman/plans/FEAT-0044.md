# Sync plan — FEAT-0044 Drag-to-resize sidebar width

Started: 2026-06-23T12:03:46.100Z
Snapshot state: new
Drift summary: 0 added (whole spec)

## Approach

A draggable handle on the sidebar/editor border drives the sidebar's `flex-basis`
via a CSS custom property, persisted as a pixel width. Small change in the
existing sidebar layer (chisel) — a pure clamp + a `wireSidebarResize` glue
function in `ui.ts` (home of the other wiring helpers), a new HTML handle element,
a CSS var basis, and two session helpers. No new module.

### Markup — `index.html`

- Add `<div id="sidebar-resizer" role="separator" aria-orientation="vertical"
  aria-label="Resize sidebar" hidden></div>` between `</aside>` (the sidebar) and
  `<div id="editor">`, so it is a flex child of `.workspace` sitting on the border.
  `hidden` initially (revealed by `showWorkspace`, like the sidebar itself).

### Styles — `src/styles.css` (AC-4, AC-6)

- `#sidebar` flex basis becomes a var: `flex: 0 0 var(--sidebar-width, 14rem)`
  (default unchanged). `min-width: 0` and the collapse rule are untouched.
- `#sidebar-resizer { flex: 0 0 5px; cursor: col-resize; background: transparent }`
  with a subtle `:hover` hairline (e.g. `background: #e0e0dd`) so the grab area is
  discoverable. `touch-action: none` so a pointer drag isn't stolen by scroll.
- `.workspace.sidebar-collapsed #sidebar-resizer { display: none }` — gone when
  collapsed (AC-6). The `hidden` attribute (UA `display: none`) covers the
  no-folder state; our ID rule sets no `display`, so it doesn't override `hidden`.

### Pure clamp — `src/ui.ts` (AC-1, AC-2, AC-3)

```ts
export const SIDEBAR_MIN_PX = 144
export const SIDEBAR_MAX_PX = 560
export function clampSidebarWidth(px: number): number {
  return Math.min(SIDEBAR_MAX_PX, Math.max(SIDEBAR_MIN_PX, px))
}
```
Pure. `Math.max(MIN, NaN)` is `NaN`, so guard non-finite explicitly → `MIN`
(a corrupt value yields a usable sidebar). Unit-tested directly.

### Resize glue — `src/ui.ts` (AC-4)

```ts
export function wireSidebarResize(
  handle: HTMLElement,
  sidebar: HTMLElement,
  opts: { initialWidth: number | null; onChange: (px: number) => void },
): void
```
- `setWidth(px)` = `sidebar.style.setProperty("--sidebar-width", \`${px}px\`)`.
- If `initialWidth !== null`, `setWidth(clampSidebarWidth(initialWidth))` on wire
  (AC-5 restore through the clamp).
- `pointerdown`: `preventDefault()`; record `startX = e.clientX` and
  `startWidth = sidebar.getBoundingClientRect().width`; `setPointerCapture`;
  add `pointermove`/`pointerup` on the handle; set `document.body.style.userSelect
  = "none"` for the drag.
- `pointermove`: `setWidth(clampSidebarWidth(startWidth + ev.clientX - startX))`
  — delta-based, independent of the sidebar's left offset.
- `pointerup`: detach move/up, restore `userSelect`, and
  `opts.onChange(clampSidebarWidth(startWidth + ev.clientX - startX))` to persist.

### Reveal with the workspace — `src/ui.ts` `showWorkspace`

- Add `resizer: HTMLElement` to `WorkspaceRefs` and set `refs.resizer.hidden =
  false` in `showWorkspace` (called on every `onListChanged`; idempotent).

### Persistence — `src/session.ts` (AC-5)

- `const SIDEBAR_WIDTH_KEY = "brulion:sidebar-width"`.
- `saveSidebarWidth(px: number): Promise<void>` → `set(KEY, px)`.
- `loadSidebarWidth(): Promise<number | null>` → returns the stored number, or
  `null` when unset / not a finite number (defensive: a corrupt value falls back
  to the default basis).

### Wiring — `src/main.ts` (AC-5, AC-6)

- `const sidebarResizerEl = document.querySelector<HTMLElement>("#sidebar-resizer")`
  added to the mount-point null-check and to the `showWorkspace` refs object.
- After load: `void loadSidebarWidth().then((px) => wireSidebarResize(
  sidebarResizerEl, sidebarEl, { initialWidth: px, onChange: (w) => void
  saveSidebarWidth(w) }))`.

### Tests

**Unit — `src/ui.test.ts`**: `clampSidebarWidth` — in-range unchanged (AC-1),
below-min / 0 / negative → min (AC-2), above-max → max (AC-3), non-finite → min.
Plus a glue check: `wireSidebarResize` with `initialWidth` sets the
`--sidebar-width` custom property to the clamped px (AC-5 restore path), and with
`null` leaves it unset.

**Unit — `src/session.test.ts`**: `saveSidebarWidth`/`loadSidebarWidth`
round-trip; `null` when unset; `null` for a non-number stored value.

**E2E — `e2e/sidebar.spec.ts`** (AC-4, AC-5, AC-6, AC-7 against the real drag):
drag the `#sidebar-resizer` right → the sidebar widens (bounded box width grows,
within max); the new width survives a reload (auto-restored folder); the resizer
is hidden while collapsed and reappears when expanded; dragging leaves `start.md`
bytes untouched.
