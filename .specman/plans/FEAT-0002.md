# Sync plan — FEAT-0002 Folder access

Started: 2026-06-19T20:14:20.203Z
Snapshot state: new
Drift summary: 0 added (whole spec)

New module `src/fs.ts` holds the File System Access logic; `src/main.ts` grows
the button wiring + list rendering; `index.html` gains the button and list
elements. FSA types come from `@types/wicg-file-system-access` (devDep) —
`showDirectoryPicker` and the directory-handle async iterators are not in the
stock TS DOM lib.

## AC-1 (added) — An explicit control opens the directory picker

Approach: NEW CODE. `src/fs.ts` exports `pickFolder(): Promise<FileSystemDirectoryHandle | null>`
which calls `window.showDirectoryPicker({ mode: "readwrite" })`. `index.html`
gets a `<button id="open-folder">Open folder</button>`; `src/main.ts` adds a
click listener that calls `pickFolder()`. The "called with `{ mode: 'readwrite' }`
from a gesture" contract is unit-tested on `pickFolder` (mocked picker); the
button-to-picker wiring is confirmed by browser verification (a picker can only
open from a real user gesture).
Files: `src/fs.ts`, `src/main.ts`, `index.html`, `package.json` (devDep)

## AC-2 (added) — A picked folder's `.md` files are listed, and only those

Approach: NEW CODE. `src/fs.ts` exports `listMarkdownFiles(dir): Promise<string[]>`
that iterates `dir.values()`, keeps entries with `kind === "file"` whose name
ends in `.md` (case-insensitive), and returns them sorted via `localeCompare`.
`main.ts` renders the names into `<ul id="file-list">` (one `<li>` each, via
`replaceChildren`). Unit-tested with a fake directory handle yielding mixed
files + sub-directories.
Files: `src/fs.ts`, `src/main.ts`, `index.html`

## AC-3 (added) — An empty (no-markdown) folder lists nothing, without error

Approach: Falls out of `listMarkdownFiles` returning `[]` and `renderFileList`
clearing the list to empty. No error path is taken. Unit-tested with a folder
that has only non-markdown entries.
Files: (covered by AC-2 code)

## AC-4 (added) — Dismissing the picker is a no-op

Approach: `pickFolder` wraps `showDirectoryPicker` in try/catch; a
`DOMException` named `AbortError` returns `null` (other errors rethrow). The
click handler returns early on `null`, leaving the list untouched. Unit-tested
by mocking the picker to reject with an `AbortError`.
Files: (covered by AC-1 code)
