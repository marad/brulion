# Sync plan — FEAT-0067 Preserve scroll and caret on external refresh

Started: 2026-06-25T16:14:10.672Z

Drift summary: 0 added (whole spec)

Implemented via `/chisel` (a small pure helper + editor glue + a one-line swap in the
controller). One genuinely new unit — a prefix/suffix diff function — but it's a single
pure function, so chisel over excavate.

## AC-3 (minimal diff) — new helper

Approach: NEW `src/text-diff.ts` — `diffRange(old, next): TextChange | null` returning
`{from, to, insert}` for the single middle span that differs (longest common prefix +
longest common suffix, suffix clamped so it can't overlap the prefix), or `null` when
`old === next`. Pure, allocation-light, no library. Thoroughly unit-tested (prefix-only,
suffix-only, middle, insertion, deletion, full replace, identical, empty↔nonempty).

Files: src/text-diff.ts, src/text-diff.test.ts

## AC-1 / AC-2 (scroll + caret preserved) — minimal reload

Approach: NEW `reloadEditorText(view, text)` in `src/editor.ts`. Computes
`diffRange(currentDoc, text)`; returns immediately if `null` (no-op). Otherwise:
captures the top-of-viewport document position (`view.posAtCoords` at the scroller's
top edge, falling back to `view.viewport.from`), builds the change, maps the anchor
through it (`ChangeSet.mapPos`), and dispatches the `changes` + a
`scrollIntoView(mappedAnchor, {y:"start"})` effect, carrying the same `ProgrammaticLoad`
annotation `setEditorText` uses. No explicit `selection` — CodeMirror maps the existing
caret through the change (AC-2). The caret/no-collapse behavior is unit-testable in
happy-dom (dispatch works without layout); the scroll restoration is covered by e2e in
real Chromium.

Files: src/editor.ts, src/editor.test.ts

## AC-1 / AC-4 (controller uses it; guards unchanged) — swap the safe reload call

Approach: in `src/note-controller.ts` `refreshFromDisk`, swap the single
`setEditorText(view, note.content)` on the safe-to-replace path for
`reloadEditorText(view, note.content)`. Everything around it — `safeToReplaceBuffer`,
the post-read re-check, `raiseConflict`, the deleted-note branch — is untouched, so the
conflict guard (AC-4) is preserved. Initial load / note-switch keep `setEditorText`.

Files: src/note-controller.ts

## AC-5 (no note bytes change) — property

Approach: property — the reload only dispatches editor changes; no `.md` write path is
touched. Covered by the existing external-refresh bytes assertions plus a new e2e that
asserts scroll/caret are preserved AND the file content is what the external writer
wrote (no echo-write back).

Files: (none — property)
