# Sync plan — FEAT-0022 Conflict diff view

Started: 2026-06-21T07:44:35.646Z
Snapshot state: new
Drift summary: 0 added (whole spec)

## Approach (whole spec)

New module `src/conflict-view.ts` wrapping `@codemirror/merge`'s `MergeView`,
plus wiring it into the existing conflict modal and feeding it the two versions
from the controller.

### AC-2 — controller hands the UI both versions
`src/note-controller.ts`: change `onConflict?` from `() => void` to
`(versions: { mine: string; theirs: string | null }) => void`. Make
`raiseConflict` async: set `conflict = true` first (so the save loop / change
guard see it immediately), capture `mine = view.state.doc.toString()`, then read
the disk via `readNote(dir, activeName)` and set `theirs = disk.lastModified ===
null ? null : disk.content` (`null` mtime = deleted on disk), then call
`opts.onConflict?.({ mine, theirs })`. Await it at the three call sites: the
`doSave` conflict branch and the two `refreshFromDisk` branches (all already in
async contexts). `mine` is captured before the await and `conflict` is set before
it, so a keystroke during the read can't change what is diffed and is blocked by
`handleChange`'s `if (conflict) return`.

### AC-1, AC-3, AC-6 — the diff view (read-only, side-by-side, deleted case)
NEW `src/conflict-view.ts`: `mountConflictDiff(parent, mine, theirs):
ConflictDiff` where `ConflictDiff = { destroy(): void }`. Builds a
`.conflict-diff` container with a labels row ("Your version" | "On disk", the
disk label gaining " (deleted on disk)" when `theirs === null`) and a `MergeView`
under it: `a = { doc: mine }` on the left, `b = { doc: theirs ?? "" }` on the
right, both with read-only extensions (`EditorState.readOnly.of(true)`,
`EditorView.editable.of(false)`, `EditorView.lineWrapping`). MergeView highlights
the changed lines by default (AC-1); the deleted case diffs the buffer against an
empty disk doc, so the whole buffer reads as removed (AC-3); read-only extensions
satisfy AC-6 (the panes never edit). `destroy()` calls `mergeView.destroy()` and
removes the container.

### AC-4 — resolving tears down the view
`src/main.ts`: hold `let conflictDiff: ConflictDiff | null`. In `onConflict`,
after un-hiding the backdrop and setting the editor read-only, mount the diff into
a `#conflict-diff` container and store the handle. In `onConflictResolved`,
destroy it and null the handle alongside hiding the backdrop / restoring
editable. `index.html`: add `<div id="conflict-diff"></div>` inside `#conflict`
between the message `<p>` and `.conflict-actions`. `src/styles.css`: style the
container (bounded height, the MergeView scrolls), the labels row, and the
read-only panes.

### AC-5 — FEAT-0015 modality and semantics preserved
No change to the resolution path: `resolveKeepMine` / `resolveTakeTheirs`,
modality, refused navigation, and read-only editor are untouched. The only
controller change is the `onConflict` payload (AC-2). Covered by re-running the
existing FEAT-0015 controller tests unchanged, plus an assertion that the payload
shape doesn't alter resolution behavior.
