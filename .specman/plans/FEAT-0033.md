## Approach (whole spec)

Two new modules + wiring; no note-model code changes. Built via /excavate
(top-down: diagram → stubs → tests → bodies).

### `src/fuzzy.ts` — pure ranking (unit-tested)
- `export interface Ranked { path: string; score: number }`
- `export function fuzzyScore(query: string, target: string): number | null` —
  case-insensitive subsequence match of `query` in `target`; `null` if not a
  subsequence; higher score for contiguous runs / start-of-string / segment-start
  hits. Empty query → score 0 (matches all).
- `export function rankNotes(query: string, paths: readonly string[]): Ranked[]` —
  map each path's display form (strip `.md`) through `fuzzyScore`, drop nulls, sort
  by score desc then path asc. Empty query → all paths, name order.

### `src/quick-switcher.ts` — the overlay (DOM glue)
`export interface QuickSwitcherDeps {`
`  getNotes: () => readonly string[]   // current paths (.md)`
`  openNote: (path: string) => void    // → controller.switchTo`
`  createNote: (name: string) => Promise<{ ok: boolean; reason?: string }>  // → controller.addNote`
`}`
`export interface QuickSwitcher { open(): void; close(): void; destroy(): void; isOpen(): boolean }`
`export function mountQuickSwitcher(backdrop: HTMLElement, input: HTMLInputElement,`
`  list: HTMLElement, error: HTMLElement, deps: QuickSwitcherDeps): QuickSwitcher`
- `open()`: clear input/error, `backdrop.hidden = false`, render all notes,
  highlight first, focus input.
- On input: `rankNotes(query, getNotes())`; render rows (display name). If the
  trimmed query is non-empty and matches no existing display path exactly, append a
  "Create «query»" row. Highlight resets to first row.
- Keyboard (on the input): ArrowDown/Up move highlight (clamp, no wrap), Enter
  activates the highlighted row, Esc closes. Activation: a note row →
  `openNote(path)` + close; the Create row → `await createNote(query)`; on `ok`
  close, else show `reason` in `error` and keep open.
- Mouse: click a row activates it; click on the backdrop (target === backdrop)
  closes; clicks inside the dialog don't.
- Rows are `<button class="switch-row">`; the highlighted one carries
  `.active` + `aria-selected` and is scrolled into view.
