# Sync plan — FEAT-0011 Note list and switching

Started: 2026-06-20T12:03:28.316Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Turn the single-note controller into one that tracks an **active note** among
many, render the folder's notes as a clickable list, and remember the active
note across reloads. No new module: the list renders from `ui.ts` (the existing
UI/DOM module, already unit-tested via `ui.test.ts`), persistence reuses
`session.ts`'s idb-keyval pattern (one more key beside the folder handle). Tests
before bodies throughout.

Design:
- `note-controller.ts` gains `activeName` + a cached `notes: string[]`, and a
  pure exported `pickActiveNote(names, persisted)` policy. `open(dir)` lists the
  folder, picks the active note, loads it, persists it, and reports the list via
  a new `onListChanged(notes, active)` callback. New `switchTo(name)` flushes the
  open note (awaited, through the one guarded save), then loads `name`, resets
  save state, and persists it. `doSave` writes to `activeName`; after a save that
  materializes a not-yet-listed note (the lazy `start.md`), it re-lists and fires
  `onListChanged` so the new file appears.
- `session.ts` gains `saveActiveNote(name)` / `loadActiveNote()` on key
  `brulion:active`.
- `ui.ts` gains `renderNoteList(container, notes, active, onSelect)` — one row per
  note (display name = filename without `.md`), the active row marked, click →
  `onSelect(filename)`.
- `index.html` adds a `#note-list` aside next to `#editor`; `styles.css` lays
  them side by side and styles the active row.
- `main.ts` wires `onListChanged` → `renderNoteList(..., name =>
  controller.switchTo(name))`.

## AC-1 — The folder's notes are listed
`open(dir)` calls `listNotes` (FEAT-0010) and reports the result via
`onListChanged`; `renderNoteList` builds one row per note showing the name
without `.md`, in the already case-insensitive order from `listNotes`.
Files: `src/note-controller.ts`, `src/ui.ts`, `src/ui.test.ts`, `index.html`, `src/styles.css`, `src/main.ts`

## AC-2 — The open note is marked active
`renderNoteList` adds an active marker (class + `aria-current`) to the row whose
filename equals the active name. Unit-tested in `ui.test.ts`.
Files: `src/ui.ts`, `src/ui.test.ts`

## AC-3 — Clicking a note switches the editor to it
Each row's click calls `onSelect(filename)` → `controller.switchTo(name)`, which
loads the note's content into the editor and re-reports the list with the new
active name (so the marked row moves). Clicking the active note is a no-op.
Files: `src/note-controller.ts`, `src/note-controller.test.ts`, `src/ui.ts`, `src/main.ts`

## AC-4 — Switching flushes the previous note's pending edits
`switchTo` first awaits a drain of the open note's pending edits through the
existing guarded save loop (cancel the debounce, await `doSave` to completion)
before changing `activeName` and loading the new note — so A's unsaved text is
written to A's file first. Covered by a controller unit test that leaves A dirty,
switches, and asserts the save to A happened before B loaded.
Files: `src/note-controller.ts`, `src/note-controller.test.ts`

## AC-5 — Edits after a switch save to the newly active note
After `switchTo(b)`, `doSave` uses the updated `activeName`, so `saveNote` is
called with B's filename. Unit test: switch to B, type, assert `saveNote` called
with B.
Files: `src/note-controller.ts`, `src/note-controller.test.ts`

## AC-6 / AC-7 — Active note restored on reopen; graceful fallback
`open` reads the persisted active name (`loadActiveNote`) and runs
`pickActiveNote(names, persisted)`: persisted-if-present, else `start.md`-if-
present, else first note, else `start.md` (empty folder). `pickActiveNote` is a
pure exported function, unit-tested for all branches. e2e covers reopen-restores.
Files: `src/note-controller.ts`, `src/note-controller.test.ts`, `src/session.ts`, `src/session.test.ts`

## AC-8 — Empty folder still seeds `start` lazily
On an empty folder `pickActiveNote` yields `start.md`; `open` loads it as an empty
buffer and writes nothing (FEAT-0004 preserved). After the first save creates
`start.md`, `doSave` notices it wasn't in the cached `notes`, re-lists, and fires
`onListChanged` so it appears as the active row. Controller unit test + e2e.
Files: `src/note-controller.ts`, `src/note-controller.test.ts`
