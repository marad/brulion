# Sync plan — FEAT-0065 Light dark theme

Started: 2026-06-25T13:33:44.200Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Implemented via `/chisel`. A semantic CSS-variable palette (light defaults + dark
overrides + a `prefers-color-scheme` media query for `system`), the colors migrated
from literal hex; a `theme` setting; a settings-modal picker; an apply step setting
the `data-theme` root attribute. The editor theme reads the same vars.

## AC-1 (added) — theme persists + normalizes

Approach: add `theme: "light"|"dark"|"system"` to `Settings`, default `"system"`;
`normalizeSettings` coerces anything else to `"system"`. Rides load/save.

Files: src/settings.ts

## AC-2/AC-3/AC-4 (added) — dark / light / system apply correctly

Approach: `styles.css` `:root` defines the light palette tokens (today's colors) +
`color-scheme: light`; `:root[data-theme="dark"]` and `@media (prefers-color-scheme:
dark) { :root:not([data-theme]) }` define the dark palette + `color-scheme: dark`. An
`applyTheme(theme)` (in `main.ts`, called from `applySettings`/`updateSettings`) sets
`document.documentElement.dataset.theme` to `light`/`dark`, or removes it for
`system`. The hex colors across `styles.css` are migrated to `var(--token)`.

Files: src/styles.css, src/main.ts, src/settings.ts

## AC-5 (added) — settings modal picks the theme

Approach: a Theme radio group (Light / Dark / System) in the settings modal, emitting
`{ theme }` through the existing `onChange`; `seed()` reflects the current value.

Files: src/settings-modal.ts

## AC-6 (added) — the editor themes with the app

Approach: the editor theme (`editor.ts` `EditorView.theme`) and the inline/block
render colors (`markdown-render.ts` `baseTheme`) use `var(--…)` for
background/text/selection/code/link colors, so the editing surface follows the theme.

Files: src/editor.ts, src/markdown-render.ts

`.md` write. Verified by an e2e (switch theme, assert note bytes unchanged + the dark
background applied).
