# Sync plan — FEAT-0026 Link interaction and editing

Started: 2026-06-21T11:55:05.732Z
Snapshot state: new
Drift summary: 0 added (whole spec)

## Approach (whole spec)

No new module. `markdown-render.ts` grows autolink rendering + caret-reveal +
hover title and turns on the GFM Autolink parser extension; `editor.ts` flips the
click model and toggles a modifier-held class; `main.ts` opens external links via
an anchor. The pure resolution helpers (`note-name.ts`) are unchanged.

### AC-6, AC-7 — bare web-URL autolinks (emails excluded)
Enable the parser's autolinker: `markdown({ addKeymap: false, extensions: [Autolink] })`
(`Autolink` from `@lezer/markdown`) — only Autolink, not the rest of GFM. A bare
URL then parses as a standalone `URL` node (verified: `https://…`, `www.…`, and
`a@b.com` all become `URL`). In `markdownSyntaxRanges`, handle a `URL` node whose
`parent.name !== "Link"` (a Link's own `(url)` is handled by `collectLink`): if its
text matches `/^(https?:\/\/|www\.)/i` style it `cm-link` over `[from,to]` with
`data-href`/`title` = the url (a `www.` url normalized to `https://www.…` so the
follow path treats it external); otherwise (email, etc.) emit nothing — left plain
(AC-7). No hidden runs: the URL text *is* the visible link.

### AC-8 — caret-reveal of `[text](url)` markup
`markdownSyntaxRanges` reads `state.selection.main.head`; `collectLink` takes it and,
when the caret is within the link's outer span `[open.from, lastMark.to]`, returns
without emitting hide/mark — so the raw `[text](url)` shows for editing. The inline
`renderPlugin` adds `update.selectionSet` to its rebuild trigger so moving the caret
in/out re-renders. This reveal is links-only: bold/heading logic ignores the caret,
so they keep always-hiding (no regression to FEAT-0006 / the "no flicker" rule for
non-links). Autolinks have no hidden markup, so nothing to reveal there.

### AC-9 — hover tooltip with the target
`collectLink` sets `title` in the mark's `attrs`: the resolved note path
(`resolveNotePath(activeNote, href) ?? href`) for an internal link, the url for an
external one; the autolink handler sets `title` = the url. Rendered as the span's
native `title` (already supported via `MarkRange.attrs`).

### AC-1, AC-2, AC-3, AC-4 — click model (flip) + reliable open
`editor.ts` `domEventHandlers.mousedown`: **return false when `metaKey||ctrlKey`**
(let CodeMirror place the caret — the edit path, AC-4), otherwise if the target is
within `[data-href]` `preventDefault` + `opts.onFollowLink(href)` + return true
(plain click follows). With the caret-reveal, a click *inside* an already-revealed
link hits raw text (no `[data-href]`) and just places the caret — fine.
`main.ts` `onFollowLink`: internal existing → `switchTo`; internal missing →
confirm + `addNote`; external → open via a programmatic anchor (`a.href`,
`target=_blank`, `rel="noopener noreferrer"`, `a.click()`) instead of
`window.open(_, _, "noopener")` (AC-2, reliable new tab).

### AC-5 — cursor reflects the modifier
`renderTheme` (markdown-render.ts): `.cm-link { cursor: pointer }` and
`.cm-mod-held .cm-link { cursor: text }`. `editor.ts`: after building the view, add
`window` keydown/keyup/blur listeners that toggle `view.dom.classList`'s
`cm-mod-held` while Control/Meta is held (cleared on blur so a modifier released
outside the window doesn't stick).

### Tests
- `markdown-render.test.ts`: a bare `https://…`/`www.…` styles `cm-link` with the
  (normalized) href + title and no hidden runs (AC-6); a bare email is left plain
  (AC-7); with the caret inside a `[text](url)` link, `markdownSyntaxRanges` emits
  no hide/mark for it, and with the caret outside it does (AC-8); the link mark
  carries a `title` (AC-9). (Selection is set via `EditorState.create({ selection })`.)
- NEW `e2e/link-interaction.spec.ts`: plain-click an internal link switches notes
  (AC-1); plain-click an external link calls the anchor/open path and doesn't switch
  (AC-2, stub the anchor click / capture target); Ctrl/Cmd+click does not switch and
  the link reveals its raw `](` markup (AC-4 + AC-8); a typed bare URL renders
  `.cm-link` (AC-6).
