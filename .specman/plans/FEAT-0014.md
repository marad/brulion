# Sync plan — FEAT-0014 Live non-conflicting refresh from disk

Started: 2026-06-20T14:27:21.916Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Wire FEAT-0013's detection to action for the non-conflicting cases. No new
module — a new controller method + small `main.ts` wiring → **chisel** scope
(`src/note-controller.ts`, `src/main.ts`, their tests + an e2e).

**Controller refactor.** Extract the probe out of `checkDisk` so detection and
application can share one serialize slot:
- private `probeDisk(folder): Promise<DiskCheck>` — the `listNotes` + `statNote`
  + `classifyDiskCheck` core (no serialize, no folder/save guards).
- `checkDisk()` keeps its current contract: `serialize(() => folder? && !savePromise ? probeDisk : EMPTY)`.

**New `refreshFromDisk(): Promise<void>`** on the controller (added to the
interface), run in one `serialize` slot so detect+apply are atomic:
- no folder or `savePromise` in flight → return (AC-6).
- `const check = await probeDisk(folder)`.
- list: if `check.listChanged`, set `notes = check.listChanged` (mark the list
  needs re-render).
- active, only when `!check.active.dirty && !conflict` (the safe path; the
  dirty/conflict case is FEAT-0015, left to the existing save-time guard):
  - `kind:"changed"` → `readNote(folder, activeName)`, set `lastModified`,
    `setEditorText(view, content)`, `dirty = false` (AC-3: adopt the new mtime).
  - `kind:"deleted"` → switch off it via the existing `load` + `saveActiveNote`
    to `pickActiveNote(notes, null)` (notes already excludes it); mark re-render
    (AC-5).
- if the list changed or the active note was switched (its name changed), call
  `opts.onListChanged?.(notes, activeName)`. A pure content reload needs no list
  re-render.
- Reuses existing private `load`/`activate` building blocks where they fit; do
  not duplicate the load/persist/announce sequence.

**Wiring (`main.ts`).** `createPoller(() => controller.refreshFromDisk(), POLL_MS)`
with a sensible `POLL_MS` (~2000). `start()` it once a folder is open (idempotent
start → re-pick doesn't double-arm; the single loop follows the controller's
current `dir`). Refresh-driven list/active changes flow through the existing
`onListChanged` callback, which already re-renders the sidebar and (for a silent
reload of the active note's content) the editor is updated by the controller.

## AC-1 / AC-2 — External list add / remove refreshes the list
`refreshFromDisk` adopts `check.listChanged` and re-renders via `onListChanged`,
keeping the active note and editor untouched. Controller unit tests: mock
`listNotes` to grow/shrink between `open` and `refreshFromDisk`; assert `notes`
and the `onListChanged` payload, and that the editor buffer is unchanged.
Files: `src/note-controller.ts`, `src/note-controller.test.ts`

## AC-3 — Open note's external edit reflected (no local edits)
With `dirty=false`, a `changed` active reloads content via `readNote`, adopts the
new `lastModified`, and updates the buffer. Unit test: bump the active file's
mtime + content, `refreshFromDisk`, assert buffer shows new content and a
subsequent `saveNote` is called with the adopted mtime (no false conflict).
Files: `src/note-controller.ts`, `src/note-controller.test.ts`

## AC-4 — No silent reload with unsaved local edits
With `dirty=true`, `refreshFromDisk` leaves the buffer and disk untouched (no
`readNote` reload, no `saveNote`). Unit test: type to set dirty, change disk
mtime, `refreshFromDisk`, assert buffer unchanged and no write.
Files: `src/note-controller.ts`, `src/note-controller.test.ts`

## AC-5 — Open note deleted externally switches away (no local edits)
With `dirty=false`, a `deleted` active loads `pickActiveNote(notes, null)` and
persists it; the list reflects the removal. Unit tests for the three fallbacks
(`start.md` / first / empty `start` buffer when the folder is now empty).
Files: `src/note-controller.ts`, `src/note-controller.test.ts`

## AC-6 — No-op without a folder and while saving
`refreshFromDisk` returns early with no `dir` and while `savePromise` is set.
Unit tests: never-opened controller → no reads/writes; in-flight save (hanging
`saveNote`) → `refreshFromDisk` does nothing (mirrors the FEAT-0013 checkDisk
skip test).
Files: `src/note-controller.ts`, `src/note-controller.test.ts`

## E2E — real FSA, real poller
Playwright over OPFS: open a folder with the app; from outside the app's UI
(directly via the OPFS handle) add a file, edit the open note's file, and delete
a note, each time asserting the app catches up within a poll interval (list row
appears/disappears; open-note content updates). Drives the real `createPoller` +
`refreshFromDisk` + FSA read paths.
Files: `e2e/external-refresh.spec.ts`
