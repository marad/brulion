# Sync plan — FEAT-0013 External-change detection layer

Started: 2026-06-20T14:10:23.904Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Detection only — no acting (FEAT-0014/0015 build on this). Introduces one new
module (`watch.ts`, a generic poller) plus two additions to the existing
storage/controller layer. Implement via **excavate** (new module + new
controller capability): contracts first, tests before bodies.

Three units:

1. **`src/watch.ts` (new module)** — a generic non-overlapping poller.
   ```ts
   export interface Poller { start(): void; stop(): void }
   export function createPoller(
     tick: () => void | Promise<void>,
     intervalMs: number,
   ): Poller
   ```
   - `setInterval(intervalMs)`; on each beat, if a previous tick is still in
     flight (a boolean guard), **skip** the beat; else run `tick`, awaiting it,
     clearing the guard in `finally`. (AC-2)
   - `start`: if already running, no-op; else arm the interval. (AC-1)
   - `stop`: `clearInterval` and forget the id; a beat that elapsed but hasn't
     fired never runs after stop. A tick already awaiting may finish but never
     reschedules (the interval, not the tick, drives beats). (AC-3)
   - No dependency on the FSA, the controller, or the DOM — pure timer logic.

2. **`src/note.ts` — `statNote`** — the cheap existence/mtime probe detection
   needs without reading content:
   ```ts
   export async function statNote(
     dir: FileSystemDirectoryHandle, name: string,
   ): Promise<number | null>   // lastModified, or null if absent
   ```
   Reuses the existing private `getExisting`.

3. **`src/note-controller.ts` — `checkDisk` + a pure classifier.**
   - Pure, exported, FS-free classifier (mirrors the "pure transforms" decision
     so the logic is unit-tested without mocking the FSA):
     ```ts
     export type ActiveDiskState =
       | { kind: "changed"; lastModified: number; dirty: boolean }
       | { kind: "deleted"; dirty: boolean }
     export interface DiskCheck {
       listChanged: string[] | null   // current listing iff the *.md set changed
       active: ActiveDiskState | null // active note's on-disk change, else null
     }
     export function classifyDiskCheck(args: {
       knownNotes: string[]; diskNotes: string[]; activeName: string
       knownLastModified: number | null; diskActiveLastModified: number | null
       dirty: boolean
     }): DiskCheck
     ```
     - `listChanged`: `null` when the sorted sets are equal, else `diskNotes`.
     - `active`:
       - disk-absent (`diskActiveLastModified === null`): `{kind:"deleted",dirty}`
         **only if** `knownLastModified !== null` (we had it before); if we never
         had it on disk (lazy seed not yet written) → `null` (nothing changed).
       - disk-present: `{kind:"changed",lastModified,dirty}` when it differs from
         `knownLastModified` (including `null → value`: a file appeared where we
         had none); else `null`.
   - `checkDisk(): Promise<DiskCheck>` on the controller, run inside the existing
     `serialize()` queue (AC-7): with no folder → `{listChanged:null,active:null}`;
     else `listNotes(dir)` + `statNote(dir, activeName)`, then `classifyDiskCheck`
     over the controller's current `notes`/`activeName`/`lastModified`/`dirty`.
   - **Non-destructive (AC-7):** `checkDisk` mutates **no** controller state — it
     does not adopt the new `lastModified` or `notes`, does not reload the buffer,
     writes nothing. (Adopting the new mtime here would silently disarm the
     save-time conflict guard; the acting phases decide what to adopt.)
   - Add `checkDisk` to the `NoteController` interface.

## AC-1 / AC-2 / AC-3 — Poller fires, never overlaps, stops cleanly
`createPoller` in `src/watch.ts`. Unit tests with `vi.useFakeTimers()`:
- fires once per elapsed interval (advance N·T → N ticks);
- a tick whose promise outlives the interval blocks the next beat (overlapping
  beat skipped, not queued) — assert tick count while one is held pending;
- after `stop()` no further ticks run, including an already-elapsed-but-unfired
  beat; `start()` twice doesn't double-arm.
Files: `src/watch.ts`, `src/watch.test.ts`

## AC-4 — Detect an externally added/removed note
`classifyDiskCheck` returns `listChanged = diskNotes` when the sets differ, else
`null`. `checkDisk` surfaces it from a re-list. Unit tests: classifier (added,
removed, reordered-only → no change since both sorted) + a controller test where
the mock folder gains/loses a file between `open` and `checkDisk`.
Files: `src/note-controller.ts`, `src/note-controller.test.ts`

## AC-5 — Detect an external edit to the active note
Classifier returns `active:{kind:"changed",lastModified,dirty}` when the disk
mtime differs from the known one; `dirty` reflects the buffer's unsaved state.
Unit tests cover changed-with-dirty and changed-without-dirty; controller test
bumps the active file's mtime on disk and asserts the report.
Files: `src/note-controller.ts`, `src/note-controller.test.ts`, `src/note.ts`, `src/note.test.ts`

## AC-6 — Detect the active note deleted on disk
Classifier returns `active:{kind:"deleted",dirty}` when the active file is absent
**and** we had it before (`knownLastModified !== null`); a never-materialized seed
absent on disk reports `null`. Unit tests both; controller test deletes the
active file underneath and asserts the report.
Files: `src/note-controller.ts`, `src/note-controller.test.ts`

## AC-7 — Non-destructive and serialized
`checkDisk` runs inside `serialize()`, writes nothing, reloads nothing, mutates
no controller state, and is a no-op with no folder open. `statNote` reads mtime
without reading content. Tests: assert no write to the mock after `checkDisk`,
buffer text unchanged, `lastModified`/`notes` still detect the same change on a
second `checkDisk` (proving state wasn't adopted), and the no-folder no-op.
Files: `src/note-controller.ts`, `src/note-controller.test.ts`, `src/note.ts`
