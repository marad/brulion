## Approach

Pure UI on top of FEAT-0034's `renameActive`. No new module — the header
identity component lives in the existing shell-UI module `src/ui.ts`, alongside
`renderNoteList`/`showWorkspace`.

### Markup + style (AC-1, AC-2, AC-3)

- `index.html`: add `<div id="note-identity" class="note-identity" hidden></div>`
  right after `.wordmark` in the `<header>`. Starts hidden (like the other
  in-note controls).
- `src/styles.css`: move `margin-right: auto` off `.wordmark` and onto
  `.note-identity` so the layout becomes `[Brulion] [identity] …… [controls]`.
  Style the display button (path muted, name emphasized) and the inline input +
  error to sit inline in the header.

### Component in `src/ui.ts` (AC-1..AC-8)

NEW CODE (in the existing module). Add:

  export interface NoteIdentityHandle { update(activePath: string): void }
  export function mountNoteIdentity(
    container: HTMLElement,
    onRename: (name: string) => Promise<AddNoteResult>,
  ): NoteIdentityHandle

`AddNoteResult` is imported as a type from `./note-controller` (no runtime
cycle — note-controller does not import ui).

Internals:
- Build a display `<button class="note-identity-display">` and a hidden
  `<input class="note-identity-edit">` + `<span class="note-identity-error">`
  inside `container`.
- `update(path)` records the current path and renders the display: split the
  path into a muted folder prefix + emphasized name (`displayName` for the
  no-`.md` form). While the editor is open, `update` only stores the new path so
  it does not clobber what the user is typing.
- Clicking the display enters edit mode: value = `displayName(path)` (full path,
  no `.md`), focus + select, clear any error.
- Enter → `await onRename(value)`; on `{ ok: true }` exit to display (the
  controller's `onListChanged` will have called `update` with the new path); on
  `{ ok: false }` show `reason` in the error span and keep editing (text intact).
- Esc → cancel: exit to display showing the unchanged current path.
- blur → cancel (same as Esc), guarded by an `isCommitting` flag so the
  programmatic blur during a successful Enter-commit does not re-trigger a
  cancel.

### Wiring in `src/main.ts` (AC-3, AC-4)

- Query `#note-identity`; add it to `WorkspaceRefs` so `showWorkspace` unhides it.
- `mountNoteIdentity(identityEl, (name) => controller.renameActive(name))`.
- In `onListChanged`, call `identity.update(active)` so the header tracks the
  active note (AC-4) — same callback that already feeds sidebar/link-context.

### `showWorkspace` (AC-3) — `src/ui.ts`

Add `identity: HTMLElement` to `WorkspaceRefs` and unhide it in `showWorkspace`.
