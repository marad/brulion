# Sync plan ‚Äî FEAT-0058 Customizable action bar and settings pinning

Started: 2026-06-24T22:30:19.752Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Implemented via `/chisel` (extends existing modules; no new module boundary). Pure
list helpers + the bar renderer live in `src/command-palette.ts` (the action
module, FEAT-0057); the setting rides the existing model; the configure UI extends
the settings modal; the host wires it in `main.ts`.

## AC-1 (added) ‚Äî `actionBar` persists as an ordered id list and is normalized

Approach: Add `actionBar: string[]` to the `Settings` interface (`settings.ts`),
default `[]` in `DEFAULT_SETTINGS`. In `normalizeSettings`, coerce `actionBar` to a
string-only, de-duplicated (first wins) array ‚Äî a new pure helper `dedupeStrings`.
`loadSettings`/`saveSettings` already round-trip the whole object, so persistence is
free. `applySettings` does not touch `actionBar` (it's host UI, not an editor
property).

Files: src/settings.ts

## AC-2 (added) ‚Äî Pinned actions render as ordered icon+label buttons that run on click

Approach: New `#action-bar` container in the header (`index.html`), revealed with the
workspace (added to `WorkspaceRefs`/`showWorkspace` in `ui.ts`). New
`renderActionBar(container, actions)` in `command-palette.ts`: clears the container
and, for each resolved `Action`, appends a `<button class="action-bar-button">` with
the Lucide icon (`createElement(icon, ‚Ä¶)`) + a label span, wired to `action.run`.
Host (`main.ts`) keeps a `refreshActionBar()` that calls
`renderActionBar(actionBarEl, resolvePinned(currentSettings.actionBar, actions))`;
called on folder open (after `applySettings`) and from `updateSettings`.

Files: index.html, src/ui.ts, src/command-palette.ts, src/main.ts, src/styles.css

## AC-3 (added) ‚Äî An unknown pinned id is ignored

Approach: New pure `resolvePinned(ids, actions): Action[]` in `command-palette.ts`
maps each id to its registered action and drops misses (`.map(find).filter(Boolean)`),
preserving id order. So an id with no action yields no button and cannot throw;
resolvable ids still render. Unit-tested directly.

Files: src/command-palette.ts

## AC-4 (added) ‚Äî The settings modal can pin and unpin an action

Approach: Extend `settings-modal.ts` with an **Action bar** section. New handler dep
`getActions(): readonly { id: string; label: string }[]` supplies the registry meta
(host maps the FEAT-0057 registry to id/label). The section renders, per action, a
pin checkbox; toggling emits `{ actionBar: togglePinned(current, id) }` (new pure
helper in `command-palette.ts`: add if absent, remove if present). `seed()` re-renders
the section from `getSettings().actionBar`. The host's `updateSettings` persists and
`refreshActionBar()`s, and `settingsModal.sync()` re-seeds the section.

Files: src/settings-modal.ts, src/command-palette.ts, src/main.ts

## AC-5 (added) ‚Äî The settings modal can reorder pinned actions

Approach: In the Action bar section, the pinned actions render in order with
move-up/down controls; clicking emits `{ actionBar: movePinned(current, id, dir) }`
(new pure helper in `command-palette.ts`: swap the id with its neighbour, clamped at
the ends). Bar reflects the new order via `refreshActionBar()`.

Files: src/settings-modal.ts, src/command-palette.ts

## AC-6 (added) ‚Äî The choice persists across reloads and travels with the vault

Approach: Property of riding the existing settings file ‚Äî `actionBar` is in
`.brulion.json` (AC-1), and `openNote` already `loadSettings` + `applySettings` per
folder open; `refreshActionBar()` after that paints the restored bar. Covered by an
e2e that pins, reloads, and asserts the bar.

Files: src/main.ts

## AC-7 (added) ‚Äî Default is empty: no behavior change

Approach: `DEFAULT_SETTINGS.actionBar = []` ‚áí `resolvePinned([], ‚Ä¶) = []` ‚áí the bar
renders no buttons; the rest of the header is unchanged. No code beyond the default.
Covered by an e2e asserting an empty `#action-bar` on a fresh vault.

Files: (none ‚Äî property of the empty default)

## AC-8 (added) ‚Äî Only the preference is written (moat)
