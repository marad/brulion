# Sync plan ‚Äî FEAT-0004 Read and write the start note

Snapshot state: new

Module breakdown (this is the big phase):
- `src/note.ts` ‚Äî read/write `start.md` against a dir handle + the lastModified
  conflict check (pure-ish FSA logic).
- `src/debounce.ts` ‚Äî tiny trigger/cancel debounce.
- `src/editor.ts` ‚Äî extend mountEditor with `onChange`/`onSave`; add
  `setEditorText` (load content) using a transaction annotation so a programmatic
  load doesn't look like a user edit.
- `src/note-controller.ts` ‚Äî owns the live state (dir, lastModified, dirty,
  conflict), wires debounced autosave + flush + conflict surfacing.
- `src/ui.ts` / `src/main.ts` ‚Äî call `controller.open(dir)` whenever folder
  access is gained (pick or restore); wire blur/visibilitychange flush.
- `index.html` ‚Äî a `#status` line for the conflict indication.

## AC-1 (added) ‚Äî An existing `start.md` is loaded into the editor

Approach: `note.ts readNote(dir)` does `getFileHandle("start.md")` (no create);
on success returns `{ content: await file.text(), lastModified: file.lastModified }`,
on NotFoundError returns `{ content: "", lastModified: null }`.
`note-controller.open(dir)` calls readNote, records lastModified, and loads the
text via `setEditorText` (annotated as external so it doesn't mark dirty).
`ui.ts` calls `controller.open(dir)` on folder-ready (pick/restore).
Unit: note.test.ts (file exists ‚Üí content+lastModified); controller.test.ts
(open ‚Üí view shows content); e2e: seeded start.md shows in real editor.
Files: `src/note.ts`, `src/editor.ts`, `src/note-controller.ts`, `src/ui.ts`, `src/main.ts`, `src/note.test.ts`, `src/note-controller.test.ts`, `e2e/note.spec.ts`

## AC-2 (added) ‚Äî `start.md` is created on first save, not on open

Approach: `open()` never writes. `saveNote(dir, content, known)` uses
`getFileHandle("start.md", { create: true })` + `createWritable()` only when a
save fires. Opening a folder with no start.md leaves it absent; the first
`doSave` (triggered by an edit) creates it. Unit: note.test.ts (save with no
existing file creates it); e2e: open empty folder ‚Üí assert no start.md ‚Üí type ‚Üí
assert start.md now exists with the text.
Files: `src/note.ts`, `src/note-controller.ts`, `src/note.test.ts`, `e2e/note.spec.ts`

## AC-3 (added) ‚Äî Edits autosave to disk after a pause

Approach: `debounce.ts` schedules `doSave` ~600 ms after the last
`controller.handleChange()` (fired by the editor's onChange). `doSave` writes
only when `dirty` and not in conflict. Unit: debounce.test.ts (fake timers, one
call after the window, coalesced); controller.test.ts (handleChange ‚Üí after
timer ‚Üí saveNote called once). e2e: type, wait, assert disk content.
Files: `src/debounce.ts`, `src/note-controller.ts`, `src/editor.ts`, `src/debounce.test.ts`, `src/note-controller.test.ts`, `e2e/note.spec.ts`

## AC-4 (added) ‚Äî Pending edits flush on focus loss and on Ctrl/Cmd+S

Approach: `controller.flush()` cancels the debounce and runs `doSave`
immediately. `main.ts` wires `window` `blur` and `document` `visibilitychange`
(when `document.hidden`) to flush; `editor.ts` adds a `Mod-s` keymap entry
(preventDefault) that calls `onSave` ‚Üí flush. Unit: controller.test.ts (flush
saves immediately, cancels pending). e2e: edit then Ctrl+S ‚Üí disk content
updated without waiting for the debounce.
Files: `src/note-controller.ts`, `src/editor.ts`, `src/main.ts`, `src/note-controller.test.ts`, `e2e/note.spec.ts`

## AC-5 (added) ‚Äî A save never silently overwrites an externally changed file

Approach: in `saveNote`, before writing, re-read the file's current
`lastModified`. Conflict iff the file exists now AND (`known === null` OR
`current !== known`); return `{ status: "conflict" }` without writing.
`doSave` on conflict sets a `conflict` flag (halting further autosaves), leaves
the buffer intact, and calls an `onConflict` callback that shows `#status`.
Unit: note.test.ts (changed lastModified ‚Üí conflict, no write; appeared-from-null
‚Üí conflict); controller.test.ts (conflict ‚Üí onConflict, dirty kept, lastModified
unchanged). e2e: load, mutate start.md via OPFS, type+save ‚Üí file unchanged +
status shown.
Files: `src/note.ts`, `src/note-controller.ts`, `index.html`, `src/main.ts`, `src/note.test.ts`, `src/note-controller.test.ts`, `e2e/note.spec.ts`

## AC-6 (added) ‚Äî Saved content survives a restart

Approach: emergent from AC-1 + AC-3 + FEAT-0003 restore ‚Äî saved bytes are in
`start.md`; on reload the handle is restored and readNote reloads them. e2e:
write content, reload the page (OPFS persists per-origin; handle restored from
IndexedDB), assert the editor shows it. True browser-restart-to-disk is the one
manual spot-check.
Files: (covered by AC-1/AC-3 code) `e2e/note.spec.ts`
