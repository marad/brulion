# Sync plan — FEAT-0023 Path-addressed storage layer

Started: 2026-06-21T09:48:11.583Z
Snapshot state: new
Drift summary: 0 added (whole spec)

## Approach (whole spec)

Two existing modules change; no new module. `src/note.ts` gains directory-tree
traversal so every operation is keyed by a folder-relative POSIX path, and
`src/note-name.ts`'s `normalizeNoteName` becomes path-aware. The controller is
untouched: it already treats the active note as an opaque string, so passing it a
path "just works". The unit-test fake folder (`src/note.test.ts`) is upgraded to a
recursive tree backing so the nested cases are exercised against a realistic mock.

### Path helpers (shared, in `src/note.ts`)
- `splitPath(path)` → `{ dirs: string[]; file: string }`: `path.split("/")`, last
  element is the file, the rest are folder segments.
- `resolveParent(dir, dirs, create)` → walks `getDirectoryHandle(seg, { create })`
  per folder segment, returning the leaf directory handle; with `create=false`,
  returns `null` the moment a segment is `NotFoundError` (a missing folder ⇒
  missing file). The existing `isNotFound` helper is reused.
- `getExisting(dir, path)` is rewritten to `resolveParent(dir, dirs, false)` then
  `parent.getFileHandle(file)` (null on either miss).

### AC-1, AC-2 — read by nested path; missing folder/file reads empty
`readNote` already delegates to `getExisting`; once that walks the tree, a nested
read returns content+mtime (AC-1) and a missing folder or file yields
`{ content: "", lastModified: null }` with nothing created (AC-2). `statNote`
likewise (covers the poller's stat-by-path).

### AC-3, AC-4 — write nested, creating folders; per-note guard preserved
`saveNote`: keep the `getExisting`-based mtime guard (unchanged semantics, AC-4),
then `resolveParent(dir, dirs, true)` to materialize intermediate folders and
`parent.getFileHandle(file, { create: true })` to write (AC-3). The guard compares
on-disk `lastModified` to `knownMtime` exactly as before.

### AC-5 — recursive listing of sorted relative paths
`listNotes`: replace the flat scan with a recursive `collect(dir, prefix, out)`
that, per `dir.values()` handle, pushes `prefix + name` for `.md` files and
recurses into directory handles with `prefix + name + "/"` (the async iterator
yields the directory handle itself — no extra `getDirectoryHandle`). Sort the full
paths case-insensitively (`localeCompare`, `sensitivity: "base"`). Directories are
never listed themselves; empty dirs contribute nothing.

### AC-6, AC-7, AC-8 — create nested / refuse overwrite
`createNote`: `getExisting` short-circuits to `{ status: "exists" }` when the file
is present (AC-7/AC-8, no clobber), else `resolveParent(dir, dirs, true)` +
`getFileHandle(file, { create: true })` + empty writable to materialize the file
and its folders (AC-6).

### AC-8 (delete) — `deleteNote` by nested path
`deleteNote`: `resolveParent(dir, dirs, false)`; a `null` parent (missing folder)
is a no-op, matching the existing already-absent idempotence; otherwise
`parent.removeEntry(file)`, swallowing `NotFoundError` as today.

### AC-9, AC-10 — path-aware name normalization
`src/note-name.ts` `normalizeNoteName`: trim the input; reject empty; `split("/")`;
for each segment trim it and reject when empty, when `.`/`..`, or when it hits the
existing `UNSAFE`/control-char test (a segment can't contain `/` post-split, and
`\` is still caught). Strip a single trailing `.md` (case-insensitive) from the
**last** segment only, re-check it's non-empty, and re-append `.md`; re-join all
segments with `/`. A bare name (no `/`) reduces to the FEAT-0010 behavior (AC-9).
Any `.`/`..`/empty/unsafe segment ⇒ `{ ok: false, reason }` (AC-10). `displayName`
is unchanged (its `\.md$` strip already yields `folder/name` from a path).

### Test mock upgrade (`src/note.test.ts`)
Rewrite `fakeFolder` to back a nested tree: entries may be directories holding
their own entry map; add `getDirectoryHandle(name, { create })` (throws
`NotFoundError` when absent and not creating, else inserts a directory node) and
make `values()` yield real file/dir handles. Add tests for AC-1..AC-8 nested
paths; extend `note-name.test.ts` for AC-9/AC-10 path cases (`projects/Diablo
builds`, `..`, `a//b`, leading/trailing `/`, `\` in a segment).
