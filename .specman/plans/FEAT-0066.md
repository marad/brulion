# Sync plan — FEAT-0066 Dark-mode syntax highlight palette

Started: 2026-06-25T15:09:53.217Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Implemented via `/chisel`. The seven distinct syntax colors in `code-highlight.ts`
become CSS custom properties in the M18 palette (light = today's GitHub-light values,
a dark override set), and `codeTokenTheme` references them via `var(--…)`.

## AC-1 / AC-3 (theme-aware syntax palette, system follows OS)

Approach: add `--tok-keyword`, `--tok-string`, `--tok-number`, `--tok-comment`,
`--tok-function`, `--tok-type`, `--tok-tag` to all three palette blocks in
`styles.css` (`:root`, `:root[data-theme="dark"]`, `@media (prefers-color-scheme: dark)
:root:not([data-theme])`). The dark values are a GitHub-dark-readable set. The tokens
flip via the same data-theme / media-query mechanism FEAT-0065 already wires — no new
switching logic, so system mode follows the OS for free.

Files: src/styles.css

## AC-2 (light unchanged)

Approach: each light token value equals the exact hex it replaces in
`code-highlight.ts` today (#cf222e keyword, #0a3069 string, #0550ae number, #6e7781
comment, #8250df function, #953800 type, #116329 tag). Classes that shared a hex
(property/escape = number's blue; meta = comment grey) keep sharing one token, so light
rendering is byte-identical. `codeTokenTheme` swaps literals for `var(--tok-…)`.

Files: src/code-highlight.ts
