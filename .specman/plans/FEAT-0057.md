# Sync plan — FEAT-0057 Action registry and command palette

Started: 2026-06-24T22:05:04.706Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Implemented via `/excavate` (new module `src/command-palette.ts`). The module design
(diagram + signatures) lives in `docs/excavation/`; this plan maps the ACs to the
resulting code. The palette mirrors the quick switcher (`src/quick-switcher.ts`) in
structure and reuses `fuzzyScore` (`src/note-search.ts`); the action registry is
assembled in `src/main.ts`.

## AC-1 (added) — The shortcut opens the palette, focused

Approach: New hidden overlay DOM in `index.html` (`#palette-backdrop` → `#palette`
dialog → `#palette-input` + `#palette-list`), mirroring `#switcher-backdrop`. New
`src/command-palette.ts` exporting `mountCommandPalette(els, deps)` returning a
handle `{ open, close, isOpen, destroy }`; `open()` reveals the backdrop, clears the
input, renders the action list, and focuses the input. In `main.ts` a capture-phase
`keydown` listener opens it on `Ctrl/Cmd+Shift+P` (`event.code === "KeyP"`,
layout-proof), gated on `workspaceShown && conflictBackdropEl.hidden`.

Files: index.html, src/command-palette.ts, src/main.ts, src/styles.css

## AC-2 (added) — Typing fuzzily filters the actions by label

Approach: `command-palette.ts` keeps a pure helper `rankActions(query, actions)`
that scores each action's `label` with `fuzzyScore` (FEAT-0038), drops nulls, sorts
by score desc then registry order (stable), and returns the ordered actions. Empty
query returns all in registry order (every score 0). `render()` rebuilds the list
from `rankActions(input.value, deps.getActions())` on every `input` event.

Files: src/command-palette.ts

## AC-3 (added) — Arrow keys and Enter run the highlighted action

Approach: Same keyboard-nav model as the switcher: a `highlight` index, ↑/↓ clamp it
and re-apply the `active` class + `scrollIntoView`, Enter calls the highlighted
row's stored `run`. Running closes the overlay first, then invokes the action's
`run()` (so the action sees focus already restored).

Files: src/command-palette.ts

## AC-4 (added) — A click runs an action

Approach: Each rendered row is a `<button>` with a click listener bound to the same
`run` closure used by Enter (one code path), which closes then invokes `run()`.

Files: src/command-palette.ts

## AC-5 (added) — Esc (or backdrop) closes without running anything

Approach: Esc in the input handler and a backdrop click (`event.target === backdrop`)
both call `close()`, which hides the backdrop and restores `restoreFocus` (captured
on open) — no `run()` invoked. Mirrors the switcher's close/focus-restore.

Files: src/command-palette.ts

## AC-6 (added) — Folder-switch and Vim toggle are runnable from the palette

Approach: In `main.ts` build an `Action[]` registry: Go to note (`switcher.open`),
Switch folder (`() => void openFolder(resumeButton, openNote)`), Toggle Vim
(`toggleVim`), Toggle note list (the sidebar toggle's `toggle()`), Open settings
(`() => settingsModal?.open()`). The palette's `getActions` dep returns this array.
`toggleVim` and the folder-switch flow already exist; the actions wrap them so the
same `run()` drives palette + existing entry points.

Files: src/main.ts

## AC-7 (added) — An action's icon shows on its palette row

Approach: `Action.icon?: IconNode` (Lucide). When present, `render()` prepends
`createElement(icon, { class: "palette-icon", "aria-hidden": "true" })` to the row
before the label span; absent → label-only. Registry actions are given Lucide icons
where one fits (reusing the M27 icon set).

Files: src/command-palette.ts, src/main.ts, src/styles.css

## AC-8 (added) — The palette opens under Vim

Approach: The opener is a capture-phase window listener with `preventDefault`, so it
runs before CodeMirror/Vim key handling regardless of mode — same mechanism as the
`Ctrl/Cmd+K` switcher shortcut (FEAT-0033 AC-9). Covered by an e2e with Vim on.

Files: src/main.ts

## AC-9 (added) — The palette is gated and never stacks

Approach: The opener checks `workspaceShown`, `conflictBackdropEl.hidden`,
`switcherBackdropEl.hidden`, `settingsBackdropEl.hidden`, and `!palette.isOpen()`
before opening; any false → no-op. Mirrors the existing `Ctrl/Cmd+K`/`,` gating.

Files: src/main.ts

## AC-10 (added) — No writes from palette use

Approach: Property of the design — the palette only reads `getActions()` and toggles
its own DOM; it performs no FSA calls. Verified by an e2e that opens/filters/closes
and asserts the folder's files are untouched.

Files: (none — property of the no-FSA palette)
