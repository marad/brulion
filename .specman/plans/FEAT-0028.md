## Approach (whole spec)

No TS module. Static assets in `public/` (Vite copies `public/**` verbatim to the
build root, served under the `/brulion/` base) plus a `<head>` edit in `index.html`
and one vitest that validates the shipped manifest. Colors come from the app
palette (`styles.css`): brand accent `#9a3b2f`, surface `#fbfbfa`.

### AC-1, AC-2 — `public/manifest.webmanifest`
A static JSON manifest committed under `public/`. Fields:
`name: "Brulion — quick capture"`, `short_name: "Brulion"`, a `description`,
`display: "standalone"`, `theme_color: "#9a3b2f"`, `background_color: "#fbfbfa"`,
`start_url: "/brulion/"`, `scope: "/brulion/"` (explicit, base is fixed in
`vite.config.ts`), and `icons`:
- `/brulion/icons/icon-192.png` 192x192 `purpose: "any"`
- `/brulion/icons/icon-512.png` 512x512 `purpose: "any"`
- `/brulion/icons/icon-maskable-512.png` 512x512 `purpose: "maskable"`

### Icons — generated once from an SVG, committed to `public/icons/`
A small notepad glyph (lined "page" on the accent color) authored as SVG, then
rasterized to PNG at 192/512 + a maskable 512 (glyph inside the ~80% safe zone on
a full-bleed accent background). Rasterization is a one-off dev script
(`scripts/gen-icons.sh`) using `rsvg-convert` (librsvg, present on the dev box) —
no new runtime/build dependency. The committed PNGs are what ships; the SVG
sources + script stay for regeneration. An apple-touch-icon (180x180) is emitted
the same way.

### AC-3 — `index.html` `<head>` wiring
Add, under the existing `<title>`:
- `<link rel="manifest" href="/brulion/manifest.webmanifest" />`
- `<meta name="theme-color" content="#9a3b2f" />` (equals manifest `theme_color`)
- `<link rel="apple-touch-icon" href="/brulion/icons/apple-touch-icon.png" />`

### AC-4 — build output
`public/` files copy to `dist/` root, so `dist/manifest.webmanifest`,
`dist/icons/*.png` exist after `npm run build` at exactly the referenced paths.

### Tests
- NEW `src/manifest.test.ts` (pure, vitest): read `public/manifest.webmanifest`,
  parse JSON; assert non-empty `name`/`short_name`, `display === "standalone"`,
  `theme_color`/`background_color` present, `start_url`/`scope` start with
  `/brulion/` (AC-1); assert an icon `192x192` and one `512x512` exist, at least
  one `purpose` includes `maskable`, and every `icons[].src` maps to a file under
  `public/` that exists on disk (AC-2); parse `index.html`, assert the manifest
  `<link>`, a `theme-color` meta equal to the manifest's `theme_color`, and an
  apple-touch-icon `<link>` whose target file exists (AC-3). AC-4 is the same
  files via the public→dist copy (asserted indirectly: the referenced files exist
  under `public/`, which Vite copies verbatim).
