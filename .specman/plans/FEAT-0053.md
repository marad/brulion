## Design

`src/selection-toolbar.ts`:
- Drop the `TOUCH` gate from the show condition ‚Üí show on any non-empty selection with
  focus (desktop + touch).
- Add **drag-settle**: a `dragging` flag set on `contentDOM` `pointerdown` (and the
  toolbar hidden then), cleared on a document `pointerup` followed by `render()`.
  `render()` returns early (hidden) while `dragging`. So a mouse drag-select shows the
  toolbar only on release; keyboard/touch-handle selections are unaffected. Remove the
  document `pointerup` listener (and keep the scroll listener) on `destroy`.

`src/context-menu.ts`:
- The `contextmenu` handler computes the wikilink toggle at the click position first;
  if there is **no** toggle, `return false` (do not preventDefault ‚Üí the browser's
  native menu shows). If there is one, `preventDefault()` and open a **one-item** menu
  (just the toggle). Drop the `FORMAT_ITEMS` import and the formatting items; keep the
  `openMenu` mechanics (Esc/outside-click dismiss) for the single toggle item.
- `format-actions.ts` is unchanged (the toolbar still uses `FORMAT_ITEMS`).

No CSS change needed (the toolbar styling already exists; the `.cm-context-menu`
styling still serves the one-item menu).
