# Sync plan — FEAT-0060 Workspace switcher and forget

Snapshot state: new
Drift summary: 0 added (whole spec)

Implemented via `/chisel` (extends command-palette + main + settings-modal; no new
module). The chooser reuses the command palette opened with a transient vault list;
forgetting reuses `removeVault` behind a settings section.

## AC-1 (added) — "Switch workspace…" lists the other granted workspaces

Approach: `command-palette.ts` `mountCommandPalette` gains `open(override?: readonly
Action[])` — when an override list is passed it renders that instead of
`getActions()` (stored in a `let override`, cleared on close). In `main.ts` a new
"Switch workspace…" action builds `(await listVaults()).filter(v => v.id !==
currentVaultId)` into `{id:`ws:${v.id}`, label: v.name, icon, run}` actions and calls
`palette.open(those)`.

Files: src/command-palette.ts, src/main.ts

## AC-2 (added) — Choosing a workspace switches the window and updates `?ws`

Approach: each vault row's `run` calls `switchToVault(vault)` (main.ts) →
`attachVault(vault)` (FEAT-0059), which stamps `?ws` and loads the vault. The palette
closes before running, per FEAT-0057.

Files: src/main.ts

## AC-3 (added) — Switching re-grants permission when the handle lost it

Approach: `switchToVault` checks `hasPermission(vault.handle)`; if not granted it
calls `requestAccess(vault.handle)` (the palette click is a user gesture) and only
attaches on grant, else returns (open vault unchanged).

Files: src/main.ts

## AC-4 (added) — The settings Workspaces section lists the granted workspaces

Approach: `settings-modal.ts` gains a **Workspaces** section + a handler
`getWorkspaces: () => Promise<{ id, name, open }[]>`; on open the section renders one
row per workspace (folder name), async like the font list. Host supplies it from
`listVaults()` marking `open = id === currentVaultId`.

Files: src/settings-modal.ts, src/main.ts

## AC-5 (added) — Forgetting a workspace removes it from the set

Approach: each non-open row has a forget control wired to a handler
`onForgetWorkspace(id)` → host `removeVault(id)` (FEAT-0059); the section re-renders
from `getWorkspaces()` after. The vault is gone from the list and from a later
"Switch workspace…". No FS delete.

Files: src/settings-modal.ts, src/main.ts

## AC-6 (added) — The open workspace cannot be forgotten

Approach: `getWorkspaces` marks the open vault (`open: true`); its row renders no
forget control (or a disabled one), so the attached vault can't be removed.

Files: src/settings-modal.ts

## AC-7 (added) — No note bytes are written

Approach: property of the design — switching attaches via the existing path (which
only reads notes), forgetting calls `removeVault` (idb only). Verified by an e2e that
switches + forgets and asserts the folders' `.md` files are unchanged.

Files: (none — property of the storage/URL-only change)
