# Sync plan — FEAT-0076 F2 renames the focused sidebar tree row

Drift summary: 0 added (whole spec)

## Approach

F2 becomes one more key the existing tree keyboard-nav pipeline (FEAT-0075)
resolves and dispatches — no new focus surface, no new rename path.

Pure core in **`src/tree-nav.ts`**:
- Extend `TreeAction` with `{ type: "rename"; index: number }`.
- In `resolveTreeKey`, add `case "F2": return { type: "rename", index: current }`.
  Like every other action it names the focused row; the glue decides note vs
  folder from that row's descriptor. `current` out of range still yields `none`
  via the existing guard. (AC-1, AC-2, AC-4, AC-5 — F2 resolves to `rename` and
  leaves every other key's resolution untouched.)

DOM glue in **`src/ui.ts`**:
- `wireTreeKeyNav` needs the list handlers to reach the rename entry points, so
  its signature gains `handlers: NoteListHandlers` and `renderNoteList` passes
  them at the one-time wiring site (same capture-once pattern the container drop
  zone already uses; the handlers object is stable across renders in `main.ts`).
- Handle the new action: `rename` → read the target row's descriptor and call
  `handlers.onRenameFolder?.(path)` for a folder header or
  `handlers.onRenameNote?.(path)` for a note row. `event.preventDefault()`
  already runs for any non-`none` action, so F2's default is suppressed. (AC-1,
  AC-2.)
- The handler already early-returns when focus is not on a `.folder-header`/
  `.note-name` row, so F2 outside the tree is inert. (AC-3.)
- F2 itself performs no FS call; it only invokes `onRenameNote`/`onRenameFolder`,
  which open the in-app prompt (FEAT-0073) and write only on commit. (AC-4.)

Tests:
- **`src/tree-nav.test.ts`**: F2 on a note row → `{type:"rename",index}`; F2 on
  a folder header → `{type:"rename",index}`; F2 with `current` out of range →
  `none`; a spot-check that Up/Down/Left/Right/Home/End/Enter/Space keep their
  actions with F2 added (AC-5).
- **`src/ui.test.ts`**: dispatching an `F2` keydown on a focused note row calls
  `onRenameNote` with its path (not `onRenameFolder`); on a folder header calls
  `onRenameFolder` (not `onRenameNote`); an `F2` keydown with focus outside the
  tree calls neither (AC-3); pressing F2 triggers no `onDelete`/`onSelect`/move
  handler (AC-4 at the glue level).
- **`e2e/tree-keyboard-nav.spec.ts`**: focus a note row, press F2, assert the
  in-app rename prompt opens seeded with the current name; cancel it and assert
  no file changed on the OPFS handle (AC-1, AC-4 end-to-end).
