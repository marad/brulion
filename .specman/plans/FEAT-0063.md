## AC-1 (added) — header + separator + body renders as a table

Approach: `findTableBlocks(state)` (pure, line scan) detects a separator row with a
non-blank header above + contiguous body below, returns `{ from, to, aligns,
header, rows }`. `tableDecorations` emits `Decoration.replace({ widget: new
TableWidget(block), block: true })`. `TableWidget.toDOM` builds a `<table>` (thead +
tbody). Registered via `tableRendering` in `editor.ts`.

Files: src/table-render.ts, src/editor.ts, src/styles.css

## AC-2 (added) — column alignment follows the separator

Approach: `parseAligns(separatorLine)` maps each cell (`:--:` etc.) to
`left|center|right|none`; `TableWidget` sets `text-align` per column on the cells.

Files: src/table-render.ts

## AC-3 (added) — outer pipes optional

Approach: `splitRow(line)` trims the line, strips a single leading/trailing `|`, then
splits on `|` and trims cells — so `a | b` and `| a | b |` yield the same cells.

Files: src/table-render.ts

## AC-4 (added) — `|` inside a fenced code block is not a table

Approach: the line scan tracks fenced-code state (toggles on ```` ``` ````/`~~~`
lines) and skips detection while inside a fence (same discipline as the M32 heading
scan). Covered by an e2e (a code block with pipes renders as code).

Files: src/table-render.ts

## AC-5 (added) — selecting inside reveals the raw source

Approach: `selectionTouches` (strict overlap, as in mermaid-render) — a block the
selection overlaps is omitted from the decoration set, so its raw lines show. The
field rebuilds on `docChanged || selection`. `tableClick` moves the caret just inside
a clicked rendered table so it reveals.

Files: src/table-render.ts

## AC-6 (added) — a header with no separator is not a table

Approach: detection is anchored on the **separator** row; a header line not followed
by a valid separator yields no block.

Files: src/table-render.ts

## AC-7 (added) — ragged body rows padded/truncated to the column count

Approach: column count = the separator's; `TableWidget` pads a short row's cells with
empties and slices a long row to the count. No crash, no stray column.

Files: src/table-render.ts

## AC-8 (added) — bytes unchanged

Approach: property — the field only adds a `Decoration.replace` over existing ranges;
no document change. Verified by an e2e reading the file/doc after render + reveal.

Files: (none — property of the decoration-only design)
