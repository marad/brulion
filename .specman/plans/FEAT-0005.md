# Sync plan — FEAT-0005 Editor typography

Started: 2026-06-20T05:28:13.997Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Editor typography goes through `EditorView.theme` (reliably overrides
CodeMirror's baseTheme monospace; plain CSS can lose to CM's runtime-injected
styles). Page chrome (body/header/list/status/layout) goes in a global
`src/styles.css` imported by `main.ts`.

## AC-1 (added) — Proportional reading font, not monospace

Approach: `src/editor.ts` adds an `EditorView.theme` extension setting
`.cm-scroller` `font-family` to a system stack constant `FONT_STACK`
(`system-ui, -apple-system, …, sans-serif`) and hides `.cm-gutters` (line
numbers read as code). Verified by e2e computed-style: `.cm-content`
font-family includes `system-ui` and not `monospace`.
Files: `src/editor.ts`, `e2e/typography.spec.ts`

## AC-2 (added) — Capped, centered reading measure

Approach: theme sets `.cm-content` `max-width: 68ch`, `margin: 0 auto`, and
comfortable padding, so the text column is centered and narrower than a wide
viewport. Verified by e2e: `.cm-content` bounding box width < viewport width and
left/right gaps roughly equal (centered).
Files: `src/editor.ts`, `e2e/typography.spec.ts`

## AC-3 (added) — Comfortable size and spacing

Approach: theme sets base `font-size: 16px` and `.cm-scroller` `line-height:
1.6`. Verified by e2e computed-style: font-size ≥ 16px and line-height ≥ 1.5×
font-size.
Files: `src/editor.ts`, `e2e/typography.spec.ts`

## Chrome (supports the Intent, not a separate AC)

`src/styles.css`: light neutral background, flex column filling the viewport,
tidy header buttons, muted inline file list (hidden when empty), conflict status
styling, editor fills remaining height. Imported at the top of `main.ts`.
Files: `src/styles.css`, `src/main.ts`
