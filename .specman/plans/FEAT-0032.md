# Sync plan — FEAT-0032 vim caret respects hidden markup

## Approach (whole spec)

A new tiny module `src/vim-caret.ts` holding (a) a **pure** snap function and (b)
a CodeMirror `EditorState.transactionFilter` extension, wired into `editor.ts`.
No file-behavior code is touched.

### Pure core (`vim-caret.ts`, unit-tested)
`export interface Span { from: number; to: number }`
`export function snapOutOfSpans(pos: number, prev: number, spans: Span[]): number`
— if `pos` is **strictly** inside one span (`from < pos < to`), return `to` when
`pos >= prev` (moved forward) else `from` (moved backward); positions on an edge
or outside every span return `pos` unchanged. Pure, no CodeMirror — directly
testable.

### Extension (`vim-caret.ts`)
`export const vimCaretGuard: Extension = EditorState.transactionFilter.of(...)`:
- Return `tr` unchanged when `!tr.selection` (no explicit selection set — typing,
  programmatic loads), when `tr.docChanged` (keep off the edit path; Vim motions
  carry no change), or when `tr.isUserEvent("select.pointer")` (a click — preserve
  the FEAT-0026 link reveal).
- Otherwise, for each range in `tr.newSelection`, compute hidden runs on the
  endpoint's line via `hiddenSpansOnLine(tr.state, pos)` and snap anchor & head
  with `snapOutOfSpans`, using the matching range in `tr.startState.selection`
  (fallback `.main`) as `prev`. If nothing moved, return `tr`; else return
  `[tr, { selection: EditorSelection.create(correctedRanges, mainIndex) }]`
  (appended spec overrides the selection in one transaction — no flicker; safe
  because there are no doc changes to re-map).

`function hiddenSpansOnLine(state, pos): Span[]` — `const line =
state.doc.lineAt(pos)`; merge `markdownSyntaxRanges(state, line.from,
line.to).hidden`, `blockSyntaxRanges(state, line.from, line.to).hidden`, and the
`.bullets` runs (mapped to `{from,to}`). These are exactly the editor's atomic
runs. Per-line scope keeps it cheap.

### Wiring (`editor.ts`)
Add `vimCaretGuard` to the extensions array (place near the other state-level
extensions; precedence doesn't matter — it's a filter, not a keymap). Always on:
a no-op for the default caret (already kept out of atomic ranges), an invariant
for Vim.

### Tests
- NEW `src/vim-caret.test.ts`: unit-test `snapOutOfSpans` — inside→forward edge,
  inside→backward edge, on-edge unchanged, outside unchanged, multiple spans,
  empty spans. (Pure; no DOM needed.)
- `e2e/vim.spec.ts` (add): with Vim on, seed `# Heading` / `> quote` / `* item`
  lines; `0` then `l` lands on the first visible glyph (assert
  `view.state.selection.main.head` via `page.evaluate` on the CodeMirror view, or
  a visible-coordinate check), and a backward motion rests at the line start —
  AC-1..AC-5. Reuse the existing `enableVimAndFocus` helper. Default-caret
  regression (AC-6) and link-click reveal (AC-7) are already covered by existing
  specs; add a focused assertion only if a gap shows.
