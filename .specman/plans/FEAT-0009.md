`src/context-menu.ts` intercepts `contextmenu` (via `EditorView.domEventHandlers`)
and opens a DOM popup (`.cm-context-menu`, styled in `src/styles.css`) at the
pointer, replacing the native menu. Items reuse the FEAT-0007 transforms:
bold/italic/code → `toggleInline`; H1-H3 / Clear → `setHeadingLines` (new,
per-line across the selection). A `ViewPlugin` destroy hook tears the menu down
with the view. Wired in `src/editor.ts`. `setHeadingLines` and the multi-line
inline wrap are unit-tested; the menu UX is e2e-tested in real Chromium.

## AC-1 — Custom menu replaces the native one

`contextmenu` is `preventDefault`ed and the popup opened (clamped into the
viewport). e2e asserts the seven items appear.
Files: `src/context-menu.ts`, `src/styles.css`, `src/editor.ts`, `e2e/context-menu.spec.ts`

## AC-2 / AC-4 — Heading / Clear apply per line across the selection

`setHeadingLines(state, level)` rewrites every line the selection touches (0 =
paragraph), with the end-at-line-start boundary handled so the next line isn't
caught. Unit-tested; e2e: 3 lines → all `## `, two headings → cleared.
Files: `src/markdown-transforms.ts`, `src/markdown-transforms.test.ts`, `e2e/context-menu.spec.ts`

## AC-3 — Bold wraps the selection

Reuses `toggleInline`; single-line wraps the span, multi-line wraps each line
separately so the file stays valid CommonMark. Unit-tested + e2e (single and
multi-line).
Files: `src/markdown-transforms.ts`, `src/context-menu.ts`, `e2e/context-menu.spec.ts`

## AC-5 — Dismissal

Esc (capture keydown) and outside click (capture pointerdown) close the menu and
remove their listeners; picking an item closes it too. e2e covers Esc and
outside-click leaving the document unchanged.
Files: `src/context-menu.ts`, `e2e/context-menu.spec.ts`
