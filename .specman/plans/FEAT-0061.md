# Sync plan — FEAT-0061 Link section anchors

Started: 2026-06-25T07:52:27.103Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Implemented via `/chisel` (extends M8's link path; no new module). Pure helpers in
`note-name.ts`; anchor split in `markdown-render.ts`; a scroll step + click-handler
threading in `editor.ts`; follow wiring in `main.ts`.

## AC-1 (added) — Markdown link with anchor switches + scrolls

Approach: `markdown-render.ts` `collectLink`: for an **internal** href, `splitAnchor`
the target → resolve/style the path part (existing `linkInfo`) and store the path in
`data-href` + the anchor in `data-anchor`. `editor.ts` click handler reads
`data-anchor` and calls `onFollowLink(href, anchor)`. `main.ts` `onFollowLink`
resolves the note (`resolveNotePath`) and `openNotePath(target, anchor)`, which
`await`s `controller.switchTo` then `scrollEditorToHeading(view, anchor)`.

Files: src/note-name.ts, src/markdown-render.ts, src/editor.ts, src/main.ts

## AC-2 (added) — Wikilink with anchor switches + scrolls

Approach: `collectWikilinks`: `splitAnchor(rawTarget)`; validate/resolve the **path**
part (`resolveWikilink`); store resolved note in `data-note` + anchor in
`data-anchor`. Click handler → `onFollowNote(note, anchor)` → `openNotePath`.

Files: src/markdown-render.ts, src/editor.ts, src/main.ts

## AC-3 (added) — Same-note anchor scrolls without switching

Approach: `splitAnchor("#sec")` → path `""`, anchor `"sec"`. `linkInfo("")` and the
wikilink path-empty branch treat an empty path as the **open note** (valid styling).
`openNotePath`: when the target is the current note (or path empty), skip `switchTo`
and just `scrollEditorToHeading`.

Files: src/note-name.ts, src/markdown-render.ts, src/main.ts

## AC-4 (added) — Heading match is slug-based, case-insensitive, Unicode-aware

Approach: `headingSlug(text)` (note-name.ts): lower-case, drop punctuation (keep
`\p{L}\p{N}`), collapse whitespace to single hyphens, trim hyphens.
`scrollEditorToHeading` slugifies both the anchor and each heading line and compares.

Files: src/note-name.ts, src/editor.ts

## AC-5 (added) — Missing target heading: open, no scroll, no error

Approach: `scrollEditorToHeading` returns without dispatching when no heading line's
slug matches; the note still opened. Pure no-op.

Files: src/editor.ts

## AC-6 (added) — External link `#fragment` not treated as a note anchor

Approach: `collectLink` only `splitAnchor`s when `!isExternalLink(href)`; an external
href is stored whole in `data-href` (no `data-anchor`). `main.ts onFollowLink` opens
it in a tab as today, fragment intact.

Files: src/markdown-render.ts, src/main.ts

## AC-7 (added) — Missing-note link still offers create (anchor ignored)

Approach: `openNotePath(path, anchor)` keeps the existing create-on-miss branch; for a
freshly-created note the heading won't exist so `scrollEditorToHeading` is a no-op.
The anchor never affects the note resolution.

Files: src/main.ts

## AC-8 (added) — No bytes written by anchor navigation

Approach: property — `splitAnchor`/`headingSlug`/`scrollEditorToHeading` are read-only
(slug compare + a scroll/selection effect); only the unchanged M8 create writes.
Verified by an e2e asserting the folder's `.md` files are untouched across anchored
navigations.

Files: (none — property of the read-only nav)
