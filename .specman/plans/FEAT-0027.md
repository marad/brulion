## Approach (whole spec)

No new module. `note-name.ts` gains a pure `resolveWikilink`; `markdown-render.ts`
scans for `[[…]]` and decorates them (reusing the link styling + reveal); `editor.ts`
+ `main.ts` add a `data-note` follow path (an already-resolved absolute note path,
distinct from a markdown link's relative `data-href`). Markdown links (FEAT-0025/26)
are untouched.

### AC-1, AC-2, AC-3 — pure `resolveWikilink` (`note-name.ts`)
`resolveWikilink(target, notePaths): { resolved: string | null; createPath: string }`.
Trim `target`; `createPath` = `target` + `.md` (not doubled). If `target` has a
`/` (a root-relative path): `resolved` = the first note in `notePaths` equal to
`createPath` **case-insensitively**, else null. Else (a bare name): `resolved` =
the first note whose **basename** (after the last `/`) equals `createPath`
case-insensitively, else null; `createPath` stays root-level (`name.md`). Iteration
order is the listing's sorted order (the Set is built from the sorted list), so an
ambiguous bare name picks the first by sorted path. Pure, unit-tested (AC-1/2/3).

### AC-4, AC-5, AC-6 — render wikilinks (`markdown-render.ts`)
In `markdownSyntaxRanges`, after the tree iterate, scan the slice `[from,to]` with
`/\[\[([^\]\[|]+)(?:\|([^\]\[]+))?\]\]/g`. For each match not overlapping the
selection (reveal-on-caret, FEAT-0026): `target` = group 1, `alias` = group 2.
Hide `[[` (and, with an alias, through the `|`) and the trailing `]]`; style the
label run (`alias` else `target`) `cm-link`, adding `cm-link-broken` when
`resolveWikilink(target.trim(), links.notePaths).resolved` is null; attrs
`{ "data-note": resolved ?? createPath, title: resolved ?? createPath }`. Empty
target can't match the regex (`+`), so `[[]]` is left raw. A helper
`collectWikilinks(slice, offset, sel, links, marks, hidden)` keeps it tidy.

### AC-7, AC-8 — follow a wikilink (`editor.ts` + `main.ts`)
`editor.ts`: `EditorOptions.onFollowNote?(path)`; the mousedown handler now reads
`[data-href]` **or** `[data-note]` on the closest ancestor — `data-href` →
`onFollowLink` (markdown/external, resolved relative in main), `data-note` →
`onFollowNote` (an absolute note path, no further resolution). `main.ts`: extract
`openNotePath(path)` = `currentNotes.includes(path) ? switchTo(path) : confirm &&
addNote(path)`, used by both the internal-markdown branch of `onFollowLink` and the
new `onFollowNote`. So an existing wikilink target switches (AC-7) and a missing one
prompts + creates at its create path (AC-8).
