# Sync plan — FEAT-0062 Weekly journal navigation

Started: 2026-06-25T11:16:41.819Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Implemented via `/chisel`. A small pure `journal.ts` (path expansion); the setting
rides the existing model; one action in `main.ts`; a settings-modal field.

## AC-1 (added) — `journalPath` persists + normalizes

Approach: Add `journalPath: string` to `Settings` (`settings.ts`), default `""` in
`DEFAULT_SETTINGS`; `normalizeSettings` coerces with `typeof === "string" ? r.journalPath : ""`.
`load/saveSettings` round-trip it for free.

Files: src/settings.ts

## AC-2 (added) — placeholders expand against the current date

Approach: New `src/journal.ts` `expandJournalPath(template, date): string` — replaces
`{year}` (4-digit), `{month}`/`{day}` (2-digit, zero-padded) from the date, and
`{mondayOfTheWeek}` (see AC-3). Pure (date passed in). `main.ts` calls it with
`new Date()`.

Files: src/journal.ts

## AC-3 (added) — `{mondayOfTheWeek}` is the week's Monday across boundaries

Approach: In `journal.ts`, `mondayOf(date)` clones the date, computes the Mon-based
offset `(getDay()+6)%7`, and `setDate(getDate()-offset)` (which rolls back across
month/year boundaries), then formats ISO `YYYY-MM-DD` from local components.

Files: src/journal.ts

## AC-4 (added) — action opens an existing journal note

Approach: `main.ts` "Open this week's journal" action: `expandJournalPath(journalPath,
new Date())` → `normalizeNoteName` → `openNotePath(filename)`. An existing note
switches (the `currentNotes.includes` branch).

Files: src/main.ts

## AC-5 (added) — missing note routes through the existing create flow

Approach: Same `openNotePath(filename)` — a missing path falls to the existing
create-on-miss `window.confirm` + `controller.addNote`. No new creation code.

Files: src/main.ts

## AC-6 (added) — empty `journalPath` opens settings

Approach: The action checks `currentSettings.journalPath`; when empty (or expansion /
`normalizeNoteName` yields nothing usable) it calls `settingsModal?.open()` instead of
navigating.

Files: src/main.ts

## AC-7 (added) — registry action (palette + pinnable)

Approach: Add `{ id: "open-journal", label: "Open this week's journal", icon:
CalendarDays, run }` to the `actions` registry in `main.ts`; it flows into the palette
and the M30 action-bar pinning automatically.

Files: src/main.ts

## AC-8 (added) — no bytes beyond the preference + confirmed note

Approach: Property — expansion/normalize/`openNotePath` read only; writes are
`.brulion.json` (the setting) and the user-confirmed create-on-miss note. Verified by
an e2e: configure + run on an existing note → that note opens, no extra `.md` written.

Files: (none — property)
