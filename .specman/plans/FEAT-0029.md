## Approach (whole spec)

A new `public/sw.js` (plain JS, copied verbatim to the build root) plus a tiny
`src/pwa.ts` registration helper called from `main.ts`. Registration is gated on
`import.meta.env.PROD`, so the dev-server e2e suite runs with no worker (zero
M1–M8 regression) and the offline path is verified against `vite preview`.

### AC-1, AC-2 — `src/pwa.ts` registration helper (unit-tested)
`registerServiceWorker(nav: Navigator, base: string, enabled: boolean): void`:
- `if (!enabled) return` (production-only gate, decided by the caller).
- `if (!("serviceWorker" in nav)) return` (no-throw degrade).
- else `void nav.serviceWorker.register(\`${base}sw.js\`, { scope: base }).catch(()=>{})`
  — a failed registration is swallowed so boot never breaks.
`main.ts` adds `registerServiceWorker(navigator, import.meta.env.BASE_URL,
import.meta.env.PROD)` after the existing wiring.

### AC-3, AC-4 — `public/sw.js` (the worker)
Plain service-worker JS, no TS (lives in public/, outside tsc):
- `const VERSION = "brulion-cache-v1"`.
- `install`: `self.skipWaiting()`.
- `activate`: delete every cache key !== VERSION, then `self.clients.claim()`.
- `fetch`: ignore non-GET and cross-origin (return → network). For `request.mode
  === "navigate"` use **network-first** (fetch, cache.put a clone on ok; on throw
  return cache.match(request) else the cached base shell `${scope}` else rethrow).
  Otherwise **cache-first** (cache.match; else fetch + cache.put clone on ok).
- A small `caches.open(VERSION)` per handler; helpers `networkFirst`/`cacheFirst`.

`dist/sw.js` exists after build (public copy) → AC-3.

### AC-4, AC-5 — e2e + the preview server
- `playwright.config.ts`: add a SECOND `webServer` running the production build +
  preview on port 4173 (`npm run build && vite preview --port 4173 --strictPort`),
  `reuseExistingServer: true`. Existing tests keep using the dev server (5173,
  baseURL); the new spec navigates to the absolute preview URL.
- NEW `e2e/pwa-offline.spec.ts`: goto `http://localhost:4173/brulion/`; await
  `navigator.serviceWorker.ready`; reload once online so the controlled
  navigation + assets populate the cache; `context.setOffline(true)`; reload;
  assert `.cm-editor` and `#open-folder` are visible (AC-4). A second assertion
  confirms `navigator.serviceWorker.controller` is non-null.
- AC-5 is the whole existing dev-server suite still green (the worker isn't
  registered in dev).
