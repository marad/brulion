## Design

New module `src/link-complete.ts` — a `@codemirror/autocomplete` source for
wikilinks, peer to `slash-commands.ts`:

- `wikilinkSource(context: CompletionContext): CompletionResult | null`
  - `context.matchBefore(/\[\[[^\]\n]*/)` — matches `[[` + partial target (no `]`,
    no newline). Null → return null.
  - `from = match.from + 2` (right after `[[`); `query = match.text.slice(2)`.
  - paths = `[...context.state.facet(linkContext).notePaths]` (one source of truth).
  - `searchNotes(query, paths).matches` → ranked note paths (ignore `.create`).
  - Empty matches → return null (nothing to suggest).
  - options: `{ label: displayName(path), type: "text", apply: applyWikilink(path) }`.
  - Return `{ from, options, filter: false }` (no `validFor`) so CodeMirror keeps our
    `note-search` order authoritative and re-queries the source per keystroke for a
    fresh ranking.
- `applyWikilink(path)` → `(view, _completion, from, to) => void`
  - `insert = displayName(path)`; reuse a trailing `]]` if `doc.sliceString(to, to+2)
    === "]]"`, else append `]]`.
  - Dispatch `changes:{from,to,insert}`, caret `selection:{anchor: from + insert.length + 2}`
    (after the `]]`, same formula whether reused or appended).
- `export const wikilinkCompletions = markdownLanguage.data.of({ autocomplete: wikilinkSource })`.

Wire in `editor.ts`: add `wikilinkCompletions` to the extensions array next to
`slashCommands`. No other file changes (the `linkContext` facet is already populated
by `setLinkContext` in `main.ts` on every active-note/list change).

  all notes (empty query) and contains no create row.
- **AC-2** — after `[[dia`, options are ranked by `searchNotes` order (assert order
  equals `searchNotes("dia", paths).matches.map(displayName)`); `from` is right after
  `[[`; `filter:false` is set.
- **AC-3** — `apply` on a buffer `[[dia` (caret at end): result text is
  `[[<displayPath>]]` and the caret lands after `]]` (build an `EditorView`, run the
  option's `apply`, assert `doc` and `selection.main.head`).
- **AC-4** — `apply` on `[[dia]]` (caret before `]]`): result is a single
  `[[<displayPath>]]`, no doubled brackets.
- **AC-5** — `wikilinkSource` returns null (or empty→null) for a no-match query; the
  source never yields a create row. (Dangling-link allowance is structural: the
  source only suggests; it inserts/creates nothing.)
- **AC-6** — with two notes sharing a basename in different folders, the inserted
  target is the full display path (covered by AC-3 using a nested note path).
- **AC-7** — `slashSource` unchanged (existing `slash-commands.test.ts` stays green);
  `wikilinkSource` returns null for a `/` slash context (no `[[`).
- **AC-8** — read-only is structural: the source/apply only read the facet and edit
  the buffer; asserted by inspection (no fs import in `link-complete.ts`).

## Addendum — name-only links by default + right-click form toggle (user feedback on M20)

The user wants Obsidian-style links: insert the bare name by default (not the full
path), and a right-click menu item to switch a link between full-path and name-only.
This reworks AC-3/AC-6 and adds AC-9..AC-12.

### New shared module `src/wikilink.ts` — pure wikilink-text helpers

One home for the wikilink regex and the "what text represents this link" logic, so
the autocomplete insert and the right-click toggle agree (and `markdown-render.ts`
stops owning a private copy of the regex):

- `WIKILINK_RE` — moved here from `markdown-render.ts`; that module now imports it.
- `shortestLinkText(path, allPaths): string` — the shortest *unambiguous* link text:
  the bare basename (`.md` stripped, no folder) when that basename is unique across
  `allPaths`, else the full display path. This is the single definition of "name-only
  when possible" used by both surfaces (AC-3, AC-9, AC-6 — ambiguous → full path).
- `findWikilinkAt(text, pos): { from, to, target, targetFrom, targetTo, alias } | null`
  — the `[[…]]` whose span contains `pos` (uses a fresh regex; pos inclusive of the
  edges).
- `computeWikilinkToggle(text, pos, notePaths): { from, to, insert, label } | null` —
  the right-click action. Finds the link at `pos`, resolves its target
  (`resolveWikilink`); returns `null` when it doesn't resolve, or when
  `shortestLinkText(resolved) === displayName(resolved)` (root note or ambiguous
  basename → nothing meaningful/safe to toggle, AC-12). Otherwise returns the target
  sub-range `[targetFrom, targetTo)`, the toggled text (full→short or short→full by
  comparing the current target to the full form), and the menu `label` ("Use name
  only" / "Use full path"). Replacing only the target keeps any `|alias` (AC-11).

### `src/link-complete.ts` — insert the shortest form

`wikilinkSource` computes each option's insert text via `shortestLinkText(path,
notePaths)` (over the full known set, for correct uniqueness) instead of
`displayName(path)`; `applyWikilink` takes the final insert string. The completion
*label* stays the full `displayName(path)` so the list disambiguates same-named
notes (AC-2 unchanged).

### `src/context-menu.ts` — add the toggle item

In `openMenu`, derive `pos = view.posAtCoords({x, y})` and call
`computeWikilinkToggle(doc, pos, view.state.facet(linkContext).notePaths)`. When it
returns a result, prepend a menu item whose `run` dispatches the `changes` (AC-10).
The formatting items are unchanged.

### `src/markdown-render.ts` — import the regex

Replace the local `WIKILINK_RE` definition with an import from `./wikilink` (dedup;
the decoration scanning logic is otherwise untouched — the 41 render tests guard it).

### Tests (addendum)

- `src/wikilink.test.ts` — NEW: `shortestLinkText` (unique basename → bare; ambiguous
  → full path; root note → bare), `findWikilinkAt` (hit/miss, alias, edges),

- `src/link-complete.test.ts` — AC-3 now asserts a unique nested note inserts the
