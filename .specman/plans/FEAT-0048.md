# Sync plan — FEAT-0048 settings modal and local-font access

Started: 2026-06-23T16:15:43.495Z
Snapshot state: new
Drift summary: 0 added (whole spec)

## Design

Two new modules (built with `excavate`) plus header/HTML/CSS/main wiring. The modal
owns no state: it reads current settings via a getter, emits change patches to the
P1 `updateSettings`, and re-seeds from the getter on `sync()`.

`src/font-access.ts` (new, leaf, pure-ish):
- `interface FontChoices { source: "local" | "preset"; families: string[] }`.
- `PRESET_FONTS: string[]` — a curated cross-OS list (e.g. Georgia, Times New Roman,
  Arial, Verdana, Tahoma, Trebuchet MS, Courier New, Consolas, Menlo, Monaco) used
  when local enumeration is unavailable/denied.
- `resolveFontChoices(): Promise<FontChoices>` — if `typeof
  (window as any).queryLocalFonts === "function"`, call it, map `.family`, dedupe,
  sort case-insensitively → `{source:"local", families}`. On absence / throw /
  denial → `{source:"preset", families: PRESET_FONTS}`. Never throws.

`src/settings-modal.ts` (new):
- `interface SettingsModalHandlers { getSettings(): Settings; onChange(patch: Partial<Settings>): void; resolveFontChoices(): Promise<FontChoices> }`.
- `interface SettingsModalHandle { open(): void; sync(): void }` (`sync` re-seeds
  controls from `getSettings()` only while open — keeps the modal consistent when
  `Ctrl/Cmd+;` toggles Vim underneath it).
- `mountSettingsModal(backdrop: HTMLElement, handlers): SettingsModalHandle` — builds
  the dialog DOM into `backdrop` (pattern of `mountMissingNoteBanner`): a font
  `<select>`, a text-size stepper (−/readout/+, clamped 12–24 via the P1
  MIN/MAX — import from settings), a Narrow/Wider/Full radio group, a Vim checkbox,
  and a close button. Every control's change calls `onChange(patch)` then re-seeds
  (so the readout/selected state reflect the now-current settings). `open()` seeds
  from `getSettings()`, resolves font choices and fills the select (default option =
  `""` → empty font list; current family added if missing from the list), then shows
  the backdrop. Esc, backdrop click, and the close button hide it.

`index.html`:
- Remove the `#toggle-vim` button. Add a `#open-settings` gear button in its header
  slot (`aria-label`, title with the `Ctrl/Cmd+,` chord). Add an empty
  `<div id="settings-backdrop" hidden>` after the conflict backdrop.

`src/styles.css`: modal styling reusing the switcher/conflict backdrop + dialog look;
classes for the control rows, the stepper, the radio group.

`src/main.ts`:
- swap the `#toggle-vim` query for `#open-settings`; update `WorkspaceRefs` (rename
  `toggleVim` → `settings`, unhide the gear in `showWorkspace`).
- `applyCurrentSettings` drops the header-button aria-pressed line (no header Vim
  button now); after a change, `updateSettings` calls `settingsModal?.sync()` (a
  forward-declared `let settingsModal: SettingsModalHandle | null = null` breaks the
  onChange↔sync cycle).
- mount the modal: `settingsModal = mountSettingsModal(settingsBackdropEl, {
  getSettings: () => currentSettings, onChange: updateSettings, resolveFontChoices })`.
- wire `#open-settings` click → `settingsModal.open()`; add a capture-phase
  `Ctrl/Cmd+,` handler (guard `(ctrl||meta) && !alt && !shift && key === "," &&
  workspaceShown && conflict hidden`, preventDefault) → `open()`. `Ctrl/Cmd+;` →
  `toggleVim` is unchanged.
- `ui.ts`: rename the `toggleVim` field in `WorkspaceRefs`/`showWorkspace` to
  `settings`.

## Implementation steps (excavate — tests before bodies)

1. `excavate` `font-access.ts`: stubs → cold-subagent unit tests (local present →
   families mapped/deduped/sorted; absent → presets; throw/deny → presets) → bodies.
2. `excavate` `settings-modal.ts`: stubs → cold-subagent unit tests (each control's
   change emits the right patch; `open()` seeds from getSettings; `sync()` reflects
   an external change; Esc/backdrop close) against a fake handlers object → bodies.
3. HTML + CSS: remove Vim button, add gear + settings backdrop; style the modal.
4. Wire `main.ts` + `ui.ts` as above.
5. e2e `e2e/settings-modal.spec.ts`: open via gear (AC-1) and `Ctrl/Cmd+,` (AC-2);
   bump size → assert `.cm-content` font-size grew + `.brulion.json` size (AC-3);
   change width → measure var + file (AC-4); toggle Vim in modal → editor Vim on +
   file, then `Ctrl/Cmd+;` flips and the checkbox reflects it (AC-5); assert no
   `#toggle-vim` and gear present (AC-6); the font select lists options and choosing
   one sets `--font-stack` + file (AC-7/AC-8); Esc/backdrop close (AC-9).
