# Sync plan — FEAT-0008 Slash commands

Started: 2026-06-20T07:31:03.210Z
Snapshot state: new
Drift summary: 0 added (whole spec)

`src/slash-commands.ts` provides a CodeMirror completion source (`slashSource`)
registered via `markdownLanguage.data.of({ autocomplete })`, so basicSetup's
autocomplete drives the menu UI. The source fires only for a `^/\w*$` token at
line start. Options `/h1 /h2 /h3 /clear` have an `apply` that strips the token
(`stripSlashToken`) and reshapes the line with the reused `withHeadingLevel`
(FEAT-0007). Pure helpers unit-tested; the menu + accept flow e2e-tested in real
Chromium. Wired in `src/editor.ts`.

## AC-1/AC-2 — Menu opens at line start and filters

`slashSource` returns the four commands when the pre-caret text matches `^/\w*$`;
autocomplete filters by the typed token. Unit-tested (`slash-commands.test.ts`);
e2e asserts the menu appears with 4 options and narrows to 3 on `/h`.
Files: `src/slash-commands.ts`, `src/slash-commands.test.ts`, `e2e/slash.spec.ts`

## AC-3/AC-4 — Accepting reshapes the line and drops the token

`applyLevel` rewrites the line to the chosen heading level (or paragraph for
`/clear`), removing the `/command` token first so it never reaches disk. e2e:
`/h2`→`## Title`; `/clear` on a heading line → plain `Heading`. The strip+reshape
composition is unit-tested to prove only clean markdown is produced.
Files: `src/slash-commands.ts`, `src/slash-commands.test.ts`, `e2e/slash.spec.ts`

## AC-5 — Esc dismisses

Handled by autocomplete's own Esc binding; e2e confirms the menu closes and the
typed text is left untouched.
Files: `e2e/slash.spec.ts`

## AC-6 — A mid-line slash does not trigger

`^/\w*$` only matches at line start; a slash after other text returns null. Unit-
tested and e2e-tested (`see /` stays plain).
Files: `src/slash-commands.ts`, `src/slash-commands.test.ts`, `e2e/slash.spec.ts`
