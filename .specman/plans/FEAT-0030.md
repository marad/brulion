## Approach (whole spec)

A small pure controller `src/install-prompt.ts`, a thin adapter in `main.ts`, and
one hidden header button in `index.html` reusing existing `header button` CSS.
Mirrors the FEAT-0020/0021 toggle split (pure core + DOM wiring).

### AC-1..AC-4 — `src/install-prompt.ts` (unit-tested)
`createInstallPrompt(isStandalone, setVisible)` returns `{ onBeforeInstallPrompt,
onInstallClick, onInstalled }` over a private `deferred` stash:
- `onBeforeInstallPrompt(e)`: if `isStandalone` return (AC-2); else
  `e.preventDefault()`, `deferred = e`, `setVisible(true)` (AC-1).
- `onInstallClick()`: read+null `deferred`, `setVisible(false)`, then
  `event?.prompt()` — single-use, so a second click is inert (AC-3).
- `onInstalled()`: `deferred = null`, `setVisible(false)` (AC-4).
Type `DeferredInstallPrompt = { preventDefault(): void; prompt(): unknown }`.

### AC-5 — wiring (`index.html` + `main.ts`)
- `index.html`: `<button id="install-app" type="button" hidden>Install</button>`
  in the header (after resume-access). No CSS (generic `header button`).
- `main.ts`: query `#install-app` (add to the mount null-check); compute
  `isStandalone = matchMedia("(display-mode: standalone)").matches ||
  (navigator as any).standalone === true`; `const install =
  createInstallPrompt(isStandalone, (v) => { installButton.hidden = !v })`; wire
  `window` `beforeinstallprompt` → `onBeforeInstallPrompt(e)` (cast), `window`
  `appinstalled` → `onInstalled()`, button `click` → `onInstallClick()`. Not
  production-gated (only SW registration was).
