# Sync plan — FEAT-0003 Persist the folder handle

Started: 2026-06-19T20:45:19.350Z
Snapshot state: new
Drift summary: 0 added (whole spec)

New module `src/session.ts` owns persistence + permissions (idb-keyval + the
handle's queryPermission/requestPermission). `src/ui.ts` grows a restore flow and
a "Resume folder access" path; `index.html` gains the (hidden) resume button;
`main.ts` runs restore on load. Adds the `idb-keyval` dependency.

## AC-1 (added) — Picking a folder persists its handle

Approach: NEW CODE. `src/session.ts` exports `saveFolder(handle)` =
`set(KEY, handle)` and `loadFolder()` = `get(KEY)` (idb-keyval), `KEY` a stable
string. `ui.ts`'s `openFolder` calls `saveFolder(dir)` after a successful pick.
Unit-tested by mocking `idb-keyval` and asserting `set` is called with the
handle (session.test.ts), and that `openFolder` calls `saveFolder` (ui.test.ts).
Files: `src/session.ts`, `src/ui.ts`, `src/session.test.ts`, `src/ui.test.ts`, `package.json`

## AC-2 (added) — A still-granted folder is restored with zero clicks on load

Approach: NEW CODE. `session.ts` exports `hasPermission(handle)` =
`(await handle.queryPermission({ mode: "readwrite" })) === "granted"`. `ui.ts`
exports `restoreFolder(list, resumeButton)`: loads the handle, and if
`hasPermission` is true calls `showFolder` (lists `.md` files) with no gesture
and the resume button left hidden. `main.ts` calls `restoreFolder` on load.
Unit-tested: mocked session returns a handle + granted → files rendered, resume
stays hidden.
Files: `src/session.ts`, `src/ui.ts`, `src/main.ts`, `index.html`, `src/ui.test.ts`, `src/session.test.ts`

## AC-3 (added) — A folder needing re-grant is restored with one click

Approach: NEW CODE. `session.ts` exports `requestAccess(handle)` =
`(await handle.requestPermission({ mode: "readwrite" })) === "granted"`. In
`restoreFolder`, when `hasPermission` is false, the resume button is un-hidden
and its click calls `requestAccess`; on `granted` the button is hidden and
`showFolder` runs. A denied request leaves the button visible for a retry.
Unit-tested: not-granted → button shown; click→granted → button hidden + files
rendered; click→denied → button stays.
Files: `src/session.ts`, `src/ui.ts`, `index.html`, `src/ui.test.ts`, `src/session.test.ts`

## AC-4 (added) — With nothing persisted, the app starts in the pick-a-folder state

Approach: In `restoreFolder`, when `loadFolder()` returns `undefined`, return
early — nothing rendered, resume button stays hidden, the FEAT-0002 pick flow is
untouched. Unit-tested with `loadFolder` mocked to resolve `undefined`.
Files: (covered by AC-2 code)
