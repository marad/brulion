# Sync plan — FEAT-0043 Folders collapsed by default

Started: 2026-06-23T11:52:10.801Z
Snapshot state: new
Drift summary: 0 added (whole spec)

## Approach

Invert the persisted folder state from *collapsed* to *expanded* across three
files. The render rule, the toggle handler, and the storage key/polarity all
flip; the "reveal the active note's ancestors" rule is untouched. No new module —
a small change in the existing sidebar layer (chisel).

### Render: `renderNoteList` / `renderNode` — `src/ui.ts` (AC-1, AC-2, AC-3)

- Rename the 5th parameter of `renderNoteList` and `renderNode` from
  `collapsed: ReadonlySet<string>` to `expanded: ReadonlySet<string>` (default
  `new Set()`).
- In `renderNode` flip the rule:
  `const isCollapsed = !expanded.has(node.path) && !isAncestorOfActive`.
  (Was `collapsed.has(...) && !isAncestorOfActive`.) Absent ⇒ collapsed (AC-1);
  present ⇒ expanded (AC-2); ancestor-of-active ⇒ always expanded (AC-3),
  overriding absence. The set is read, never mutated (constraint).
- The toggle handler is unchanged in shape — it still reports the new *collapsed*
  state via `handlers.onToggleFolder?.(node.path, children.hidden)`. The polarity
  flip lives in main.ts (the caller), keeping `onToggleFolder`'s contract
  ("collapsed = true when now hidden") stable.
- Update the doc comments on `renderNoteList`/`renderNode` to describe the
  expanded-set semantics and the collapsed-by-default behavior.

### Storage: expanded folders — `src/session.ts` (AC-4, AC-5, AC-6)

- New key `const EXPANDED_FOLDERS_KEY = "brulion:expanded-folders"`.
- Replace `saveCollapsedFolders`/`loadCollapsedFolders` with
  `saveExpandedFolders(paths: ReadonlySet<string>): Promise<void>` and
  `loadExpandedFolders(): Promise<Set<string>>` (empty set when none stored).
  Same shape (array round-trip via idb-keyval), inverted name + key.
- Drop the old `COLLAPSED_FOLDERS_KEY` constant. The old key is abandoned (not
  migrated): on first load the new key is empty ⇒ tree opens fully collapsed,
  which is the intended default (spec: "Migrating … out of scope").

### Wiring — `src/main.ts` (AC-4, AC-5, AC-6)

- Rename `collapsedFolders` → `expandedFolders`, `collapsedFoldersReady` →
  `expandedFoldersReady`, loading via `loadExpandedFolders()`.
- In `onToggleFolder(path, collapsed)`: **invert** the set update —
  `if (collapsed) expandedFolders.delete(path); else expandedFolders.add(path)`,
  then `void saveExpandedFolders(expandedFolders)`. (Was: add when collapsed.)
- Pass `expandedFolders` as the 5th arg to `renderNoteList`.
- `openNote` still awaits the ready promise before the first open (AC-6: first
  paint matches saved state — no flash).
- Update the import in main.ts (`saveCollapsedFolders, loadCollapsedFolders` →
  `saveExpandedFolders, loadExpandedFolders`) and the related comment on
  `expandedFolders`.

### Tests

- AC-1: a folder absent from the expanded set (and not an ancestor) renders with

- AC-3: ancestor-of-active force-expanded despite absence from the set, set not
