# Sync plan — FEAT-0036 Note URLs: hash route for the open note

Started: 2026-06-23T05:34:54.333Z
Snapshot state: new
Drift summary: 0 added (whole spec)

## Approach

### New module `src/note-route.ts` — the pure codec (AC-1, AC-2, AC-3, AC-4)

Two pure functions, no DOM/History/FSA dependency, so they unit-test directly:

- `pathToHash(path: string): string` — strip a trailing `.md` (case-insensitive),
  split the remainder on `/`, `encodeURIComponent` each segment, join with `/`,
  prefix `#/`. So `start.md` → `#/start`, `Allegro/Journal/Week 22.md` →
  `#/Allegro/Journal/Week%2022` (AC-1, AC-2).
- `hashToPath(hash: string): string | null` — return `null` unless the hash starts
  with `#/`; take the remainder; return `null` if empty; split on `/`; for each
  segment return `null` if it is empty (covers `#/`, `#/a//b`, trailing slash) or
  if `decodeURIComponent` throws (malformed `%`); else decode it; join with `/`
  and append `.md` (AC-4). The two are total inverses for app-produced paths
  (always lowercase `.md`), so encode-then-decode returns the original (AC-3).

### Wiring in `src/main.ts` — mirror, drive, resolve-on-load (AC-5..AC-12)

Three connections, no custom history stack:

1. **Open note → URL (mirror).** In the existing `onListChanged` callback (the one
   announcement site for every active-note change — switch, link follow, switcher,
   rename-follows-file, external delete), after `currentActive = active`, call a
   `syncRouteToActive(active)` helper. It is a no-op while `suppressRouteSync` is
   true (the initial-load window) and a no-op when `hashToPath(location.hash)`
   already equals `active` (already mirrored — avoids a duplicate push and is
   robust to browser hash re-encoding). Otherwise it assigns `location.hash =
   pathToHash(active)`, which pushes a history entry (AC-5, AC-6, AC-7). The
   already-mirrored guard is also the loop guard for AC-12.

2. **URL → open note (drive).** A `window` `hashchange` listener: ignore unless
   `workspaceShown`; compute `target = hashToPath(location.hash)`; switch only when
   `target && target !== currentActive && currentNotes.includes(target)` —
   `!== currentActive` is the no-loop guard (a mirror we just wrote is a no-op,
   AC-12), and `includes` makes a hash naming a missing note inert (AC-8). The
   switch reuses `controller.switchTo` (AC-6, AC-7).

3. **Load / bookmark (resolve once).** The existing `openNote(dir)` gains a
   one-time initial-hash consumption guarded by `initialRouteConsumed`. On the
   first call it reads `target = hashToPath(location.hash)` *before* opening, sets
   `suppressRouteSync = true`, awaits `controller.open(dir)` (which picks the
   persisted/seed note), then — if `target` names an existing note other than the
   one just opened — awaits `controller.switchTo(target)` so the hash beats the
   persisted active note (AC-9, AC-10). With no/blank/missing-target hash the
   opened note stands (AC-11). It then clears `suppressRouteSync` and calls
   `history.replaceState(null, "", pathToHash(currentActive))` so landing leaves
   exactly one history entry (no phantom previous — AC-12). Subsequent `openNote`
   calls (re-picking a folder from the header) just `controller.open(dir)` as
   before. `poller.start()` stays at the end.

No file behavior changes — the hash is navigation state only (moat-neutral).

## Addendum — missing-note banner (M19 review: AC-8 reworded, AC-13..16)

The review replaced the silent "inert" behavior for a well-formed hash naming a
missing note with a **non-blocking banner** that offers to create it.

### `index.html` + `src/styles.css` (AC-13)

A new `#missing-note-banner` element in `.workspace` (hidden by default): a slim,
non-modal bar with a message span, a **Create** button, and a **dismiss (×)**
button. Styled as a discreet top banner (consistent with the existing
conflict/welcome surfaces), never blocking the editor.

### `src/ui.ts` — `mountMissingNoteBanner` (AC-13, AC-14, AC-15)

`mountMissingNoteBanner(container, { onCreate, onDismiss }): MissingNoteBannerHandle`
with `show(name: string)` / `hide()`. `show` fills the message (`"<name>" doesn't
exist yet.`, reusing `displayName` wording from the dead-link `confirm`) and
reveals; the Create button fires `onCreate`, the × fires `onDismiss`. Pure DOM
glue, unit-tested like `mountNoteIdentity`.

### `src/main.ts` — resolve into switch / missing / none (AC-8, AC-13..16)

Replace `hashTargetToOpen()` with a 3-way `resolveHash()`: `none` (malformed hash
or already-open — ignored, AC-16), `switch` (named note exists — `switchTo`), or
`missing` (well-formed, absent — raise the banner). The `hashchange` listener and
the load-time resolution both use it. On a missing hash:
- **runtime** (hashchange): keep `#/ghost` in the bar, `showMissingBanner(target)`,
  open note unchanged (AC-13).
- **load**: open the fallback note, then keep `#/ghost` (skip the settling
  `replaceState`) and show the banner (AC-13 on load).
- **Create** (`onCreate`): `controller.addNote(displayName(target))` — the existing
  create path + validation; the resulting active-note announcement makes `#/ghost`
  name a real note (AC-14).
- **Dismiss** (`onDismiss`): hide the banner and `history.replaceState` the bar back
  to the open note's hash, so the URL stops naming an absent note (AC-15).
- Hide the banner whenever the active note changes (in `onListChanged`) so it never
  outlives the state it described.

The moat exception is explicit and user-gated: the only write a route causes is the
Create button, through `addNote`, never automatic.

### Tests

- `src/ui.test.ts` — `mountMissingNoteBanner`: show fills the name + reveals,

- `e2e/note-url.spec.ts` — runtime missing hash raises the banner and doesn't switch
