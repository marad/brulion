## Design

Chisel-scale: a CSS media query + one backdrop element + two small close behaviors
wired in `main.ts`. Breakpoint: `max-width: 40rem` (640px) — below it the
sidebar+editor side-by-side (14rem + 20rem min) can't fit.

`index.html`: add `<div id="sidebar-backdrop"></div>` inside `<main class="workspace">`
after `#sidebar` (a general-sibling so CSS can gate it on the sidebar's state). No
`hidden` attr — its visibility is pure CSS.

`src/styles.css` — a `@media (max-width: 40rem)` block:
- `#sidebar { position: absolute; inset: 0 auto 0 0; width: min(80vw, 16rem);
  z-index: 50; box-shadow: 2px 0 12px rgb(0 0 0 / 18%); }` — overlay drawer. (The
  existing `.workspace.sidebar-collapsed #sidebar { display:none }` and
  `#sidebar[hidden]` rules out-specify this, so a collapsed/no-folder drawer stays
  hidden; an open one overlays.)
- `#sidebar-resizer { display: none; }` (AC-5).
- `.wordmark { display: none; }` so the header fits a phone (the identity already
  names the note).
- `.workspace:not(.sidebar-collapsed) #sidebar:not([hidden]) ~ #sidebar-backdrop {
  display: block; position: absolute; inset: 0; z-index: 40; background: rgb(0 0 0 /
  32%); }` — backdrop shown only when narrow AND a folder is open AND the drawer is
  open. Base rule: `#sidebar-backdrop { display: none; }` (desktop + all closed states).

`src/main.ts`:
- query `#sidebar-backdrop` (add to the null-check).
- `const MOBILE = window.matchMedia("(max-width: 40rem)")`.
- module-level `let dismissDrawerIfMobile = (): void => {}`.
- in the `loadSidebarCollapsed().then(...)` block: track `let isCollapsed` via the
  `apply` callback; add `const closeDrawer = () => { if (!isCollapsed) sidebar.toggle() }`;
  wire `sidebarBackdropEl.addEventListener("click", closeDrawer)`; set
  `dismissDrawerIfMobile = () => { if (MOBILE.matches) closeDrawer() }`.
- in `renderNoteList`'s `onSelect`, call `dismissDrawerIfMobile()` after
  `controller.switchTo(name)` (AC-4) — desktop is unaffected because the guard is
  `MOBILE.matches`.

`src/ui.ts` `showWorkspace`/`WorkspaceRefs`: no change — the backdrop is CSS-only and
the sidebar/resizer reveal is unchanged.

## Implementation steps (chisel — test before body)

1. e2e first (`e2e/mobile-layout.spec.ts`, red): with `viewport 375×700` and a seeded
   folder, open it; assert the editor spans ~full width while the drawer overlays
   (sidebar box left ≈ editor box left ≈ 0, sidebar narrower than viewport) and a
   backdrop is visible (AC-1); ☰ toggles it (AC-2); a backdrop tap closes it (AC-3);
   tapping a note row switches and closes it (AC-4); the resizer is hidden (AC-5). A
   desktop case (`1280×800`) asserts the sidebar takes layout width (editor box left ≈
   sidebar width, no backdrop, selecting a note leaves it open) (AC-6).
2. Implement the HTML + CSS + main.ts wiring.
3. Confirm the existing sidebar e2e (`e2e/sidebar.spec.ts`) still passes at desktop
   width (AC-6 regression guard).
