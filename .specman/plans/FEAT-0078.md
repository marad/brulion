# Sync plan — FEAT-0078 Multi-select and batch delete/move in the sidebar tree

Started: 2026-07-16T15:03:29.708Z
Snapshot state: new
Drift summary: 0 added (whole spec)

## Approach

### New pure module `src/selection-model.ts`
The testable core of the selection set, no DOM:
- `toggleSelection(selected: ReadonlySet<string>, path: string): Set<string>` —
  a copy of `selected` with `path` added if absent / removed if present. (AC-1, AC-3, AC-4.)
- `rangeSelect(visiblePaths: string[], anchor: string, focus: string): Set<string>` —
  the set of paths in the contiguous inclusive span between `anchor` and `focus`
  in `visiblePaths` order (works either direction; empty if either is missing).
  Replace semantics — the returned set *is* the new selection — so extending and
  shrinking a Shift+arrow range both fall out of one rule. (AC-2.)

### Selection state in `src/ui.ts` (module-level, like `draggedItem`)
`let treeSelection: { selected: Set<string>; anchor: string | null }`. There is a
single tree, so one module slot suffices (same assumption `draggedItem` makes).
Exports for the host + tests: `clearTreeSelection()` (empty it) and it is applied
on every render.
- `applySelectionClasses(container)` — set `aria-selected` + a `.selected` class
  on each row from `treeSelection.selected`; called at the end of `renderNoteList`
  (re-applied after any rebuild — AC-6, and survives a background repaint) and
  after each incremental change (no full re-render on a toggle, so focus is never
  disturbed). `renderNoteList` also sets `aria-multiselectable="true"` on the tree.

### Keyboard (`wireTreeKeyNav`)
Before delegating to `resolveTreeKey`, handle the selection keys:
- **Ctrl/Cmd+Space** on the focused row → `treeSelection.selected =
  toggleSelection(...)`, anchor = that row, `applySelectionClasses`, focus stays.
  (AC-1.) Must be checked before the plain Space→activate path.
- **Shift+ArrowUp/Down** → compute the next visible row via the same index math
  `resolveTreeKey` uses, `focusRow` it, set anchor to the pre-move focus if unset,
  then `treeSelection.selected = rangeSelect(visiblePaths, anchor, newFocusPath)`,
  `applySelectionClasses`. (AC-2.)
- **Delete** with a non-empty selection → `handlers.onBatchDelete?.([...selected])`.
  (AC-7.)
- **Escape** with a non-empty selection → `clearTreeSelection()` +
  `applySelectionClasses`. (AC-5.)
- Plain arrows / Home/End / Enter/Space / typeahead are unchanged and never touch
  `treeSelection` (AC-10) — the selection branches all require Shift or Ctrl/Cmd,
  which the plain paths don't have.

### Pointer (`renderNode` / `renderNoteRow` click)
Route both a folder header click and a note-name click through one
`handleRowClick(event, path, openOrToggleFolder)`:
- Ctrl/Cmd+click, **or** a plain click while `treeSelection.selected` is non-empty
  → `toggleSelection` + anchor + `applySelectionClasses`, and do **not**
  open/toggle-expand. (AC-3, AC-4.)
- Otherwise the existing behavior: a note opens (`onSelect`), a folder toggles
  expansion. (AC-4 second half.)

### Context menu (`wireTreeMenu`)
The `items()` closure checks `treeSelection`: if the row is part of a
multi-selection (`selected.size >= 2 && selected.has(path)`), return batch items
`[{label: "Move N…", …onBatchMove}, {label: "Delete N", …onBatchDelete}]`;
otherwise the existing single-row menu. (AC-8, AC-11.)

### Host wiring in `src/main.ts`
- `onBatchDelete(paths)` — one `dialog.confirm` naming the count, then for each
  path (guarded by `ifExists`) call `controller.removeFolder` if it's a known
  folder else `controller.removeNote`; collect failures into one alert. Clears the
  selection via `clearTreeSelection()` on success. Reuses the per-item primitives
  and their guards (AC-7, AC-9).
- `onBatchMove(paths)` — open the existing move picker once; on a destination,
  run `moveNoteTo`/`moveFolderTo` (the FEAT-0070/0072 helpers, which already carry
  the existence + `switchToThenIfLanded` guards) for each path; `clearTreeSelection()`.
  (AC-8, AC-9.)
- A path is a folder when `currentFolders.includes(path) || currentNotes.some(n =>
  n.startsWith(path + "/"))`, else a note — the existing `folderStillExists`
  distinction.
- Batch runs sequentially (await each) so the shared serialize queue and
  reconcile aren't raced; one failure is alerted but does not stop the rest (AC-9).

### CSS (`src/styles.css`)
A `.selected` row style (distinct from `.active` open-note and `:focus`).

Tests:
- **`src/selection-model.test.ts`**: toggleSelection add/remove/immutability;
  rangeSelect both directions, single, missing endpoint, order.
- **`src/ui.test.ts`**: Ctrl+Space toggles focused row (focus unmoved, aria-selected);
  Shift+Down selects the range + moves focus; Ctrl+click toggles without opening;
  plain click toggles when selection active / opens when not; Escape clears; plain
  arrow/typeahead leave the selection intact; context menu shows batch entries on a
  selected row and the single-row menu otherwise; Delete key calls onBatchDelete.
- **`e2e/tree-multiselect.spec.ts`** (new): select two notes (Ctrl+click / Shift+Down),
