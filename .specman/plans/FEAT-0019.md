# Sync plan — FEAT-0019 Bullet caret/glyph sync

Started: 2026-06-20T23:05:33.583Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Change how the unordered-list marker is drawn in `src/markdown-render.ts`. Today
`blockSyntaxRanges` pushes the `*`/`- ` run into `hidden` (atomic) and adds a line
mark (`cm-list-disc`/`cm-list-dash`) whose CSS `::before` draws the glyph over the
emptied line — the glyph width and the hidden run disagree, so the caret drifts
while the marker is typed. Replace that with a **widget that occupies the marker
run**: a `Decoration.replace({ widget })` over `[markerStart, afterSpace)` drawing
a fixed-width bullet. The caret maps around a replaced range, so caret and glyph
stay in sync. No new module — extend the existing `blockSyntaxRanges` /
`blockRenderingField` plumbing (a chisel-sized change).

Data model: add `bullets: BulletMark[]` to `BlockRanges`
(`{ from, to, marker: "*" | "-" }`, the run to replace). `blockSyntaxRanges` stops
emitting list `hidden`/`lines` entries and emits `bullets` instead — and **only
when the marker is completed by a trailing space** (mirrors the bare-`#` heading
rule); a bare `*`/`-` emits nothing, staying a literal char. `buildBlockDecorations`
turns each bullet into a replace-with-widget decoration, added to both the display
set and the atomic `hidden` set (so the caret steps over it — FEAT-0016 AC-7 holds).
A `BulletWidget extends WidgetType` renders `•` (disc, for `*`) or `–` (dash, for
`-`) in a fixed-width inline-block span; the old `.cm-list-*::before` rules go.

Unit-tested in `src/markdown-render.test.ts` (the pure `blockSyntaxRanges` output);
caret behavior is e2e in `e2e/` (real CodeMirror caret + the real save path), which
happy-dom can't exercise faithfully.

## AC-1 (added) — A completed marker is replaced by a fixed-width bullet widget

`blockSyntaxRanges("* item")` returns a `bullets` entry `{from:0,to:2,marker:"*"}`
and no list `hidden`/`lines` entry; `buildBlockDecorations` makes it a
replace-with-widget decoration over `[0,2)`.
Unit: assert `bullets` for `* item`/`- item`; assert no `cm-list-*` line mark.
Files: `src/markdown-render.ts`, `src/markdown-render.test.ts`

## AC-2 (added) — A bare marker with no trailing space stays visible

`blockSyntaxRanges("*")` / `"-"` returns no bullet, no hidden run, no line mark —
the marker is left literal (the heading bare-`#` precedent).
Unit: assert empty `{hidden:[],lines:[],bullets:[]}` for `*` and `-` alone.
Files: `src/markdown-render.ts`, `src/markdown-render.test.ts`

## AC-3 (added) — `*` and `-` render as distinct glyphs

The widget carries the marker; `*` → `•` (disc), `-` → `–` (dash), distinct
classes for styling.
Unit: assert the `bullets` entries carry `marker:"*"` vs `marker:"-"`.
Files: `src/markdown-render.ts`, `src/markdown-render.test.ts`

## AC-4 (added) — The caret sits at the content, not before the marker

The replace+atomic widget means the caret at the item-text start is after the
marker run; typing inserts into the text and the `*` does not reappear.
e2e: type `* a`, Home/caret to text start, type `X`, save → `* Xa` (marker intact,
no leading `*` popping before the bullet). Assert via the saved bytes + a DOM check
that the literal `*` is not in the rendered line text.
Files: `e2e/bullet-caret.spec.ts`

## AC-5 (added) — Rendering does not alter the saved document

Widget is display-only. e2e: type `* item`, save → file is exactly `* item`.
Files: `e2e/bullet-caret.spec.ts`
