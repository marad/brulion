# Sync plan — FEAT-0024 Folder tree sidebar

Started: 2026-06-21T10:08:27.374Z
Snapshot state: new
Drift summary: 0 added (whole spec)

## Approach (whole spec)

A UI/persistence change across existing modules — no new module. `ui.ts` gains a
pure `buildNoteTree` and renders the tree; `session.ts` persists the collapsed
set; `main.ts` wires it; `styles.css` styles the tree; `index.html` gets a
placeholder hint. The controller is untouched — `addNote`/`removeNote` already
work on paths (FEAT-0023), so create-into-subfolder and nested delete already
function; this phase only renders the result as a tree.

### AC-1 — pure `buildNoteTree`
NEW in `src/ui.ts`, exported:
```
export type TreeNode = FolderNode | NoteLeaf
interface NoteLeaf   { kind: "note";   name: string; path: string }
interface FolderNode { kind: "folder"; name: string; path: string; children: TreeNode[] }
export function buildNoteTree(paths: string[]): TreeNode[]
```
Walk each path's segments in input order; intern folder nodes by their full
prefix in a `Map` so repeats reuse the same node (and keep insertion order in the
parent's `children`); push a `NoteLeaf` (name via `displayName(leafSegment)`,
`path` = full path) at the leaf. Pure, no DOM. Unit-tested directly (AC-1).

### AC-2, AC-3, AC-6 — render the tree
`renderNoteList(container, notes, active, handlers, collapsed = new Set())`
rebuilds from `buildNoteTree(notes)` and renders recursively into `container`:
- A `NoteLeaf` renders the existing `.note-row` (`.note-name` button +
  `.note-delete`), `onSelect`/`onDelete` wired with `node.path` (full path, AC-3);
  the active row keeps `active` + `aria-current` matched on `node.path === active`
  (AC-6).
- A `FolderNode` renders a `.note-folder` wrapper: a `.folder-header` button
  (disclosure caret + `node.name`) and a `.folder-children` container holding its
  recursively-rendered children, indented by CSS (AC-2).
`NoteListHandlers` gains `onToggleFolder?: (path: string, collapsed: boolean) => void`.
Existing flat callers (no folders, no handler) are unaffected — a folderless tree
renders exactly the FEAT-0011 flat rows.

### AC-4 — collapse toggles in place and persists
A folder renders collapsed when `collapsed.has(node.path)` (unless force-expanded,
AC-5): the `.folder-children` gets `hidden` and the header an aria-expanded=false /
collapsed class. Clicking `.folder-header` flips `.folder-children.hidden` in place
(no rebuild) and calls `handlers.onToggleFolder?.(node.path, nowCollapsed)`.
`session.ts`: `saveCollapsedFolders(set: Set<string>)` -> `set(COLLAPSED_KEY,
[...set])`; `loadCollapsedFolders()` -> `new Set(await get<string[]>(COLLAPSED_KEY) ?? [])`
(`brulion:collapsed-folders`). `main.ts` holds the loaded `Set`, passes it to every
`renderNoteList`, and on `onToggleFolder` adds/deletes the path and persists.

### AC-5 — ancestors of the active note force-expanded
In render, a folder is shown expanded when it is an ancestor of `active`
(`active === node.path || active.startsWith(node.path + "/")`) even if
`collapsed.has(node.path)` — the persisted set is read, never written, here. So a
freshly created/restored nested note is always visible; the user's collapse of
other folders is honored.

### AC-7 — create into a subfolder
No new code: `wireNewNote` already forwards the trimmed field value to
`onCreate`, and `main.ts`'s handler calls `controller.addNote(value)`, which
normalizes a `folder/name` path (FEAT-0023) and creates+opens it. `onListChanged`
re-renders the tree with the new note active and its `projects` ancestor
force-expanded (AC-5). `index.html`: placeholder hint "New note (or folder/name)...".
Verified by AC-8 e2e and the existing `wireNewNote` unit test (value passes through).
