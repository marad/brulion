# Sync plan — FEAT-0072 New note in folder, rename, and drag-and-drop for the tree

Snapshot state: new
Drift summary: 0 added (whole spec)

## Implementation

Whole-spec plan (brand-new). Three sub-features, each additive to what P1-P3
already built; no new file-system primitive anywhere.

### 1. `src/quick-switcher.ts` — pre-fillable open()

`QuickSwitcher.open(initialQuery?: string): void` — `openSwitcher` sets
`input.value = initialQuery ?? ""` instead of always `""`, then (after
`render()`) `input.setSelectionRange(input.value.length, input.value.length)`
so the cursor lands at the end, ready to keep typing. Everything else
(render, filtering, create-on-miss) is untouched — a pre-filled query is
just a query that happens to already contain a folder prefix.

### 2. `src/ui.ts` — new menu items, no new modules

- Folder menu items gain **"New note…"** (first) and **"Rename…"** (after
  "New subfolder…", before "Move…"): `New subfolder…, New note…, Rename…,
  Move…, Delete`. Note menu items gain **"Rename…"** before "Move…":
  `Rename…, Move…, Delete`.
- `NoteListHandlers` gains `onCreateNoteIn?(path)`, `onRenameNote?(path)`,
  `onRenameFolder?(path)`, `onDropNote?(path, destination)`,
  `onDropFolder?(path, destination)`.
- **Drag source**: both `renderNoteRow`'s `row` and `renderNode`'s folder
  `header` get `draggable = true` and a `dragstart` that stashes `{ kind:
  "note" | "folder", path: node.path }` in a module-level `let draggedItem`
  (ui.ts-scoped — only one drag is ever in flight, so one shared variable is
  enough; `event.dataTransfer.setData("text/plain", node.path)` too, since
  some engines require *some* data set for the drag to register) and clears
  it on `dragend`.
- **Drag target**: only folder `header`s (never note rows — a note isn't a
  container) and the tree root get `dragover`/`drop`. `dragover` calls
  `event.preventDefault()` (required to permit a drop) only when
  `draggedItem` exists and — for a dragged folder — the target isn't the
  dragged folder itself or one of its own descendants (`isWithin`-shaped
  check, mirrored from `note-controller.ts`'s guard so the UI's "can I drop
  here" hint agrees with what the drop will actually do); adds/removes a
  `.tree-drop-target` class for the highlight. `drop` calls
  `event.stopPropagation()` (so the root zone below doesn't *also* fire),
  reads `draggedItem`, calls `handlers.onDropNote?.(path, node.path)` or
  `onDropFolder?.(path, node.path)` by kind, clears `draggedItem`.
- **Root drop zone**: `renderNoteList`'s `container` itself gets the same
  `dragover`/`drop` pair, destination `""` — since row-level drop handlers
  `stopPropagation()`, this only fires when the drop lands on empty tree
  space, not on a row.
- The actual self-nest refusal still lives in `moveFolder` (unchanged) — the
  dragover check above is a UI hint only, matching AC-7's "no valid-drop
  indicator, and forced changes nothing" (the controller call still no-ops
  even if the hint were somehow bypassed).

### 3. `src/main.ts` — wiring, no new controller methods

- Extract `moveNoteTo(path, destination)` / `moveFolderTo(path, destination)`
  helpers from the existing `onMoveNote`/`onMoveFolder` bodies (same target-
  path computation, `renameActive`/`moveFolder` call) — reused by both the
  picker's `onPick` callback and the new drop handlers, so there is exactly
  one place "what path does a move actually target" is computed.
- `onCreateNoteIn: (path) => switcher.open(`${path}/`)`.
- `onRenameNote`/`onRenameFolder`: `window.prompt("Rename to:", currentLeaf)`
  (matching `promptNewFolder`'s native-dialog pattern), a small local
  `parentOf(path)` helper (mirrors `note-controller.ts`'s private `folderOf`,
  duplicated rather than exported since it's a one-liner and the controller's
  version is intentionally private), compute the same-parent target, call
  `renameActive`/`moveFolder`, `window.alert(result.reason)` on failure. The
  note variant switches to the note first (`controller.switchTo(path).then(...)`),
  same constraint `onMoveNote` already works around.
- `onDropNote: (path, destination) => controller.switchTo(path).then(() => moveNoteTo(path, destination))`,
  `onDropFolder: (path, destination) => moveFolderTo(path, destination)`.

### 4. `src/styles.css` — drag affordance + drop-target highlight

`[draggable="true"]` rows get `cursor: grab` (`cursor: grabbing` while
`:active`); `.tree-drop-target` (folder header or root zone mid-dragover)
gets a visible outline/background using existing tokens (`--accent`/
`--surface-2`), no new custom properties.
