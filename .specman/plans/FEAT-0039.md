## Design

Recency is an MRU list of visited note paths, maintained + persisted by the app
and fed into `searchNotes` as a new sort key. No new module: the pure logic lives
in `note-search.ts` (the M21 "search ranking & recency" home), persistence in
`session.ts`, wiring in `main.ts`/`quick-switcher.ts`.

**`src/note-search.ts`**

- `touchRecency(list: readonly string[], path: string, cap = 50): string[]` — pure
  MRU update: returns `[path, ...list.filter(p => p !== path)].slice(0, cap)`. New
  array (no mutation). Path to front, deduped, capped. (AC-1)
- `searchNotes(query, paths, recency: readonly string[] = [])` — add an optional
  recency arg (most-recent first; default `[]` keeps existing callers untouched).
  Build `rank = new Map(recency.map((p, i) => [p, i]))`; `rankOf(p) = rank.get(p) ??
  Infinity`. Extend the comparator to:
  `b.score - a.score || rankOf(a.path) - rankOf(b.path) || path asc`.
  - Empty query → all scores `0` → order collapses to `rankOf` then path = visited
    notes MRU-first, then never-visited in name order. (AC-2)
  - Non-empty query → score dominates; recency only orders equal scores; path is
    the final tiebreak. (AC-3, AC-4)
  - A stale recency entry just never appears in `paths`, so it is never emitted and
    does not perturb live ordering. (AC-7)
  `fuzzyScore` and the FEAT-0038 scoring are untouched.

**`src/session.ts`** — mirror the existing `brulion:` helpers:

- `saveRecency(paths: readonly string[]): Promise<void>` → `set("brulion:recency",
  [...paths])`.
- `loadRecency(): Promise<string[]>` → `(await get<string[]>("brulion:recency")) ??
  []`. (AC-5)

**`src/quick-switcher.ts`** — add `getRecency: () => readonly string[]` to
`QuickSwitcherDeps`; in `render()` call `searchNotes(input.value, deps.getNotes(),
deps.getRecency())`. (Read per render, so it reflects the latest visits.)

**`src/main.ts`** — maintain the MRU list and feed both the switcher and the writes:

- `let recency: string[] = []`; load once at startup (`void loadRecency().then(r =>
  { recency = r })`, alongside the other restores).
- In `onListChanged`, after computing `active`: when `recency[0] !== active`,
  `recency = touchRecency(recency, active); void saveRecency(recency)` — record the
  visit on a genuine active-note change only (skips the redundant re-touch when an
  external list change fires with the same active).
- Pass `getRecency: () => recency` into `mountQuickSwitcher`.

## Tests

Unit — `src/note-search.test.ts` (new `recency (FEAT-0039)` block):

- **AC-1** — `touchRecency`: front+dedup (`touchRecency(["a","b"], "b")` →
  `["b","a"]`); new path prepends; cap drops the oldest (`touchRecency(list, "x", 2)`
  length ≤ 2); input array not mutated.
- **AC-2** — empty query: `searchNotes("", ["a.md","b.md","c.md"], ["c.md","a.md"])`
  → `["c.md","a.md","b.md"]` (visited MRU-first, then name order), `create` null.
- **AC-3** — equal-score tiebreak: `searchNotes("note", ["note-1.md","note-2.md"],
  ["note-2.md"])` → `["note-2.md","note-1.md"]` (both score equally; recency wins).
- **AC-4** — recency never beats a better match:
  `searchNotes("abc", ["abc.md","axbxc.md"], ["axbxc.md"])` → `["abc.md", …]`
  (substring beats the recently-visited scattered match).
- **AC-7** — stale entry: `searchNotes("", ["a.md","b.md"], ["ghost.md","b.md"])` →
  `["b.md","a.md"]` (ghost absent, b first by recency).
- Back-compat: existing `searchNotes` calls without recency still pass (default `[]`).

Unit — `src/session.test.ts`: `loadRecency` returns `[]` when nothing stored;
`saveRecency`→`loadRecency` round-trips an array (idb-keyval is already used here).

Unit — `src/quick-switcher.test.ts`: add `getRecency: () => []` (or a stub) to the
deps in existing tests; one test that a non-empty recency reorders the empty-query
list (assert row order).

E2E — `e2e/quick-switcher.spec.ts` (or a focused add): seed ≥3 notes, open one,
switch to another (two visits), open the switcher with an empty query, assert the
two just-visited notes are the top rows in most-recently-visited order. (AC-6)
