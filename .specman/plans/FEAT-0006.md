# Sync plan — FEAT-0006 Hidden-syntax rendering

Started: 2026-06-20T06:56:28.872Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Rendering goes through a new `src/markdown-render.ts` module added to the editor
in `src/editor.ts`. A pure `markdownSyntaxRanges(state, from, to)` walks the
Lezer markdown syntax tree (`@codemirror/lang-markdown`) and returns the markup
runs to hide and the content spans to style — no document mutation, so the
on-disk bytes are untouched (the moat). A `ViewPlugin` turns those into
`Decoration.replace` (hide) and `Decoration.mark` (style) over the visible
viewport and exposes the hidden runs as `EditorView.atomicRanges`. Markup is
hidden on every line including the caret line; the plugin rebuilds only on
document/viewport change (not on caret moves).

## AC-1 (added) — Heading markers hidden, heading styled

The `#` run plus its trailing space are hidden; the heading line is marked
`cm-heading cm-hN`, styled larger by level in `EditorView.baseTheme`. Verified by
e2e: rendered text of a `# Title` line has no `#`, and `.cm-heading` font-size >
body font-size.
Files: `src/markdown-render.ts`, `src/editor.ts`, `e2e/rendering.spec.ts`

## AC-2 (added) — Bold markers hidden, text bold

`StrongEmphasis` content marked `cm-strong` (font-weight 700); the `**`/`__`
`EmphasisMark` children hidden. Verified by e2e: no `*` in rendered text and
`.cm-strong` computed font-weight ≥ 700. Unit-tested via `markdownSyntaxRanges`.
Files: `src/markdown-render.ts`, `e2e/rendering.spec.ts`, `src/markdown-render.test.ts`

## AC-3 (added) — Italic markers hidden, text italic

`Emphasis` content marked `cm-em` (font-style italic); `*`/`_` marks hidden.
Verified by e2e: no `*` in rendered text and `.cm-em` font-style italic.
Files: `src/markdown-render.ts`, `e2e/rendering.spec.ts`, `src/markdown-render.test.ts`

## AC-4 (added) — Inline-code backticks hidden, text monospace

`InlineCode` content marked `cm-inline-code` (monospace font); `CodeMark`
backticks hidden. Verified by e2e: no `` ` `` in rendered text and
`.cm-inline-code` font-family includes monospace.
Files: `src/markdown-render.ts`, `e2e/rendering.spec.ts`, `src/markdown-render.test.ts`

## AC-5 (added) — Markup stays hidden on the caret line

The plugin rebuilds decorations only on doc/viewport change, never on selection,
and never special-cases the caret line — so markup stays hidden when the caret is
on the line. Verified by e2e: caret placed on a `**bold**` line, rendered text
still has no `*`.
Files: `src/markdown-render.ts`, `e2e/rendering.spec.ts`

## AC-6 (added) — Rendering does not alter the saved document

Decorations only hide/style; `markdownSyntaxRanges` never mutates state. Verified
by e2e: type `# Title` + `a **bold** word`, Ctrl+S, on-disk `start.md` equals the
exact markdown typed.
Files: `src/markdown-render.ts`, `e2e/rendering.spec.ts`

## AC-7 (added) — Caret steps over hidden markers atomically

Hidden runs are provided as `EditorView.atomicRanges`. Verified by e2e: Home then
ArrowLeft over a `**bold**` line jumps the whole `**` run, so an inserted char
lands before it (`X**bold**`), never between the two `*`.
Files: `src/markdown-render.ts`, `e2e/rendering.spec.ts`
