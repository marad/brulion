## Approach

Typeahead is another key the existing tree keyboard-nav pipeline (FEAT-0075)
handles, but it does not go through `resolveTreeKey` (a fixed key‚Üíaction switch):
it kicks in only for printable keys that resolve to `none`, and the matching is a
new pure function.

Pure core in **`src/tree-nav.ts`**:
- `typeaheadMatch(labels: string[], current: number, buffer: string): number` ‚Äî
  return the index of the next label (searching `current+1 ‚Ä¶ end`, then wrapping
  `0 ‚Ä¶ current`) whose lowercased value `startsWith` the lowercased `buffer`, or
  `-1` if none. Searching after `current` and wrapping gives both "jump to next
  match" (AC-1) and same-letter cycling (AC-4) for free; including `current` last
  means a buffer that only the focused row matches keeps focus there. Empty
  `buffer` or empty `labels` ‚Üí `-1`. (AC-1, AC-4, AC-5.)
- `isTypeaheadKey(e: {key: string; ctrlKey: boolean; metaKey: boolean; altKey:
  boolean}): boolean` ‚Äî true for a single-character `key` with no Ctrl/Cmd/Alt.
  This distinguishes a printable character from `ArrowDown`, `F2`, etc. Space
  (`" "`) is a single char but `resolveTreeKey` already claims it as `activate`,
  so it never reaches the typeahead branch. (AC-7.)
- `isModifierKey(key: string): boolean` ‚Äî true for a bare modifier key
  (`Shift`/`Control`/`Alt`/`Meta`/`AltGraph`/`CapsLock`). Used so a modifier
  keydown does not end a typeahead session (Shift+letter must still compose). (AC-8.)

DOM glue in **`src/ui.ts`** (`wireTreeKeyNav`, already holds the container-scoped
closure and the `descriptors`/`els` for the visible rows):
- Add a per-container typeahead buffer + reset timer in the closure. Extract a
  `TYPEAHEAD_TIMEOUT_MS = 500` constant (the coalescing window; WAI-ARIA-typical).
- A `resetTypeahead()` closure helper clears the timer and empties the buffer;
  it runs (a) when a real action is resolved, (b) when a non-typeahead,
  non-modifier key claims no action, and (c) on `focusout` of the container when
  focus leaves the tree. Together these end a stale session so a fresh letter
  never extends it. (AC-8.)
- When `resolveTreeKey` returns `none`, if `isTypeaheadKey(event)`: extend the
  buffer ‚Äî **except** a repeat of the single character already buffered keeps it
  one character (`typeaheadBuffer === event.key` ‚Üí leave as-is) so same-letter
  presses cycle instead of growing to `aa` (AC-4). Restart the reset timer
  (`clearTimeout` + a fresh `setTimeout` emptying the buffer after the timeout ‚Äî
  AC-2, AC-3), compute `labels = els.map(el => el.textContent ?? "")`, call
  `typeaheadMatch(labels, current, buffer)`, and on a hit `event.preventDefault()`
  + `focusRow(container, els[idx])`. On `-1`, leave focus and do not
  preventDefault. (AC-1..AC-5.)
- Else (a non-typeahead key that claimed no action, and not a bare modifier):
  `resetTypeahead()` ‚Äî a boundary arrow, Escape, or Tab ends the session. (AC-8.)
- Typeahead only focuses rows ‚Äî it never clicks a row or calls a handler, so it
  opens/toggles/writes nothing. (AC-6.)
- All navigation/activation/rename keys still return non-`none` from
  `resolveTreeKey` and are handled before the typeahead branch, unchanged, but
  now `resetTypeahead()` first so a following letter starts fresh. (AC-7, AC-8.)

Tests:
- **`src/tree-nav.test.ts`**: `typeaheadMatch` ‚Äî next match after current;
  wrap-around; same-letter cycling across repeated calls; case-insensitive;
  multi-char prefix; no match ‚Üí -1; empty buffer/labels ‚Üí -1; only-current-matches
  ‚Üí returns current. `isTypeaheadKey` ‚Äî true for `"a"`/`"7"`/`"-"`; false for
  `"ArrowDown"`/`"F2"`/`"Enter"`; false when ctrl/meta/alt held.
- **`src/ui.test.ts`** (fake timers): typing `p` focuses the next `p`-row (AC-1);
  `r` then `e` within the window focuses the `re`-row (AC-2); advancing the timer
  past the window then typing resets to a single-char search (AC-3); repeating a
  letter cycles + wraps (AC-4); an unmatched key leaves focus and calls no
  handler (AC-5); typing calls neither `onSelect` nor `onToggleFolder` (AC-6).
- **`e2e/tree-keyboard-nav.spec.ts`**: with several notes, focus one, type a
  leading letter, assert focus lands on the matching row and no editor content
  changed (movement only ‚Äî AC-1, AC-6 end-to-end).
