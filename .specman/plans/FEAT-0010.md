# Sync plan — FEAT-0010 Multi-note storage layer

Started: 2026-06-20T11:51:08.755Z
Snapshot state: new
Drift summary: 0 added (whole spec)

Generalize the storage layer from the single hardcoded `start.md` to named
notes, and add the folder-level operations the M3 UI will need. Two units:

- `src/note.ts` — `readNote(dir, name)` / `saveNote(dir, name, content, mtime)`
  gain a `name` parameter (the `start.md` default moves up to the callers);
  `listNotes(dir)`, `createNote(dir, name)`, `deleteNote(dir, name)` are added.
  The existing per-note conflict guard and `getExisting` helper are reused.
- `src/note-name.ts` (NEW) — pure `normalizeNoteName(input)` returning either a
  valid `.md` filename or an invalid result, no FSA/DOM dependency.

`src/note-controller.ts` and `src/main.ts` already call `readNote`/`saveNote`
with the old single-arg shape; they're updated to pass `"start.md"` so M1/M2
behavior is unchanged (the active-note wiring is FEAT-0011). Tests come before
bodies per the excavate/chisel discipline.

## AC-1 / AC-2 — Read a note by name; missing reads as empty
`readNote(dir, name)` takes the filename instead of the module constant. Body is
the existing read: `getExisting(dir, name)` → text + `lastModified`, or
`{content:"", lastModified:null}` when absent. No file is created on read.
Files: `src/note.ts`, `src/note.test.ts`

## AC-3 / AC-4 — Write by name, creating when absent; conflict guard preserved
`saveNote(dir, name, content, knownMtime)` takes the filename. Body unchanged
otherwise: compare on-disk `lastModified` to `knownMtime` before overwriting,
return `{status:"conflict"}` on mismatch (or a file appearing where none was
seen), else write via `getFileHandle(name,{create:true})` + `createWritable` and
return the new mtime. The guard is now per the passed name.
Files: `src/note.ts`, `src/note.test.ts`

## AC-5 / AC-6 — List the folder's `.md` files, sorted, markdown only
`listNotes(dir)`: iterate `dir.values()`, keep entries whose `kind === "file"`
and whose name ends in `.md` (case-insensitive), collect names, sort with
`localeCompare(..., {sensitivity:"base"})` for case-insensitive order. Non-`.md`
files and directories are skipped; an empty/markdown-free folder yields `[]`.
Files: `src/note.ts`, `src/note.test.ts`

## AC-7 / AC-8 — Create an empty note; refuse to overwrite
`createNote(dir, name)`: if `getExisting(dir, name)` returns a handle, return an
"already exists" result without touching it; otherwise `getFileHandle(name,
{create:true})` + open/close an empty writable to materialize an empty file.
Returns a small result type distinguishing `created` from `exists`.
Files: `src/note.ts`, `src/note.test.ts`

## AC-9 — Delete a note
`deleteNote(dir, name)`: `dir.removeEntry(name)`. (Absent file: the FSA throws
`NotFoundError`; deleting a note we just listed is the normal path, so we let it
surface — callers operate on listed names.)
Files: `src/note.ts`, `src/note.test.ts`

## AC-10 / AC-11 — Normalize a typed name; reject unsafe/empty
NEW `src/note-name.ts`: `normalizeNoteName(input): {ok:true; filename} |
{ok:false; reason}`. Trim; reject empty/whitespace-only; reject if it contains a
path separator (`/`,`\`) or other filename-unsafe chars (control chars and the
Windows-reserved set `<>:"|?*`); strip a trailing `.md` (case-insensitive) then
re-append `.md` so exactly one extension results. Pure — unit-tested directly.
Files: `src/note-name.ts`, `src/note-name.test.ts`

## Callers — keep M1/M2 behavior (no AC; non-regression)
`src/note-controller.ts` and `src/main.ts` pass `"start.md"` to the now-named
`readNote`/`saveNote` so the single-note flow is unchanged until FEAT-0011 wires
the active note. `note-controller.test.ts` updated for the new call signature.
Files: `src/note-controller.ts`, `src/note-controller.test.ts`, `src/main.ts`
