# M7 — Conflict diff/preview

**Goal:** make the M4 conflict choice (FEAT-0015) an *informed* one. Today, when
the conflict modal appears, the user is asked to "keep my version" or "use the
version on disk" with no way to see *what* differs — "use the version on disk" is
a blind choice. M7 shows both sides, changes highlighted, while the conflict
stands. **Display-only: no new bytes touch the file** — the diff is a read view
over the buffer and the disk; resolution still runs through the existing
controller path (FEAT-0015), re-basing on live disk state.

Success: when a conflict is raised, the modal shows your buffer beside the
on-disk file with the differences highlighted, so the keep/take pick is informed;
the deleted-on-disk case shows the disk side as empty/deleted; resolving either
way clears the preview and editing resumes exactly as before.

## Phase 1 — Conflict diff view
The conflict modal (FEAT-0015) currently shows only a message and two buttons.
Surface the two versions in it: the unsaved buffer ("your version") beside the
on-disk file ("on disk"), changes highlighted, read-only.

To do that the controller must hand the UI both sides when it raises the
conflict: `onConflict` carries `{ mine, theirs }`, where `theirs` is the on-disk
content or `null` when the file was deleted on disk. `raiseConflict` reads the
disk file at raise time (`readNote` — a `null` mtime means deleted). The view is
a read-only `@codemirror/merge` `MergeView` (mine left, disk right, matching the
button order), built fresh when the conflict is raised and destroyed when it
resolves.

- **Done =** raising a conflict shows the buffer and the on-disk content
  side-by-side with the differences highlighted, read-only, inside the modal; the
  deleted-on-disk case shows the disk side empty under a "(deleted on disk)"
  label; the two buttons keep FEAT-0015 semantics and resolving either way
  destroys the diff view and returns the editor to normal; unit tests cover the
  `onConflict` payload (mine/theirs, null = deleted) and the view build/teardown,
  an e2e test confirms the diff is visible in the modal and resolution clears it.

## Out of scope (deliberately)
- **Three-way / line-level merge or per-hunk apply** — still the two-way
  keep/take choice from FEAT-0015; M7 only *shows* the difference, it does not add
  a merge or cherry-pick path.
- **Editing inside the preview** — the diff panes are read-only; the buffer is
  edited only after resolution, in the normal editor.
- **Diffing background (non-open) notes** — only the open note carries unsaved
  local state and can conflict (FEAT-0015 scope).
