# M16 — Settings modal

**Goal:** a single home for preferences — real new appearance settings (font, text
size, editor width) plus the relocated Vim toggle. Settings live in **a file in the
open folder** (`.brulion.json` at root) so they travel with the vault across
machines and OSes; **no idb cache, no "defaults vs current" concept** — the file is
the single source of truth, and before a folder is open the built-in defaults apply.

The file is an **opaque, non-`.md` file**: `listNotes` already collects only `.md`
files, so `.brulion.json` never appears in the note list and never trips the M4
poller — that requirement is satisfied by the existing filter (verified, not new
code).

Scope (agreed with the user, from the roadmap):

- **Font** — pick from installed fonts (`queryLocalFonts()`, one-time permission,
  Chromium-only — consistent with the FSA-only stance) and build an **ordered font
  stack** with CSS-style fallback; a generic family floor is auto-appended so there
  is always a usable font across OSes. When font-access is unavailable/denied, fall
  back to a small curated preset list — **never** free-typing font names.
- **Text size** — a px stepper (~12–24); H1/H2/H3 are relative multipliers already
  baked into the theme (the headings use `em`), so one base knob scales the whole
  hierarchy proportionally for free.
- **Editor width** — three presets: Narrow (current 68ch default) / Wider / Full.
- **Vim toggle** — moved here from the header; the visible header button is removed.
  The `Ctrl/Cmd+;` toggle shortcut (FEAT-0021) stays unchanged.
- **Entry point** — a gear icon in the header + `Ctrl/Cmd+,`.

Out (deliberately): theme (split to **M18**), sidebar width (M13, set by drag),
default folder-collapse (a sidebar behavior, not appearance).

## P1 — Settings model, `.brulion.json` persistence, and the apply layer

The engine under the modal: a pure settings model + the disk IO + the function that
applies settings to the live editor. No modal UI yet — but the apply path is wired
into folder-open and the `Ctrl/Cmd+;` Vim shortcut, so the milestone is functional
(and Vim is migrated off idb) before the modal lands.

A new `settings.ts` owns:
- the `Settings` type (`font: string[]`, `textSize: number`, `editorWidth`, `vim`)
  and `DEFAULT_SETTINGS`;
- `normalizeSettings(raw)` — defensively coerce arbitrary loaded JSON to a valid
  `Settings` (clamp text size to 12–24, validate the width enum, keep only string
  font entries, coerce `vim` to boolean), so a hand-edited or stale file never
  breaks the app;
- `buildFontStack(fonts)` — quote families that need it, join, and append the
  generic floor;
- `loadSettings(dir)` / `saveSettings(dir)` — read/write `.brulion.json` at the
  folder root (read returns `DEFAULT_SETTINGS` on absent file or bad JSON).

`applySettings(view, settings)` sets three CSS custom properties
(`--font-stack`, `--editor-font-size`, `--editor-measure`) on the document root and
toggles Vim via the existing `setVimMode`. The editor theme reads the two new vars
(headings already use `em`, so text-size scales the hierarchy); width maps
Narrow→68ch / Wider→90ch / Full→none.

**Vim migrates from idb to the settings file.** The `brulion:vim` idb key and
`saveVimMode`/`loadVimMode` are retired; Vim is now a field in `.brulion.json`, so
it travels with the vault. The `Ctrl/Cmd+;` shortcut flips `settings.vim`, persists
the file, and re-applies.

**Done =** opening a folder reads `.brulion.json` and applies font/size/width/Vim to
the editor; an absent or malformed file falls back to defaults without error; the
`Ctrl/Cmd+;` shortcut toggles Vim and writes the file; the file never appears in the
note list or trips the poller (verified); unit tests cover `normalizeSettings`
(clamping/validation), `buildFontStack` (quoting + generic floor), and the
load/save round-trip; an e2e spec opens a folder containing a `.brulion.json` and
asserts the editor reflects it.

## P2 — Settings modal UI, entry points, and local-font access

The visible surface: a modal with the four controls, its two entry points, and the
local-font picker.

A new `settings-modal.ts` builds the modal DOM (font control, text-size stepper,
width preset group, Vim toggle), handles open/close (Esc, backdrop click), and
reports every change through one `onChange(patch)` callback that main.ts merges into
the current settings, applies, and persists. The header gains a **gear icon**
(replacing the removed Vim button) and `Ctrl/Cmd+,` opens the same modal.

The **font control** calls `queryLocalFonts()` behind a one-time permission; on
success the user picks from installed families to build the ordered stack. When the
API is unavailable (non-Chromium) or permission is denied, it falls back to a small
curated preset list — selection only, never free-typing a font name.

**Done =** the gear icon and `Ctrl/Cmd+,` both open the modal; changing text size,
width, font, or Vim applies live and writes `.brulion.json`; the header Vim button
is gone (the modal toggle and `Ctrl/Cmd+;` remain the controls); the font control
lists local fonts where the API allows and the curated presets where it doesn't;
unit tests cover the modal's change wiring and the font-list fallback; an e2e spec
drives the modal on real Chromium (open via gear + shortcut, change each control,
assert applied and persisted to `.brulion.json`).
