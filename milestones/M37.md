# M37 — Sidebar tree follow-ups

**Goal:** the comfort items that came out of the M36 review — a direct rename
shortcut plus the movement niceties M36 deliberately deferred. Builds on M35
(context-menu actions, FEAT-0069…0074) and M36 (keyboard tree, FEAT-0075).

Scope (decided autonomously in the M36 review + here; recorded in
`DECISIONS.md`; reviewed live at the end):

- **F2 renames the focused row** — reuse the *exact* existing rename flow
  (`onRenameNote`/`onRenameFolder` → the M35 in-app prompt), so there is one
  rename path, not a second. F2 is the file-explorer / IDE convention.
- **Typeahead moves focus, it never opens** — typing a letter (or a short run,
  coalesced within a timeout) moves the roving focus to the next *visible* row
  whose name starts with the buffer; it changes no file and opens nothing. Pure
  matching core in `tree-nav.ts`, mirroring `resolveTreeKey`.
- **Multi-select earns its keep via batch actions** — selection is not built as
  dead UI. It exists to drive **batch Delete and batch Move** through the
  existing `removeNote`/`moveNote`(/folder) primitives; without a consumer it
  would violate the lean ethos. Keyboard (Shift+Up/Down to extend, Ctrl/Cmd+Space
  to toggle) *and* pointer/touch (tap-to-toggle once a selection exists) reach it,
  which is also where the genuine **touch** value lives.
- **A standalone "touch gesture for movement" is descoped** — on touch, movement
  already works (scroll to move, tap to open, tap a folder header to
  expand/collapse); a bespoke movement gesture would only duplicate the scroll.
  The real touch win is multi-select above. Recorded as a decision for the review,
  not silently dropped (the user added it in the M36 review; the leanest honest
  reading is "no separate movement gesture").

## P1 — F2 to rename

**Done =** with focus on a tree row, pressing F2 opens the same rename prompt the
row's context-menu "Rename…" opens (note → `onRenameNote`, folder →
`onRenameFolder`); no other key changes behavior and no file is written except
through the existing rename path.

## P2 — Typeahead-to-focus

**Done =** with focus in the tree, typing a printable character moves focus to
the next visible row whose display name starts with what was typed (successive
characters within a short timeout extend the search; the timeout resets it);
matching wraps around from the focused row, is case-insensitive, and never opens
a note or writes a file.

## P3 — Multi-select + batch delete/move

**Done =** several rows can be selected (keyboard: Shift+Up/Down extends a
contiguous range, Ctrl/Cmd+Space toggles the focused row; pointer: tap toggles a
row once a multi-selection is active), the selection is visually marked, and a
batch **Delete** and batch **Move** act on the whole selection through the
existing per-item primitives (each with the same existence/conflict guards a
single-row action uses); Escape clears the selection.
