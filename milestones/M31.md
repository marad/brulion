# M31 — Journaling: weekly journal navigation

**Goal:** make reaching *this week's* journal one keystroke. Builds on M30 (actions)
and M16 (settings). Scoped down live with the user to **navigation only** — the
quick day-log and a week template are deferred (see `DECISIONS.md`).

Scope (decided live with the user, recorded in `DECISIONS.md`):

- **A date-templated `journalPath` setting** (`.brulion.json`, default empty) with
  placeholders expanded against today: `{year}`/`{month}`/`{day}` and
  `{mondayOfTheWeek}` (ISO date of the week's Monday; week starts Monday, fixed).
- **An "Open this week's journal" action** (M30 registry → palette + pinnable): expand
  the template, normalize to a note path, and open it through the **existing**
  open-note path — switch if it exists, else the existing create-on-miss prompt. No
  bespoke creation, no template seeding. An empty/invalid path opens settings.

Out (deliberately, deferred): the quick day-log (append to today's section — needs a
defined weekly-note structure); a week template seeding per-day headings; a general
per-note-type template system; configurable week-start/locale.

## P1 — Weekly journal navigation

A pure `src/journal.ts` `expandJournalPath(template, date)` (placeholder expansion +
the week's-Monday computation, correct across month/year boundaries); a `journalPath`
field on the M16 settings model + a text input (with a token hint) in the settings
modal; an "Open this week's journal" registry action in `main.ts` that expands →
`normalizeNoteName` → `openNotePath`. Pure logic unit-tested with fixed dates; the
real open/create + the empty-path→settings nudge covered by e2e.

**Done =** with `journalPath` configured, the action opens this week's journal note
(creating it via the existing prompt if missing); an empty path opens settings; path
expansion is correct across month/year boundaries; no bytes are written beyond the
preference and the user-confirmed note.
