# M26 — Table rendering

**Goal:** a leading-pipe markdown table (`| a | b |` + the `|---|` separator row)
renders as a real aligned table, not raw pipes and dashes. Same spirit as M5
(rendering gaps), M15 (highlighting), M28 (Mermaid): **visual only** — moat-critical
that we do NOT touch the bytes.

Scope (decided autonomously, recorded in `DECISIONS.md`):

- **Render a detected table block as an aligned `<table>` block widget** (a
  `Decoration.replace` from a whole-doc `StateField`, the Mermaid mechanism); the raw
  pipe source is revealed when the selection is inside the block.
- **Detect by a fence-aware line scan** (the base markdown grammar doesn't parse GFM
  tables): a separator row (`:---`/`:--:`/`---:`/`---`) with a non-blank header above +
  contiguous body below; outer pipes optional; lines inside fenced code skipped.
- **Alignment from the separator; ragged rows padded/truncated; cells are plain text**
  (inline markdown inside cells deferred). Bytes unchanged.

Out (deliberately): inline markdown inside rendered cells; editing cells in place (tab
between cells); a GFM-tables parser extension; escaped-pipe (`\|`) handling.

## P1 — Table block rendering

A new `src/table-render.ts` (mirrors `mermaid-render.ts`): `findTableBlocks` (pure,
fence-aware line scan → header/rows/aligns + block range), a `TableWidget` that builds
the `<table>` (pad/truncate to the separator's column count, per-column alignment), a
`StateField` + click-to-reveal, registered after `markdownRendering` in `editor.ts`;
styles in `styles.css`. The pure detection is unit-tested (alignment, outer-pipes,
fence skip, ragged rows, flush-adjacent tables, boundaries); the render + reveal +
bytes-unchanged covered by e2e.

**Done =** a `| … |` table with a separator row renders as an aligned table with
column borders and per-column alignment; selecting inside reveals the raw source;
a table inside a code fence stays code; the on-disk bytes are unchanged.
