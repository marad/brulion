# M27 — Settings & header polish

**Goal:** two small follow-ups to the M16 settings modal, both from real daily use;
no change to file behavior, no new module.

Scope (agreed with the user in the M27 elicitation):

- **Move "change folder" into Settings.** Relocate the folder-switch action out of
  the header (`#reopen-folder`, the plain "Switch folder" button) into the M16
  settings modal as a dedicated **Folder** section at the bottom, showing the
  **current folder's name** plus a **"Switch folder…"** button. Picking a new
  folder runs the existing open flow (reloads notes + the new folder's
  `.brulion.json` settings) and the modal closes. The header keeps **no** folder
  indicator — it stays lean (note identity + sidebar toggle + gear).
- **Better settings icon.** The header entry-point glyph is the Unicode `⚙`, which
  renders inconsistently across OS/fonts (often a colored emoji or a boxy shape)
  and "doesn't look like a gear". Swap it for an **inline SVG gear** (line-style,
  monochrome, `currentColor`) so it looks identical everywhere and matches the
  other header controls. Metaphor unchanged (still a gear); the `Ctrl/Cmd+,`
  shortcut and the `aria-label`/title are unchanged.

Out (deliberately): any change to the settings model, the `.brulion.json` format,
the first-run welcome flow (the welcome CTA still opens the first folder — switching
only applies once a folder is open), or the open/restore folder logic itself.

## Phases

- [x] **P1 — Folder section in Settings + SVG gear icon** (`FEAT-0054`)

## P1 — Folder section in Settings + SVG gear icon

Both header-polish changes in one phase: the substantive one (relocate the
folder-switch into the settings modal, with the modal closing on a successful pick)
and the trivial one (replace the `⚙` glyph with an inline SVG gear). Touches
`index.html`, `src/settings-modal.ts`, `src/main.ts`, `src/styles.css` — DOM/CSS
glue only, no new module and no change to file behavior.

**Done =** the header no longer shows a "Switch folder" button; the settings modal
has a Folder section naming the open folder with a "Switch folder…" button that
reopens the picker, reloads the workspace, and dismisses the modal; the header
settings entry point is a crisp inline-SVG gear that renders consistently and keeps
its `Ctrl/Cmd+,` shortcut.

## P2 — Consistent icon set (Lucide) + header button normalization

From the M27 review: P1's hand-rolled gear SVG sat inconsistently beside the ☰
sidebar toggle (a Unicode glyph), and the two header buttons even rendered at
different heights — a text glyph sizes by line-height, an SVG by its box. Rather
than keep hand-authoring one-off SVGs, adopt a tree-shakeable icon set (**Lucide**,
the maintained successor to Feather — so the existing gear stays visually the same)
and route the header icons through it. Convert the ☰ toggle to a Lucide icon from
the same family, size all header icons uniformly, and normalize the header button
box so the controls share one height. Editor/file behavior untouched; this is pure
header chrome. Touches `package.json`, `index.html`, `src/main.ts`, `src/styles.css`.

**Done =** the settings gear and the sidebar toggle are both Lucide icons of equal
size, the header buttons render at the same height, and no hand-authored icon SVG
remains in `index.html`.
