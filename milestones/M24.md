# M24 — Scroll/caret preservation on external refresh

**Goal:** when the M4 poller reloads a note changed on disk, stop the view from jumping
to the top and stop losing the caret. Editor/refresh layer; moat-neutral (no `.md`
write). Surfaced as a daily-use annoyance: editing a note that another tool also touches
yanked the reader to the top on every poll.

Scope (decided autonomously, recorded in `DECISIONS.md`):

- **Minimal-diff reload, no library.** A hand-written `diffRange` computes the single
  differing span (longest common prefix + non-overlapping suffix → one
  `{from, to, insert}`); the external-refresh path replaces only that span instead of
  the whole document.
- **Caret maps for free.** The reload dispatches only `changes` (no explicit
  selection), so CodeMirror maps the existing caret through them.
- **Scroll anchored.** The top-of-viewport line is captured, mapped through the change,
  and scrolled back (`scrollIntoView`, `y: "start"`).
- **Guards untouched.** `refreshFromDisk`'s conflict / clobber-prevention logic and the
  initial-load / note-switch path keep the wholesale `setEditorText`.

Out (deliberately): a multi-hunk / line diff or sharing a diff structure back into the
M7 conflict view (MergeView computes its own); pixel-exact intra-line scroll restore;
preserving scroll/caret across a conflict resolution.

## P1 — Minimal-diff reload preserving scroll + caret

NEW `src/text-diff.ts` `diffRange(old, next)` (pure, unit-tested across prefix/suffix/
middle/insert/delete/full-replace/identical/empty/overlap-run cases). NEW
`reloadEditorText(view, text)` in `editor.ts` dispatches the minimal change with the
`ProgrammaticLoad` annotation, maps the viewport anchor, and scrolls it back; a no-op
when the buffer already matches. `note-controller.ts` `refreshFromDisk` calls it on the
safe-to-replace path. Caret preservation is unit-tested in happy-dom; the scroll-not-
jumping behavior is an e2e in real Chromium, which also asserts no echo-write.

**Done =** an external edit to the open note reloads in place — the caret keeps its
logical position and the view stays where the reader was (no jump to top) — the
conflict guard still fires on unsaved local edits, and no `.md` bytes are written.
