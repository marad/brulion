# M1 — Full pipeline on a single note

**Goal:** prove the whole plumbing works end-to-end on one note (`start`),
before adding anything pretty or multi-note. Success: notes for real survive a
restart from a deployed URL.

## Progress
- [x] Phase 1 — Skeleton + deploy
- [ ] Phase 2 — Folder access
- [ ] Phase 3 — Persist the handle
- [ ] Phase 4 — Read/write `start`
- [ ] Phase 5 — Typography polish

Phases are ordered to **de-risk early** — deploy first (confirm hosting works at
all), then build up File System Access incrementally, polish last.

> Blocked on the open **framework** decision (vanilla TS vs micro-framework) —
> settle it before Phase 1, see `ROADMAP.md`.

Constraints that apply throughout:
- **Secure context** required by the FSA API — satisfied by GitHub Pages (https)
  and localhost.
- **User gesture** required for the first folder access — must come from a click,
  not `onload`.

## Phase 1 — Skeleton + deploy
- Vite + TypeScript + CodeMirror 6, minimal app shell.
- GitHub Pages deploy via Actions.
- **Done =** a blank/hello app is live on the Pages URL; `git push` redeploys it.

## Phase 2 — Folder access
- "Open folder" button (user gesture) → `showDirectoryPicker({ mode: "readwrite" })`.
- Iterate `dir.entries()`, filter `.md`.
- **Done =** clicking the button picks a folder and the app can list its `.md`
  files.

## Phase 3 — Persist the handle
- `idb-keyval` → store the directory handle in IndexedDB.
- On reload: `queryPermission()` → if not `granted`, `requestPermission()`
  (one click). Watch for Chrome's "allow on every visit" persistent grant.
- **Done =** after a reload the previously chosen folder is remembered; access is
  restored with one click (or zero, with a persistent grant) — no re-picking.

## Phase 4 — Read/write `start`
- Folder empty → create `start` (`start.md`); otherwise open it.
- Load content into CodeMirror; edit; save via `getFileHandle` + `createWritable()`.
- **Done =** edit `start`, save to disk, close and restart the browser → the
  changes are there.

> Open: **save strategy** — debounced autosave on change vs explicit save. For
> quick-capture, debounced autosave is the right feel (zero friction); decide in
> this phase.

## Phase 5 — Typography polish
- Clean, readable editor styling (font, line spacing, content width). Must not
  be ugly — but no syntax hiding yet, that is M2.
- **Done =** the editor is pleasant to write in (your call).
