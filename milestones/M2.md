# M2 — The editor experience (the heart of the product)

**Goal:** Notion-like feel — markdown markup **never visible**, text reads as
rich content. Markdown stays purely as the on-disk serialization format. This is
the differentiator over `papier`, so it comes before multi-note.

Success: writing in Brulion feels like writing rich text, yet the folder still
holds plain, clean markdown the user owns.

## Progress
- [ ] Phase 1 — Hidden-syntax rendering
- [ ] Phase 2 — Keyboard shortcuts
- [ ] Phase 3 — Slash commands
- [ ] Phase 4 — Right-click formatting popup

Phases are ordered to **de-risk the heart first**: the rendering engine (Phase 1)
is the foundation everything else manipulates. The three input methods
(shortcuts, slash, context menu) all reshape the same underlying markdown text
and reuse one set of pure transform functions, so they build on Phase 1 and on
each other.

Constraints that apply throughout:
- **File fidelity is the moat.** The document *is* the markdown text; decorations
  only hide and style it. No transform may write anything that isn't clean,
  portable CommonMark (this is why underline is out — see `DECISIONS.md`).
- **Never reveal markup on the cursor line.** Unlike Obsidian, syntax stays
  hidden even where the cursor sits — no flicker.

## Phase 1 — Hidden-syntax rendering
- Walk the Lezer markdown syntax tree; decorate the visible viewport.
- Hide markup with `Decoration.replace`: heading markers (`#`…), emphasis marks
  (`*`/`_`, `**`/`__`), inline-code backticks. Style the content with
  `Decoration.mark`: heading sizes, bold, italic, monospace inline code.
- Hidden runs are atomic (`EditorView.atomicRanges`) so the caret never snags on
  invisible characters.
- Markup is hidden on **every** line, the cursor line included.
- **Done =** typing `# Title`, `**bold**`, `*italic*`, `` `code` `` renders as
  styled rich text with no visible markers — even with the caret on the line —
  while the on-disk file still contains the raw markdown.

## Phase 2 — Keyboard shortcuts
- `Ctrl+B` / `Ctrl+I` / `Ctrl+E` toggle bold / italic / inline code around the
  selection (or insert empty markers at the caret).
- `Ctrl+↑` / `Ctrl+↓` cycle the current line's heading level: ↑ promotes
  (paragraph → H3 → H2 → H1), ↓ demotes back down to a plain paragraph.
- `Ctrl+Shift+1/2/3` jump directly to H1/H2/H3.
- Transforms are **pure functions** on (text, selection) → (text, selection),
  unit-tested in isolation and reused by Phases 3 and 4.
- Verify no clash with a CodeMirror/browser default binding.
- **Done =** each shortcut edits the underlying markdown correctly and the result
  renders hidden; toggles are reversible (apply twice → back to plain).

## Phase 3 — Slash commands
- Typing `/` at the start of a line opens a small popup menu: `/h1`, `/h2`,
  `/h3`, `/clear` (strip to paragraph).
- Keyboard-navigable (↑/↓/Enter/Esc); picking an item reshapes the current line
  and removes the `/command` token.
- **Done =** `/h1`+Enter turns the line into an H1; `/clear` strips it back to a
  paragraph; the menu filters as you type and dismisses on Esc/blur.

## Phase 4 — Right-click formatting popup
- A custom `contextmenu` popup over the editor with: Bold, Italic, Code, H1, H2,
  H3, Clear formatting.
- Operates across a **multi-line** selection (e.g. turn five selected lines into
  H2, or bold a span that crosses lines).
- Reuses the Phase 2 transforms.
- **Done =** selecting several lines and choosing a format applies it to all of
  them; dismisses on pick / Esc / outside click. No always-visible toolbar.

## Out of scope (deliberately)
- Lists, blockquotes, links, images, tables — not part of the M2 inline/heading
  core. Revisit per-feature later.
- Underline (`Ctrl+U`) — rejected; see `DECISIONS.md`.
