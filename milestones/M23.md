# M23 — Frontmatter (visual)

**Goal:** stop a leading `---…---` frontmatter block from rendering as ugly raw
code at the top of a note. Render it as a discreet, collapsed "metadata" region
that expands to show/edit the raw text. **Visual only** — moat-critical that the
bytes are never touched (no parse-and-reserialize, no field interpretation).

Phases:

- [ ] **Phase 1 — Collapsed/expandable frontmatter rendering** (`FEAT-0042`)

## Phase 1 — Collapsed/expandable frontmatter rendering

A new rendering layer (`frontmatter.ts`) on top of the M2/M5 hide-and-style
foundation. A **pure detector** recognises a leading frontmatter block —
line 1 is exactly `---`, closed by a later line that is exactly `---` or `...` —
and returns its byte range; only a *closed* block renders (a `---` still being
typed stays raw, like an unclosed fence). A CodeMirror `StateField` (block
decorations change layout, so a field, not a viewport plugin) renders it:

- **Collapsed (default)** — the whole block is replaced by a discreet, clickable
  `▸ metadata` chip; the raw lines are hidden but the bytes stay in the document.
- **Expanded** — clicking the chip reveals the raw text in a subtle box with a
  `▾ metadata` header to collapse again; the user edits the raw frontmatter
  directly. No `---` is hidden, nothing is reparsed.

Collapse state lives in the field and **resets to collapsed on every programmatic
note load** (so switching notes never carries one note's expanded state to the
next). The region is **opaque**: no `title`/`tags`/`aliases` interpretation, no
reserialize — decoration only.

**Done =** a note that opens with a leading `---…---` block shows a collapsed
`metadata` chip instead of raw fences; clicking it expands to the raw text and
clicking the header collapses it again; a block with no closing delimiter stays
raw; saving the note round-trips the exact original bytes (frontmatter included);
switching notes reopens collapsed. Covered by unit tests (detector + field) and
an e2e test against the real editor (collapse/expand + byte fidelity).
