# M12 — Quick switcher & note creation (Ctrl+K / Cmd+K)

**Goal:** one keyboard-first surface to find and create notes. `Ctrl+K` (`Cmd+K`
on mac) opens a quick switcher: fuzzy-search the note list, arrow/Enter to jump.
When the query matches no note, Enter **creates** a note with that name. This also
retires the weak sidebar inline-create textbox — the switcher becomes the single
way to create a note. Pure UI on the existing note model (list/switch/create);
**no change to file behavior** beyond the create the user explicitly asks for.

Success: `Ctrl+K`/`Cmd+K` opens an overlay focused on a search box; typing filters
the notes fuzzily; ↑/↓ moves the highlight and Enter opens the highlighted note;
a query that matches nothing offers "Create …" and Enter makes+opens it; Esc
closes with no change; the old sidebar new-note textbox is gone.

Constraints that apply throughout:
- **Moat untouched.** The switcher only reads the in-memory note list and calls
  the existing `switchTo`/`addNote`; the sole disk write is the note the user
  creates (the same path the textbox used). Nothing else is read, written, or
  cached.
- **No regression.** Note listing/switching/deletion, the sidebar tree, links,
  Vim, the conflict UX, and the poller keep working. The switcher must open with
  Vim on, in any mode.
- **Lean.** Hand-rolled fuzzy match (no new dependency), one overlay module
  following the existing `conflict-view` modal pattern, reusing `normalizeNoteName`
  for create validation. Vanilla TS + CSS.

## Phase 1 — The quick switcher overlay + create-on-miss
A `Ctrl+K`/`Cmd+K` shortcut opens a centered modal overlay with a single text
input and a results list. The list ranks the known notes by a fuzzy match of the
query (empty query = all notes); ↑/↓ move the highlight, Enter opens the
highlighted note via the existing switch path, a click does the same, Esc (or a
backdrop click) closes it with the open note unchanged. When the query matches no
existing note, the list shows a "Create «query»" row; choosing it normalizes the
name (the same validator note creation already uses) and creates + opens the note,
surfacing a validation error inline instead of creating on a bad name. The sidebar
`#new-note` form/input is removed along with its `wireNewNote` glue — creation now
lives only in the switcher.

- **Done =** `Ctrl+K`/`Cmd+K` (with Vim on or off) opens the overlay focused on the
  input; typing fuzzily filters the notes; ↑/↓ + Enter opens a note and closes the
  overlay; a no-match query offers Create and Enter makes+opens that note (an
  invalid name shows an error, creates nothing); Esc/backdrop closes with no
  change; the sidebar textbox is gone; a unit test covers the fuzzy ranking and an
  e2e covers open → filter → Enter (switch) and open → type new name → Enter
  (create).

## Out of scope (deliberately)
- **Full-text / content search** — the switcher matches note *names/paths*, not
  body text; searching inside notes is a separate, later concern.
- **Command palette actions** (delete, rename, settings from the switcher) — this
  is note find/create only, not a general command runner.
- **Recency/frecency ranking** — rank by match quality (and name), not by a usage
  history we don't keep yet.
