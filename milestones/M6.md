# M6 — Editor comfort

**Goal:** make daily writing nicer. Lean additions that live in the CodeMirror
layer; **none of them touches the file format** — the on-disk bytes stay exactly
the CommonMark the user typed (the file-fidelity moat). The work is split into
three independent phases, ordered so the M5-review debt is paid first.

Success: a bare `*`/`-` no longer makes the caret and the bullet glyph disagree
while you type; the note list can be collapsed for an editor-only view and that
choice survives a reload; and an opt-in Vim layer is available without breaking
the slash/format commands.

- **Lean.** Reuse the existing decoration/persistence plumbing; add the smallest
  thing that holds.

## Phase 1 — Bullet caret/glyph sync (M5-review debt)
The bullet renderer (`markdown-render.ts`) hides the `*`/`- ` marker as an atomic
range and draws a `•  `/`–  ` glyph with a line `::before`. Because the glyph and
the hidden marker have different widths — and the glyph carries trailing spaces
the document doesn't — the caret and the rendered bullet drift apart **while the
marker is mid-typing**: type a bare `*` and the caret looks one space in, but the
doc is still just `*`, so the next character lands *before* the `*`, which pops
back into view at the line start.

Fix: render the bullet as a **fixed-width widget that replaces the actual `*`/`- `
run**, sized to the marker, instead of a `::before` glyph layered over an
atomically-hidden marker. The widget occupies the marker's document range, so the
caret position and the glyph stay in sync as the line changes. Follow the heading
precedent: only render the bullet once the marker is **completed by a trailing
space** — a bare `*`/`-` with no space yet stays visible, so there is nothing to
disagree with while typing the marker itself.

- **Done =** typing `*` then a space shows a stable bullet with the caret at the
  content position (no pop-back, no width jump); a bare `*`/`-` with no trailing
  space renders as a literal marker; `*` and `-` still read as distinct glyphs;
  unit tests assert the marker run is replaced (not hidden-plus-`::before`), and
  an e2e test confirms the caret sits after the bullet, not before the marker.

## Phase 2 — Collapsible note sidebar
The left note list (M3) is always visible. Add a toggle to hide/show it for a
distraction-free, editor-only view, and **remember the collapsed state** across
reloads (same persistence layer the app already uses — IndexedDB via
`idb-keyval`, or `localStorage` if that is lighter for a single boolean; pick the
leaner one and record why). A keyboard shortcut and an affordance to toggle it.

- **Done =** a control collapses and restores the sidebar; the editor takes the
  full width when collapsed; the state persists across a reload; unit tests cover
  the toggle + persistence, an e2e test covers the collapse/restore round-trip.

## Phase 3 — Opt-in Vim mode
Add an **opt-in** Vim keybinding layer via `@replit/codemirror-vim`. Off by
default (the audience is quick-capture, not Vim users by default); a toggle
enables it and the choice persists. Crucially it **must not fight the existing
commands**: the slash menu, the format shortcuts (Ctrl+B/I/E, Ctrl+↑/↓), and the
markdown-aware Enter (FEAT-0018) must still work when Vim mode is on — verify the
precedence so Vim's normal/insert handling doesn't swallow them.

- **Done =** a toggle turns Vim mode on/off and the choice persists; with Vim on,
  insert-mode typing plus the slash/format/Enter commands still work; with Vim
  off, the editor behaves exactly as before; tests cover the toggle + persistence
  and that the custom commands survive Vim mode.

## Out of scope (deliberately)
- **Multi-level / nested list rendering** — still M5's flat-list scope; Phase 1
  only fixes the caret/glyph sync on the flat case.
- **Vim ex-commands / macros polish** — ship the stock `@replit/codemirror-vim`
  behavior; deep Vim customization is not a quick-capture need.
- **Per-pane / multi-window layout** — the sidebar toggle is one boolean, not a
  resizable/dockable panel system.
