# M32 — Link section anchors (`#anchor`)

**Goal:** support a section anchor in a note link — `[text](note#section-title)` and
the wikilink equivalent — resolving to the target note **and** scrolling to that
heading. Extends M8 (links); reuses the existing resolvers, adds anchor parsing and a
scroll-to-heading step. Moat-neutral (still plain markdown the user owns).

Scope (decided autonomously, recorded in `DECISIONS.md`):

- **`#` splits an internal link target** into note path + section anchor (markdown
  `[t](note#sec)` / `[t](sub/note#sec)` / same-note `[t](#sec)`, and wikilink
  `[[note#sec]]` / `[[note#sec|alias]]` / `[[#sec]]`). External URLs keep their
  `#fragment` untouched.
- **Heading match by a Unicode-aware slug** (lower-cased, punctuation dropped, `_`
  kept like GitHub, whitespace → hyphens); first matching heading wins; a missing
  heading is a silent no-op.
- **Scroll by scanning the document's heading lines** (parse-independent; skips
  fenced code), bringing the heading to the top + placing the caret. A same-note
  anchor jumps without switching notes.

Out (deliberately): heading-name autocomplete when typing `#`; duplicate-heading
disambiguation (GitHub `-1`/`-2`); interpreting external-URL fragments as note anchors.

## P1 — Link section anchors

Pure helpers `splitAnchor` + `headingSlug` (`note-name.ts`); anchor split off internal
targets in `markdown-render.ts` (carried in `data-anchor`, external URLs left whole);
`scrollEditorToHeading` + click-handler/option threading in `editor.ts`; anchor-aware
follow (switch-then-scroll, or same-note scroll) in `main.ts`. Pure logic unit-tested;
the real follow + scroll covered by e2e.

**Done =** following `[t](note#sec)` / `[[note#sec]]` switches to the note and scrolls
to the heading; `[t](#sec)` jumps within the open note; matching is slug-based and
case/Unicode-correct; a missing heading opens the note without erroring; external
`#fragment`s are untouched; no note bytes change.
