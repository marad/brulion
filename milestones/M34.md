# M34 — Motion & fluidity

**Goal:** make Brulion *feel* fluid. Today every state change is a hard cut —
overlays pop in, the sidebar snaps shut, the theme flips in one frame. This
milestone adds a small, cohesive motion system so those changes read as smooth and
calm, without ever touching the file format or the editor's text.

Pure presentation. **Moat-irrelevant**: no `.md` bytes change, and motion stays on
the *chrome* (overlays, sidebar, buttons, tree, theme), never on CodeMirror content
(text, caret, decorations) — animating the editor's heart is the easiest way to make
typing feel janky, so it's explicitly out of scope.

Scope (decided autonomously, recorded in `DECISIONS.md`; reviewed live at the end):

- **One motion system, token-driven.** Durations/easings live as CSS custom properties
  on `:root` (`--motion-fast/medium/slow`, `--ease`, `--ease-out`), so the whole app's
  tempo is tuned in one place — exactly what the live review needs.
- **Two gates, one mechanism.** The tokens resolve to ~0s until `html.motion-ready` is
  set (after the first paint), so nothing animates on load (no welcome/theme flash,
  re-litigating the FEAT-0031 fight). `prefers-reduced-motion: reduce` forces the tokens
  back to ~0s even when ready — so honouring the OS setting and the load-gate share one
  lever, and every transition reads it for free.
- **Overlays animate via modern CSS, no JS choreography.** `@starting-style` +
  `transition-behavior: allow-discrete` animate enter *and* leave for the
  `[hidden]`-toggled overlays (palette, switcher, settings, conflict, welcome) with zero
  changes to their show/hide JS. Chromium-only is fine — Brulion is Chromium-only (FSA).
- **Right-sized process.** This is presentation-only with no new module and no
  file-fidelity surface, so the three phases below ride **one** spec (`FEAT-0068`) rather
  than three near-empty CSS specs; the small JS hooks (motion-ready, the resize-drag
  guard, the selection-toolbar `hidden` toggle) are the testable parts and are covered.

## Phases

- [ ] **P1 — Motion foundation + overlay enter/leave** (`FEAT-0068`)
- [ ] **P2 — Structural motion: sidebar + folder tree** (`FEAT-0068`)

## P1 — Motion foundation + overlay enter/leave

The shared system plus the biggest perceptual win — overlays stop popping.

- Motion tokens on `:root`; the `html.motion-ready` first-paint gate (a 2×rAF hook in
  `main.ts`); the `prefers-reduced-motion` override.
- Every `[hidden]`-toggled overlay backdrop cross-fades; its dialog fades + rises/scales
  in and fades out, via `@starting-style` + `allow-discrete`. Covers the command palette,
  quick switcher, settings modal, conflict modal, and welcome screen.
- The dynamically-created surfaces (context menu, autocomplete/slash popup) get a free
  enter animation from `@starting-style`.

**Done =** opening/closing any overlay fades/rises instead of cutting; nothing animates
on first paint; `prefers-reduced-motion` removes all of it. Existing e2e overlay specs
still pass (visibility assertions unaffected by the fade).

## P2 — Structural motion: sidebar + folder tree

- **Mobile drawer** slides in/out (`translateX`) while the backdrop cross-fades.
- **Desktop collapse/expand** animates the sidebar's `flex-basis` + opacity, with a
  `resizing` class on `#sidebar` (set by `wireSidebarResize` during a drag) that disables
  the transition so dragging the divider never lags.
- **Folder tree** reveals its children with a gentle fade/rise on expand.

**Done =** toggling the sidebar (Ctrl+\, the ☰, the mobile drawer) animates smoothly;
dragging the resizer is still instant (no lag); reduced-motion makes all of it instant.

## P3 — Micro-interactions + theme cross-fade

- Buttons, note rows, folder headers, the delete-X, and the active-note highlight
  transition their hover/active/background states (no more instant colour snaps).
- **Theme swap** cross-fades the chrome colours (background/text/border) instead of a
  hard flip — gated behind `motion-ready` so the initial theme apply doesn't flash.
- **Selection toolbar** stops toggling inline `style.display` and uses the `hidden`
  attribute, so it animates like every other overlay.

**Done =** hovering controls and switching theme feel smooth; no first-load flash; the
selection toolbar fades in; reduced-motion keeps everything instant.
