# M28 — Mermaid diagram rendering

**Goal:** a fenced ```` ```mermaid ```` code block renders as the diagram it
describes, instead of showing raw Mermaid source. Same family as M5 (rendering
gaps), M15 (code highlighting), and M26 (tables): **visual only** — moat-critical
that we do NOT touch the bytes. The diagram source stays a plain fenced code block
in the file; we decorate how it paints.

Scope (decided autonomously, recorded in `DECISIONS.md`):

- **Render fenced `mermaid` blocks as diagrams.** Detect a closed `FencedCode` node
  whose info string is `mermaid`, replace the block with a rendered SVG widget
  (`Decoration.replace({ widget, block: true })`), source preserved verbatim on disk.
- **Lazy-load the engine.** Mermaid is large; load it via a `import("mermaid")`
  singleton so it stays out of the main bundle and only loads when a note actually
  contains a Mermaid block — the editor stays instant for notes that don't. The
  runtime-caching service worker (FEAT-0029) caches the chunk for offline use.
- **Selection reveals the source.** When the selection/cursor is inside the block,
  show the raw fenced source (so it's editable); otherwise show the diagram —
  reusing the established selection-overlap reveal pattern. On a parse error, show
  the error in place rather than a broken render.
- **Async widget done right.** `toDOM()` returns a placeholder synchronously, the
  diagram renders asynchronously into it; the widget's `eq()` compares the source so
  an unrelated edit elsewhere never re-runs Mermaid for an unchanged block.

Out (deliberately): a Mermaid **authoring** aid (live-preview pane, snippet menu) —
this is render-on-display only, like the other M-rendering milestones.

## P1 — Lazy-loaded Mermaid block rendering

A new `src/mermaid-render.ts` extension (a `StateField` emitting a block
`Decoration.replace` widget, registered in `editor.ts` after `markdownRendering`),
a lazy `import("mermaid")` loader, and the async `MermaidWidget`. Detection mirrors
the existing fenced-code path (Lezer `FencedCode` + info string). Pure pieces
(block detection, selection-reveal decision, loader singleton) are unit-tested with
Mermaid itself not invoked; the real SVG render is covered by an e2e in Chromium.

**Done =** a note containing a ```` ```mermaid ```` block shows the rendered diagram
in the editor; clicking into the block reveals the raw source for editing and moving
out re-renders; an invalid diagram shows an error in place, not a crash; the Mermaid
engine is a lazily-loaded chunk (absent from the main bundle); the on-disk bytes are
unchanged.
