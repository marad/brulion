# M22 — Copy fidelity

**Goal:** copying a selection must yield well-formed markdown that reproduces the
formatting *visible in the selection*, fixing the loss that happens at selection
boundaries. Editor/clipboard layer; the on-disk file is untouched. One cohesive
change, one phase.

The bug: CodeMirror's default copy returns `sliceDoc(from, to)` — the raw source
*inside* the selection. Because markdown markup is rendered as **atomic, hidden**
ranges, the caret can't land inside a `# `/`**`/`` ` `` run, so a selection that
starts from a heading's visible text begins *after* the hidden `# ` and the slice
drops it (paste is no longer a heading); a selection starting or ending inside a
bold span drops one or both `**` (lost formatting or malformed markdown).

The loss is purely at the **boundaries**: the interior of the slice is raw source
and already carries every complete marker. So the fix is boundary repair, not a
re-serialization of the whole selection.

- **Line markers pulled in for a partially-selected first line.** When the
  selection's start sits past a line's leading construct marker (heading `#`,
  blockquote `>`, unordered-list `*`/`-`), prepend that marker so the pasted line
  keeps its block role.
- **Inline markers synthesized around the fragment.** When the selection's start
  is inside a bold/italic/inline-code span (past its opening mark) the opening
  delimiter is prepended; when its end is inside such a span (before the closing
  mark) the closing delimiter is appended — reading the *actual* delimiter from
  the document (`**` vs `__`, `*` vs `_`, the backtick run). Nested spans are
  repaired outermost-first on the open side, innermost-first on the close side.
- **Cut** copies the same repaired markdown, then deletes only the selected range
  (never the synthesized markers — they live outside the selection, in the file).

**Links and wikilinks are already handled** and out of scope: markdown-render
reveals their raw `[text](url)` / `[[target]]` markup whenever the selection
touches them (FEAT-0026), so the raw markdown is already inside the slice.

## P1 — Copy/cut re-serializes the selection to valid markdown

A new pure serializer (`copy-markdown.ts`) turns `(state, ranges)` into the
clipboard markdown by repairing the two boundaries of each selected range against
the syntax tree, plus a thin `copy`/`cut` DOM-event extension wired into the
editor that sets the repaired text on the clipboard and (for cut) deletes the
selection.

**Done =** copying from a heading's visible text yields `# …`; copying a fragment
inside a bold/italic/code span yields the fragment wrapped in its delimiters;
copying across a span boundary keeps the markdown well-formed; a full-construct
selection is byte-identical to the raw source (no over-reach); cut copies the same
and removes only the selected text; unit tests cover the boundary-repair cases and
an e2e spec reads the real clipboard after a copy.
